# I set all 376 Vim options and I'm still a fool - Vimの376個の設定を全部いじっても、私はまだ下手くそだった
思わず試したくなる！「全部設定してみた」から分かるVim習熟のリアル

## 要約
Vim（とNeovim）の全376オプションを読み切って設定してみた著者は、エディタの理解は深まったが「真の流暢さ」は別問題だと結論づける。設定作業で得た実践的な気づきが多い。

## この記事を読むべき理由
設定やカスタマイズで「万能になる」と思いがちな人、あるいはVim/Neovimを企業で使う日本のエンジニアが「どこを押さえれば本当に効くか」を知るための近道になるから。設定だけでは解決しない学習の本質がわかります。

## 詳細解説
- 全オプションを読むというアプローチ  
  著者は全376のオプションを逐一理解して .vimrc / init.vim に落とし込みました。設定の目的や副作用を理解する過程で、Vimの内部挙動（ファイル書き込みの仕組み、バックアップの扱いなど）を深掘りしています。

- 外部コマンドとの連携  
  バッファのテキストを外部コマンドに渡したり、外部ツールの出力をバッファに取り込む方法（:help filter、:help write_c）を再発見。エディタから離れずに整形・置換を行えるのは効率向上に直結します。

- ファイル書き込みの複雑さ  
  単に :w するだけに見えて、Vimは既存ファイルのバックアップ作成や一時ファイルを使う挙動など細かい仕様があります。意図せぬロールバックや権限問題が起き得るため、バックアップや書き込みオプションの理解は重要です。

- コマンドラインウィンドウ（q:）の有用性  
  誤って開く人が多いこのウィンドウはコマンド履歴の編集に強力。長いコマンドの修正や履歴検索に通常のVimキー操作が使えるため覚えておくと便利です。

- 隠れ機能：digraphや条件付き設定  
  CTRL-K で特殊文字（例：分数）を入力する digraphs のような地味だが便利な機能や、if exists('+option') のようにオプションが存在する場合だけ設定するテクニックで、VimとNeovimの差分にも対応できます。

- Vim と Neovim の差  
  Neovimは貼り付けの扱いが滑らか、Q が最後のマクロを繰り返す、任意コード実行の安全性向上、デフォルトのカラースキームが扱いやすい、など実用差があるため、企業の環境やプラグイン互換性を考慮して設定を分けるべきです。

- 結果：設定だけで「流暢」にはならない  
  著者は2900行近い設定を書いても、未だにミスタイプや非効率な操作をしてしまうと認めています。キーバインドや挙動を知ることは重要だが、繰り返しの練習と日常的な利用が不可欠です。

## 実践ポイント
- 最初は全部設定しようとしない  
  必要なオプション（表示系、検索、インデント、バックアップ）を優先して少数から始める。巨大な .vimrc は管理が難しい。

- 外部フィルタを活用する  
  範囲選択して :!cmd や :%!cmd で整形処理を行えば、エディタ内で完結するワークフローが作れる。

- ファイル書き込み設定を確認する  
  backup、writebackup、swapfile の挙動はプロジェクトや共有環境で影響する。会社のファイルサーバや権限ポリシーに合わせて設定しておく。

- 設定は条件付きで書く  
  if exists('+option') を使って、Vim/Neovimの違いでエラーが出ないようにする。例：Neovim専用の設定は has('nvim') 条件で分ける。

- コマンド履歴や長いコマンドは q: で編集  
  長い置換コマンドや複雑なコマンドは q: で直すとミスが減る。

- 基本操作を数値＋動作で覚える  
  j を20回押すより 20j、テキストオブジェクト（di( など）やマクロを習慣化することで体感速度が劇的に上がる。

- 設定はモジュール化して管理する  
  設定ファイルを分割（pluginごと、環境ごと）してGitで管理するとメンテナンスが楽になる。Neovim移行を考えるなら init.lua／init.vim を分ける。

短い結論：設定は理解と安全性を高めるが、最終的に効率を上げるのは「反復練習」と「使い続けること」。全部やりきる挑戦は知見を深める良い方法だが、まずは日常で効く小さな改善から始めよう。
