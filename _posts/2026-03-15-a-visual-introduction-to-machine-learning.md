# A Visual Introduction to Machine Learning - 機械学習の視覚的入門
たった図でつかめる！決定木で学ぶ「家データ」から都市を見分ける直感

## 要約
視覚化を通して「機械学習とは何か」を直感的に示す入門。散布図やヒストグラムで特徴（例：標高、平米単価）を観察し、決定木で境界（split）を見つけて分類する過程を解説する。

## この記事を読むべき理由
データの可視化と単純モデル（決定木）で、機械学習の核心──特徴、境界、過学習（overfitting）、汎化の概念を初心者でも感覚的に理解できるため。日本の不動産価格分析や初めての分類タスクにすぐ応用できる。

## 詳細解説
- 分類タスクの直感：ある家が「San FranciscoかNew Yorkか」を当てる問題は典型的な分類。標高や平米単価などの次元（特徴）が判断材料になる。  
- 可視化で発見：散布図やヒストグラムにより、どの特徴が区別に有効かを目で確認できる。例えば標高が高ければSFである可能性が高い、低標高では平米単価が有効になる、といった具合。  
- 決定木の仕組み：決定木は一連の if-then（フォーク）でデータを二分し、各分岐点を「split point」と呼ぶ。各分岐は部分集合の純度（homogeneity）を高める方向に選ばれる。純度を測る指標としてはジニ不純度や交差エントロピーが使われる。例：
  - ジニ不純度 $G = 1 - \sum p_i^2$  
  - 交差エントロピー $H = -\sum p_i \log p_i$  
- 再帰的分割：一度分割すると各枝で同様の最適分割を繰り返す（再帰）。木を深くすると訓練データに対する精度は上がるが、過学習のリスクが増す。  
- 訓練 vs テスト：訓練データでは完全に当てられても、新しい（未見の）データで性能が落ちる場合は過学習。汎化性能を確かめるために必ずテストデータや交差検証を使う。  
- トレードオフ：深い木はバイアス低・分散高、浅い木はバイアス高・分散低という、いわゆるバイアス/分散トレードオフに直結する。

## 実践ポイント
1. まず可視化：scatter、histogram、pairplotで特徴間の関係を確認する。  
2. シンプルに試す：scikit-learnのDecisionTreeClassifierで深さ制限（max_depth）を設定して学習してみる。  
3. 評価は複数指標で：Accuracyだけでなく混同行列、精度・再現率、ROCなどを確認する。  
4. 過学習対策：max_depth、min_samples_leaf、剪定（pruning）、交差検証でモデルの安定性を確かめる。  
5. ビジネスへの応用例（日本向け）：不動産の地域分類・価格帯推定、工場センサーの異常検知、求人の適合判定などでまずは決定木を可視化して説明性を確かめる。

短い読み物で機械学習の「なぜ」と「どこから始めるか」が掴めます。興味があれば次は過学習対策とバイアス／分散の話を学んでください。
