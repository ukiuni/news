# Building a Minimal Transformer for 10-digit Addition - 10桁の加算を学ぶ最小トランスフォーマー
トランスフォーマーに10桁の筆算を学ばせて「仕組み」を覗いてみる――最小構成で見えるモデルの本質

## 要約
トランスフォーマーを極小に設計し、10桁同士の足し算をシーケンス変換問題として学習させることで、注意機構や位置情報が「桁合わせ」「繰り上がり」をどのように表現するかを明らかにする話です。

## この記事を読むべき理由
小さなモデルで明確なタスク（多桁加算）を扱うと、学習された内部表現や注意の役割が直感的に追跡でき、モデル解釈・低リソース推論・教育用途に直結する実用的な知見が得られます。日本の開発現場でも、組み込みやオンデバイス推論での軽量化や挙動検証に役立ちます。

## 詳細解説
- 問題設定：2つの10桁整数をテキスト列（例: "1234567890+9876543210"）として与え、出力は合計の桁列を逐次予測する。トークンは数字0–9・区切り記号・終了記号などの有限語彙。
- モデル設計：極小のトランスフォーマー（層数やヘッド数、埋め込み次元を小さくしたもの）を使い、自己回帰的に1トークンずつ出力する設定が一般的。位置エンコーディングは桁対応（桁位置を明示）に重要。
- 学習と効率化：ランダムに生成した加算データを大量に与え、クロスエントロピーで学習。カリキュラム学習（短い桁数→長い桁数）やミニバッチで安定化を図ると収束が早い。
- メカニズム観察：注意重みを可視化すると、出力桁が入力のどの桁に注目しているかが見える。繰り上がりの処理は局所的な相互作用（隣接桁への情報伝播）と、特定ヘッドの論理的な役割分担として現れることが多い。
- 解釈上の示唆：加算のような明確なアルゴリズム的タスクは、トランスフォーマー内部に「アルゴリズム風の回路」が自発的に形成されることを示し、ブラックボックスではなく部分的に可解なモデルであることを示唆します。

## 実践ポイント
- データ生成（例、Python）
```python
# python
import random
def gen_pair(n_digits=10):
    a = random.randint(0,10**n_digits-1)
    b = random.randint(0,10**n_digits-1)
    src = f"{a}+{b}"
    tgt = str(a+b)
    return src, tgt
```
- まずはごく小さなモデル（層1〜2、ヘッド1〜2、埋め込み64）で試す。
- カリキュラム：桁数を徐々に増やすと学習安定化。
- 評価は「完全一致率」（全桁正解）を使うと実用性がわかる。
- 注意可視化や層ごとのアクティベーションを解析して、どのヘッドが繰り上がり処理に関与しているかを確認する。
- 応用例：軽量算術サブモジュール、教育用デモ、解釈可能性研究、オンデバイスの検証ケース。
