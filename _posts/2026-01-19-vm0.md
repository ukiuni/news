# Vm0 - Vm0（ブイエムゼロ）
自然言語で「仕事を走らせる」──AIワークフローを自動化する最短ルート

## 要約
VM0は「自然言語で定義したワークフロー」をスケジュール実行・サンドボックスで安全に動かせるプラットフォームです。Claude CodeやCodexなどのエージェントを隔離コンテナで回し、観測性や永続化も備えます。

## この記事を読むべき理由
- 日本でもAIエージェントや業務自動化の導入が進む中、実運用で重要な「安全性」「再現性」「可観測性」を兼ね備えた実験的エコシステムを手早く試せる数少ない選択肢だからです。
- リポジトリはCLIで5分程度で動かせる設計。プロトタイプからPoCまで素早く試せます。

## 詳細解説
- コア概念
  - 自然言語でワークフロー（スキル群や手順）を記述し、VM0がそれをエージェント（Claude Code、Codex等）として実行します。
  - 実行は「サンドボックス化されたコンテナ」で行われ、外部アクセスや実行ログ、ネットワークトラフィックが可視化されます。

- 主な機能
  - Cloud sandbox：隔離された実行環境でエージェントを安全に走らせる。
  - スキル連携：GitHub、Slack、Notion、Firecrawl等60以上のスキルを持ち、外部サービスと統合可能。
  - 永続化とバージョン管理：チャット継続、セッションの再開・フォーク・バージョン管理が可能で、開発・デバッグが容易。
  - 観測性：各実行のログ、メトリクス、ネットワーク可視化を提供し、本番運用に備えられる。
  - CLIによる素早いスタート：npmパッケージ @vm0/cli で初期化～実行まで簡単です。

- 技術スタックと現状
  - 実装は主にTypeScript。DockerコンテナやCLIの構成ファイルが揃っておりローカルとクラウド両方で動かせます。
  - リポジトリは頻繁にリリースされており、エコシステムは成長段階（サポートされるモデルやCLIが追加予定）。

## 実践ポイント
- 今すぐ触る（CLI最短手順）
```bash
npm install -g @vm0/cli
vm0 auth login
mkdir my-agent && cd my-agent
vm0 init
# エージェント定義を編集して
vm0 cook "run your workflow"
```
- セキュリティ運用の注意
  - サンドボックスとはいえ、外部接続や機密データの扱いは最小権限で。環境変数やシークレット管理を厳格に。
  - ネットワークポリシーやIP制限を活用し、実行ログの保管先を明確に。
- 日本での活用例
  - ドキュメント自動生成、問い合わせの一次対応（Slack連携）、CI周りの補助タスク、定期レポート作成の自動化など、業務プロセスの自動化にすぐ役立つ。
- 探索のすすめ
  - まずは非本番データでProof of Conceptを回し、ログ・観測性・復元性（resume/fork）を確認してから運用に移すと安全です。
- ライセンスと互換性確認
  - OSSライセンスを確認し、商用利用や組織ポリシーとの整合性を事前にチェックすること。

短時間でAIエージェントを試作して観測性と安全性を確認したい開発チーム、または業務自動化のPoCを手早く回したいプロダクト担当には、VM0は有力な選択肢です。興味があれば公式リポジトリをフォローし、CLIでまず一度動かしてみてください。
