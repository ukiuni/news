# If You See These Red Flags in an IT Interview — Run (Real Stories) - IT面接でこれらの赤旗を見たら逃げろ（実話）
面接でわかる「入社前の地雷」5選 — 時間と心を守るための見分け方

## 要約
面接は会社が候補者を評価する場であると同時に、会社の文化や組織力が透けて見える瞬間。放置すべきでない「赤旗」例とその意味を実話ベースで解説する。

## この記事を読むべき理由
日本でも採用競争とリモート化で面接の形が多様化する中、早期に悪条件やリスクを見抜く力はキャリアとワークライフバランスを守る上で必須だから。

## 詳細解説
- 採用が何ヶ月も停滞する（例：1年続いた選考）
  - 意味：プロセスが整理されておらず、プロジェクト運営も混乱している可能性が高い。合意形成やリソース配分に問題あり。
- 「仕事と生活のバランスをどれだけ重視する？」の過度な数値化
  - 意味：残業や常時の高負荷を前提にした文化。提示される期待値と実態が乖離している危険性。
- 面接中の性別や家庭状況をあてにする発言（例：配偶者が手伝っているか）
  - 意味：無意識の偏見や人権軽視が蔓延している職場環境の予兆。
- 面接側が脆弱性を認めず防御的（技術的指摘に対して開かれていない）
  - 意味：改善や学習を重視しない文化。セキュリティやコード品質が後回しになりやすい。
- 過剰な監視型のコーディングテスト（フルスクリーン・録画・カメラ必須・外部資料禁止）
  - 意味：候補者を「疑う」文化。実務スキルより記憶力や試験耐性を測るだけで、実際の仕事能力とは乖離しがち。

技術的観点では、ライブコーディングやTake-home課題、オンラインプロクターはそれぞれ目的と限界がある：
- ライブコーディング：スピードとプレッシャー耐性を測るが、設計力やチーム開発能力は見えにくい。
- Take-home：実務に近いが時間配分や評価基準の透明性が重要。
- プロクター監視：不信を招き、AI時代の実務性を正しく評価できないケースが多い。

## 実践ポイント
- 面接前にプロセス・担当者・期限を明確に聞く（曖昧ならリスクを疑う）。
- タスクの目的と評価基準を確認する（練習か実装のどちらが目的か）。
- 過剰な監視や不適切な質問は断る／記録を残す。
- 面接中の態度（尊重・説明責任・改善姿勢）を観察する。防御的なら要注意。
- 直感で「おかしい」と感じたら、オファーでも条件を厳しくチェックして断る勇気を持つ。

日本市場への短い補足：日本は長時間労働やハラスメントの問題が特に敏感なため、面接での小さな違和感が実務での大きな問題に直結することが多い。面接で境界を設定する習慣を持つことが若手にも重要。

どうしてもオファーが魅力的でも、面接で見えた「赤旗」は将来の負担を予告するサインになり得ます。
