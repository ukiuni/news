# The Code Review Economy Built its Foundations on Sand, and It's Grown too Large to Move - コードレビュー経済は砂の上に基盤を築き、動かせないほど大きく成長した（AIバブル短編）
社内の「レビュー通貨」が暴走すると何が起きるか？――AI時代の生産性幻想を暴く寓話

## 要約
社内で「優先レビュー券」を発行したことがきっかけで、小さな社内通貨が急成長し、やがて外部評価や買収、法務・会計問題にまで波及するという寓話を通じて、AIツールや生産性バブルのリスクを描いている。

## この記事を読むべき理由
AIや自動化ツールが「仕事を爆速化する」と喧伝される今、日本の企業文化（決済承認やレビューがボトルネックになりがち）と非常に相性が良い。だが、その便利さに飛びつくと、組織のインセンティブや法律・会計の境界を超えて危険な「バブル」を作る可能性がある。本記事は、そのメカニズムと防止策を初級者にも分かりやすく解説する。

## 詳細解説
- 発端：主人公が「優先コードレビュー鍵」を作り、レビュー待ちを金銭的に短縮する仕組みを社内で流通させる。最初は効率化装置だが、他部署が模倣・派生トークンを作りはじめる。
- トークン化の波：値付け、模造品、より高価な特典（有給やボーナスなど）への交換が生まれ、社内で流通する通貨（MFD）の供給量が急増。これがインフレや投機を招く。
- 市場化と仲介：社内のエンジニアが取引所と代替通貨のアービトラージ（裁定取引）を自動化し、流動性を生む一方で利害関係の集中や情報非対称が拡大する。
- 外部化の危機：MFDを外部で換算したり、住宅手当やボーナスに組み込んだ瞬間、社内通貨は現実世界の価値と衝突し、規制・会計・税務・労働法上の問題が発生する（寓話では上場取引所の話まで出る）。
- 本来の教訓：これらのトークンは「AI」や「自動化された優先化システム」のメタファー。AI導入で見かけ上の生産性が上がっても、根拠のない価値付けやガバナンスの欠如があれば、組織的なリスクと市場のバブルを生む。

技術的ポイントとしては、トークン経済の基礎（発行量・流通・可換性・仲介者の利益相反）と、エージェント的ボットが作る内部アービトラージ、及び「生産性の可視化指標」が操作可能である点に注意する必要がある。

## 実践ポイント
- AIはツールとして段階的導入：まずは限定的なパイロット運用とA/B評価を行い、「品質」と「安全性」を数値化して継続的に検証する。
- インセンティブ設計に注意：社内トークンやランキングで報酬を与える前に、法務・人事と協議し、賃金や手当との整合性を確認する。
- 発行量と流動性を管理：内部ポイントや優先権は中央で発行上限と有効期限を決め、インフレを制御する。
- ガバナンスと透明性：取引履歴、発行ルール、利害関係を公開し、第三者監査や監視ルールを設ける。
- 技術的防御（エンジニア向け）：レビュー自体の品質保証はCIテスト・自動解析・人間によるサンプリングレビューで担保し、AIの誤提案をそのまま信頼しない。
- 経営層への提言：短期的な生産性指標に踊らされず、長期的な価値創造・法務リスク・社員のモラルへの影響を評価する。

まとめ：便利な「優先化メカニズム」やAIが生み出す効率は、適切なルールと監査なしでは社内バブルを起こす温床になり得る。日本の現場では特に「文化的に見過ごされやすい」面があるため、導入時の慎重な設計と透明なガバナンスが不可欠だ。
