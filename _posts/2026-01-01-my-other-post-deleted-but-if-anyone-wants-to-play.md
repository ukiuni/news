# My other post deleted but if anyone wants to play a daily coding game I found this - 他の投稿は削除されたが、毎日遊べるコーディングゲームを見つけた
毎日3問×3言語で実力を磨く──忙しいエンジニアのための最短インターバルトレーニング

## 要約
The Coding Problemは「3問・3言語・毎日」を掲げる軽量なデイリープロブレムサービスで、短時間でアルゴリズム力と言語間の感覚を同時に鍛えられる点が特徴。

## この記事を読むべき理由
短時間で継続できる学習法を探す日本のエンジニアや転職志望者、社内研修担当者にとって、毎日少しずつ積むことで面接対策や実務スキルに結びつく実践的なアプローチだから。

## 詳細解説
- コンセプト
  - 「3 problems. 3 languages. Everyday.」というシンプルなルールは、問題数・反復頻度・マルチランゲージの練習を同時に実現する設計思想を示す。
  - 毎日こなせる小さな負荷にしているため、学習の継続率が上がりやすい点が利点。

- 何が鍛えられるか
  - アルゴリズム的思考：短い問題を複数解くことで様々なアルゴリズム／データ構造に触れられる。
  - 言語横断力：同じ問題を複数言語で解く機会があれば、言語の idiom（慣用表現）や標準ライブラリの使い分けが身につく。
  - コードの速さと質：短時間での実装→リファクタのループが回せるため、読みやすさとバグの減少に寄与する。

- サイト利用を想定したワークフロー（例）
  - 毎朝や就業後に20–40分を確保 → 3問に取り組む（言語は日替わりまたは同一問題で別言語）。
  - ローカルでまず解き、VS Codeなどにユニットテストを書いて実行 → リファクタ → Gitで履歴を残す。
  - 解説や他者のソリューションを読んで気づきをメモする。

- 注意点
  - 「毎日」を続けることが目的化しないよう、復習日や問題カテゴリの深掘り日を設ける。
  - 本番の面接で出る問題はより複雑なことが多いので、デイリー問題は導入／習慣化として位置づける。

## 実践ポイント
- 習慣化：まずは7日間続けるルールを作る。連続記録を可視化すると継続しやすい。
- 言語の選定：母語（日本語）で考える癖を捨て、実務で使う言語1本＋学びたい言語1本＋興味のある言語1本の組合せを試す。
- VS Code活用：タスクをワークスペースにまとめ、テスト→フォーマット→Lintの自動化を設定して短時間で回せる環境を用意する。
- 共有とレビュー：社内の勉強会やSlackで毎日の学びを共有してフィードバックをもらう。
- 面接対策との結合：デイリー問題で出たパターンをタグ分けし、週1回は「面接想定問題」を集中演習する。
