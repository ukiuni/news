# Native Linux VST plugin directory - ネイティブLinux VSTプラグインディレクトリ
魅力的タイトル: Linuxで使える音源・エフェクト探しが一瞬で終わる──ネイティブLinuxプラグインの最強カタログ

## 要約
Native Linux VST plugin directory は、Linuxネイティブ（VST2/VST3/LV2/CLAPなど含む）な音楽プラグインをカテゴリ・フォーマット・価格で横断検索できるデータベースサイトです。1500件以上のエントリで最新のLinuxオーディオ事情を俯瞰できます。

## この記事を読むべき理由
日本でも自宅スタジオやサウンドデザイン、教育用途でLinuxを使うケースが増えています。プラグイン互換性や導入の手間を減らし、すぐに試せるリストがあると作業効率と選択肢が飛躍的に広がります。

## 詳細解説
- カテゴリ体系: エフェクト（ディストーション、コンプレッサー、リバーブ等）とインストゥルメント（シンセ、ドラム、サンプラー）に細かく分類。用途別に絞り込めます。  
- フォーマット: VST2/VST3/LV2/CLAP/DSSI/LADSPA/standalone 等を表記。ネイティブ対応は互換性と低遅延の面で重要です。  
- 検索・並び替え: 更新順／名前などでソート、"Show untested"トグルで未検証プラグインの表示切替が可能。価格と無料表記も確認できます。  
- 情報の粒度: プラグイン名・開発元・簡単な説明・対応フォーマット・価格が一覧で確認でき、クリックで詳細へ飛べます。  
- 実務面の互換性: Linux上の主要DAW（Ardour、Reaper Linux版、Bitwigなど）やホスト（Carla、JACK）との相性確認が肝。特にLV2/CLAPはLinuxネイティブで安定感があります。

## 実践ポイント
1. 目的で絞る: 「コンプレッサー」や「リバーブ」などカテゴリでまず絞る。  
2. フォーマット優先: ネイティブ安定性を重視するならLV2/CLAP→VST3→VST2の順でチェック。  
3. テスト環境を用意: CarlaやArdourでまず単体テスト。JACK接続やサンプルレートで動作確認を。  
4. インストール場所: VSTは ~/.vst/、LV2は ~/.lv2/ に置いてホストに再スキャン。パーミッションに注意。  
5. 日本市場の感覚: 有料プラグインは為替・購入手続き（海外決済）を確認。商用利用やライセンス（フリー/トライアル）も要チェック。  

これだけ押さえれば、Linuxで「欲しかった音」をスムーズに探して試せます。サイトをブックマークして、まずはカテゴリ検索→Carlaで動作確認、という流れを試してください。
