# Raspberry Pi Pico 2 at 873.5MHz with 3.05V Core Abuse - Raspberry Pi Pico 2 を 3.05V コア電圧で 873.5MHz に挑戦
乾式氷で冷やして突き抜ける！Pico 2を“コア電圧ぶん回し”で873.5MHz達成の舞台裏

## 要約
Pimoroniの実験で、RP2350搭載のRaspberry Pi Pico 2を外部給電と乾式氷冷却で最大873.5MHzまで動作させた記録。電圧上げと冷却で性能を大幅に引き出す一方、安定性と熱・電流供給が制約になる。

## この記事を読むべき理由
Pi Picoシリーズは日本のホビー／教育現場やプロトタイピングで大人気。Pico 2（RP2350）のオーバークロック挙動、冷却・電源まわりの実測データ、RISC‑Vコアの挙動などは、性能トレードオフや実装設計に直結する知見です。

## 詳細解説
- 背景：RP2350はオンボード電圧レギュレータの制限を解除でき、1.3V以上のコア電圧を要求可能。これがオーバークロック余地につながる。
- 初期試験（簡易テスト）：MicroPythonで100!やベンチを実行し、電圧毎の最大安定クロックを探索。得られた代表値（空冷）：
  - 1.1V → 312MHz（約25.6°C）
  - 1.3V → 420MHz（約33.6°C）
  - 1.5V → 512MHz（約44.4°C）
  - 1.7V → 570MHz（約53.7°C）
- 冷却と給電改善：小型ヒートシンク＋ファンで温度低下。Pico 2背面の「Test Point 7」を使い外部電源でコア電圧を直接注入（オンボード供給を無効化）することで、より高電圧／高クロックを達成。
- 乾式氷（約−80°C）を用いた実験：外部電源で電圧を上げつつ冷却した結果、800MHz超が可能に。最終的に「3.05Vで873.5MHzを一時的に動作」させたが、長時間安定は難しく電流（≈1A）と発熱が制約となった。
- リングオシレータ：コア電圧・温度に追従するはずの自動オーバークロック機構だが、実測ではARMコアで不安定な挙動を示した（ドライブ強度調整で改善は可能）。
- RISC‑Vコア：同条件でCoreMarkを回すと、ARMより約5%高いMHzあたり性能を示した。整数演算主体の用途ではRISC‑Vの検討価値あり。
- 個体差：サンプル選別でオーバークロック耐性に差があるため「当たり」を選ぶ重要性あり。

## 実践ポイント
- 安全第一：高電圧・極低温の実験はリスク大。初心者は手を出さないか、専門環境で行うこと。
- 測定ポイント：背面のTest Point 7でコア電圧を監視／供給することで正確な電源制御が可能。
- 冷却：ヒートシンク＋十分なエアフローで安定性向上。乾式氷やLN2は経験者のみ。
- 電源経路：コア電圧のグラウンドは短く太い配線にして電圧降下を防ぐ（長い経路だと実効電圧が下がる）。
- ベンチ：CoreMarkや既知のベンチで両コア動作を確認し、エラー出現時はクロックを下げる。
- コア選択：整数中心の処理はRISC‑Vコアを試すと良い（MHz当たり性能が高い可能性）。

以上を踏まえ、Pico 2は小型・安価なボードながら「電源と熱管理」を制御すれば大きな性能伸びしろがある、というのが今回の核心です。
