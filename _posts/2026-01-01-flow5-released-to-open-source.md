# Flow5 Released to Open Source - Flow5 がオープンソース化してリリースノート公開
魅惑の“流体解析＋メッシュエンジン”が変わる──Flow5 v7.x 系列の進化とオープン化で何が変わるのか？

## 要約
Flow5 がオープンソース化され、Gmsh SDK 統合やモデル/ソルバーのライブラリ化、XFoil 連携など大幅なアーキテクチャ改善と使い勝手向上が進んでいます。複数バージョンのリリースノートは、メッシュ方式／解析手法／ファイル互換性の重要な変更点を明示しています。

## この記事を読むべき理由
日本の航空・船舶系エンジニアや研究者、ツールチェーンを自前で改良したい企業にとって、Flow5 のオープン化は“内部ロジックを読み・改変できる”チャンスです。Gmsh や XFoil といった既存ツールとの連携強化は、社内ワークフロー統合や教育用途にも直結します。

## 詳細解説
- オープン化のコア
  - flow5 は FOSS（v7.53 以降）となり、ソースレベルでの改変や内製化が可能に。これに伴い将来的に XFoil 連携でのオンザフライ粘性抵抗評価など、外部ライブラリを活かす機能が有効化されます。
- アーキテクチャ／ライブラリ化
  - モデラーとソルバーのクラスをライブラリへ分離。プラグイン化や他ツールからの再利用が容易に。
- メッシュ／CAD 周りの変化
  - Gmsh SDK にリンクし、全メッシュ・インタフェースが Gmsh と組み合わせて動作可能に。加えて独自メッシャーも併存。
  - ウィングの中翼弦（mid-camber）に合わせて fuselage メッシュを作るオプションなど、メッシュ整合性向上機能を追加。
  - OCC（Open CASCADE）依存の影響で IGES の入出力が削除された箇所あり。シンボル名変更に起因するため CAD ワークフローの見直しが必要。
- 解析機能の改善
  - LLT（線形翼理論）や誘導抵抗、誘導角の評価バグ修正。三角分割法の誘導抵抗計算の堅牢化や、局所誘導角追加など精度／安定性が改善。
  - XFoil 分析が有効化され、2D/3D 解析で粘性評価を組み合わせられる設計（ただし一部はオープンソース化後に有効化）。
  - マルチスレッドやバッチ解析の不具合修正で大規模解析の信頼性向上。
- UI / プラットフォーム
  - OpenGL デフォルトを 4.6 に引き上げ（以前は 3.3）、Wayland や Qt6 互換性の改善。Windows バイナリは Qt6.8.2 によりネイティブダークモードに対応。
  - ファイル読み書きのフォーマット変更が頻発しているため、v7.50 / v7.53 などで新規保存ファイルは旧バージョンで読めない点に注意。
- その他の影響
  - フラップやパネル接続の扱い変更により、微小ながらピッチングモーメントや慣性評価に影響する変更が散見される。設計再現性の確認が必要。

## 実践ポイント
- 既存プロジェクトの互換性確認
  - 新バージョンで保存した .fl5 は古いバージョンで読めないため、プロジェクト保存前にバックアップを必ず取る。
- ツールチェーン準備
  - Gmsh SDK と XFoil を導入・ビルド環境に組み込み、メッシュ生成や粘性評価の連携を試す。
- 検証テストを実行
  - フラップ/パネル接続や慣性評価（特に xfl-type fuselage）で評価値が変わる可能性があるため、既知ケースで差分比較を行う。
- 社内での利活用戦略
  - オープン化でライブラリ化されたモデル／ソルバーを社内ツールへ組み込む、教育用のカスタムケースを作るなどの活用が可能。
- 環境要件の確認
  - OpenGL 4.6、Qt6 周りのサポート状況を確認し、特に Linux + Wayland 環境での表示・互換性テストを実施する。
