# WebHaptics – Haptic feedback for the mobile web - モバイルWeb向けハプティック（触覚）フィードバック
魅力的なタイトル: スマホWebが触れる体験に変わる — 「WebHaptics」で作る指先のフィードバック

## 要約
WebHapticsはモバイルWeb上で触覚フィードバック（バイブや短いタップ感）を実現するためのアプローチと実例をまとめたプロジェクトで、ブラウザごとの制約を吸収してより豊かなUXを提供します。

## この記事を読むべき理由
日本はモバイル中心の利用が圧倒的で、PWAやWebベースのサービスでネイティブに近い体験を作るニーズが高い。触覚フィードバックは操作の確度や没入感を上げ、差別化につながるため、実装方法や注意点を知っておく価値があります。

## 詳細解説
- 背景：従来のWebは視覚と音声が中心だったが、モバイル端末の振動モーターやタプティック機能を使うことで「触れる」UXが可能になる。  
- ブラウザ事情：Vibration API（navigator.vibrate）は古くからあるが、ブラウザやOSごとに対応状況や表現力が異なる。例えばAndroidでは比較的利用しやすい一方、iOSではサポートや挙動に制約がある。  
- WebHapticsの役割：単一の抽象APIで機能検出（feature detection）→能力に合わせたマッピング→フォールバックを行う設計が中心。これにより開発者は「短いタップ／長めのタップ／強い振動」などの意図をコードで表現し、実機で最適な出力に変換できる。  
- デザインのポイント：触覚は強すぎると不快、弱すぎると意味を成さない。振幅・長さ・間隔の調整、視覚・音声との組合せ、設定で無効化できる配慮が重要。バッテリーやプライバシー（無断振動の悪用）にも注意する必要がある。

## 実践ポイント
- まずは機能検出を行い、対応しない環境では何もしない（プログレッシブエンハンスメント）。  
- 短いフィードバック（10–50ms）をクリックやスワイプの確認に使う。長い振動は注意や通知向けに限定。  
- ユーザー設定で触覚のオン／オフ、強度切替を用意する。  
- 実機で必ずテストする（AndroidとiOSで感じ方が大きく異なる）。  
- アクセシビリティを考え、振動だけに依存しないフィードバック（視覚・音声）を併用する。

この考え方を取り入れれば、PWAやモバイルWebサービスで指先に残る操作感を付与でき、ユーザー体験の差別化につながります。
