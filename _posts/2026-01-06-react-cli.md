# React CLI. - React CLI（React用コード生成CLI）
1ファイルでTypeScriptコンポーネントを量産する高速ボイラープレートジェネレータ

## 要約
React/TypeScriptのコンポーネント雛形をCLIで自動生成する軽量ツール。設定ファイルで出力形式を制御でき、CSS/SCSS/SASS対応やprops型の生成をサポートする。

## この記事を読むべき理由
プロジェクトでコンポーネントの雛形がバラつくと保守性が低下する。日本の開発チームでも、統一されたテンプレートを素早く作成できるツールはコードレビュー負荷低下とオンボーディング短縮に直結するため必読。

## 詳細解説
- 何ができるか  
  - コマンド名は rc（バイナリ名は react-cli）。Componentフォルダとファイルを自動生成する（例: UserList/UserList.tsx, UserList/UserList.css, UserList/index.ts）。  
  - スタイル形式は css / scss / sass / none を選択可能。style作成をスキップするオプションもあり。propsの型注釈を含む雛形を生成する。

- インストール方法（代表例）  
  - Unix/macOS: curl経由でインストーラを流し込む  
  - Windows: PowerShell経由で同様のインストールスクリプトを実行  
  - macOS/Linux向けにHomebrew tapあり。npmグローバルにも公開されている（react-cli）。

  例:
  ```bash
  curl -fsSL https://raw.githubusercontent.com/khotcholava/zhvabu-cli/main/install.sh | sh
  ```

- 主要コマンド
  - rc init : プロジェクトルートに react-cli.json を作成（デフォルト設定）
  - rc generate component <ComponentName> ["<props>"] : コンポーネント生成
  - rc version / rc --version : バージョン確認

  例:
  ```bash
  rc generate component UserList "userList, onActionClick"
  rc generate component Card --style scss --path src/components
  ```

- 設定ファイル（react-cli.json）の例
  ```json
  {
    "defaults": {
      "component": {
        "style": "css",
        "path": ".",
        "skipStyle": false,
        "componentStyle": "functional"
      }
    },
    "project": {
      "prefix": ""
    }
  }
  ```

- 実装とビルド
  - 実装言語は Go。バイナリを配布しており、ローカルでビルドする場合は Go 1.24 以上が必要。配布用ビルドは ./scripts/build.sh で行う。

## 日本市場との関連
- 日本の多くのフロントエンド案件は社内UIライブラリやスタイル規約を持つため、生成テンプレートを会社ルールに合わせるだけで即戦力になる。  
- VSCode中心の開発ワークフロー（ターミナル、出力パネル、テスト統合）との相性が良く、タスク登録でワンクリック生成→編集の流れが作れる。  
- Storybookやデザインシステムと組み合わせ、コンポーネントの質を均一化する運用が現場で効果的。

## 実践ポイント
- まずはインストールして rc init を実行し、react-cli.jsonをチーム規約に合わせて編集する。  
  ```bash
  rc init
  rc generate component Button "label, onClick"
  ```
- VSCodeでの導入例  
  - ターミナルから直接実行するか、.vscode/tasks.jsonにgenerateタスクを登録してショートカット化する。  
- 拡張案（現場で即導入できる工夫）  
  - テンプレートを社内ルールに差し替え（className命名規則、CSS Modules対応、テストファイルのひな形追加）。  
  - pre-commitフックでファイル名やプレフィックスを検証して命名規約を担保。  
- CIでのチェック  
  - 生成されたコンポーネントがビルド/型チェックを通るように、CIにrcで生成したコードの型チェック（tsc）やレンタリングテストを組み込む。

短時間でコンポーネントの標準化を進めたいチームにとって、まず試す価値のあるツール。拡張性が高いので社内テンプレートと組み合わせる運用を推奨する。
