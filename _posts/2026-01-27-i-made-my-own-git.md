# I made my own Git - 自作のGitを作ってみた
自作Gitで分かる！バージョン管理の「中身」を手を動かして理解する方法

## 要約
元記事は、Viteで作った簡易アプリを使って「自分でGitを実装してみた」挑戦記。オブジェクトストレージ、コミット構造、参照管理などGitの基本構成を自作を通じて学ぶ内容です。

## この記事を読むべき理由
Gitは日本の開発現場で圧倒的に多用されるツールだが、その内部挙動をちゃんと理解している人は少ない。自作を通じて仕組みを知れば、トラブルシュートやCI設定、パフォーマンス改善、面接対策などに直結する実践的な力が身につきます。

## 詳細解説
- 何を自作するか：最小限のGitなら「オブジェクト（blob/tree/commit）をハッシュで保存し、refで参照する」機能が核。元記事はViteでUIを作り、ブラウザ/Node上でこれらを実装して可視化している。
- 主要コンポーネント：
  - オブジェクトストア：ファイル内容をハッシュ（歴史的にはSHA-1）で名前付けして保存する「コンテンツアドレッサブルストレージ」。
  - tree/blob：ディレクトリとファイルの表現。ツリーをたどることで作業ツリーを再構築できる。
  - commit：親参照・ツリー参照・メタ情報（作者・時刻・メッセージ）を保持するノード。
  - refs/HEAD：ブランチやタグの参照。HEADが指す場所でチェックアウトやコミット操作が決まる。
  - インデックス（ステージング）：作業ツリーと次のコミットの橋渡し。
- 実装上の要点：オブジェクトの直列化とハッシュ計算、ツリー構築のロジック、コミット履歴の辿り方、マージやコンフリクトの基本的な考え方。パフォーマンス改善ではpackfileや差分圧縮が重要になるが、まずは正しい動作を優先する。
- Viteアプリの役割：ブラウザ上で操作の可視化（コミットの追加、ブランチ切替、履歴表示）を行うことで、抽象的な概念が直感的に理解できる点が効果的。

## 実践ポイント
- まずは最小実装を作る：blob -> tree -> commit -> ref の順に実装して、ファイルをコミットして履歴を表示できるようにする。
- 言語はNode.js/TypeScriptが手軽：ファイル操作・ハッシュ計算・簡易CLIやViteでのUIが組みやすい。
- 可視化を加える：コミットグラフやツリー表示を作ると理解が早まる（ViteやReactで簡単に実現可能）。
- 実際の.gitと比較する：自作と本家Gitの挙動を比べて差分を学ぶ（.gitフォルダの構造を覗く）。
- 学習の効果：内部構造を知ることでマージ戦略や大規模リポジトリの運用、CIキャッシュの最適化などに応用できる。

興味があれば、まず「blob → tree → commit」を実装して、小さなViteデモで可視化してみてください。
