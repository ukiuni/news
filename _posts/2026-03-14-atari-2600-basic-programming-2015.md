# Atari 2600 BASIC Programming - Atari 2600 BASIC プログラミング
64シンボルで動くゲーム!? Atari 2600 BASICが教える「極限のやりくり」

## 要約
Atari 2600用に作られたBASICは「プログラムは最大64シンボル・9行・整数0–99のみ」といった過酷な制約下で動作するインタプリタだが、その設計と実践例は今でも学びが多い。

## この記事を読むべき理由
レトロ趣味だけでなく、リソース極小環境（組込み機器やマイコン）での設計思想、コードゴルフ的な発想、アイデアを短く伝える工夫が学べるから。日本のホビー／組込みコミュニティにもヒントが多いです。

## 詳細解説
- 制約の要点
  - プログラムは最大64シンボル（1シンボル＝数値・変数・予約語など）
  - 最大9行、整数は0〜99、演算は + - * ÷ mod の5種
  - グラフィックは「横・縦座標の2点」を動かす程度、サウンドは簡易ビープ
  - 入力は2つの12ボタンパッド（実機は操作性が非常に悪い）
- 歴史的・実装的意義
  - インタプリタ自体が数十バイトレベルで収まるよう、Warren Robinettらが極限まで軽量化して実装。128バイトRAMのうち半分がプログラム表現に使われるなど、メモリ配分の工夫が際立つ。
- 実例と工夫
  - 単純なドット移動（キーで座標を±5するループ）や、短いサッカー風シミュレーション、ユークリッドの互除法、MVCの最小実装などが実際に動く。ツイート（当時140文字）に収めるために予約語の省略や表記圧縮を行う工夫も紹介される。
  - 例（ドット移動、簡略化版）:
```basic
1 If Key=2 Then Ver1=Ver1-5
2 If Key=4 Then Hor1=Hor1-5
3 If Key=6 Then Hor1=Hor1+5
4 If Key=8 Then Ver1=Ver1+5
5 Goto 1
```
  - 例（ユークリッドの互除法）:
```basic
1 Hor1=50,Hor2=30
2 If(Hor2=0)ThenGoto5
3 T=Hor2
4 Hor2=Hor1 Mod Hor2
5 Hor1=T
6 Goto 2
7 Print Hor1,Note=Hor1
```
- 実機 vs エミュレータ
  - 実機はキー配列の切替など操作が面倒。Stellaなどのエミュレータや、Nintendo DS版（タッチでキーパッドを再現）で試すのが現実的。

## 実践ポイント
- まずはエミュレータ（Stella）かDS版で動かしてみる。実機はコントローラ周りが厄介。
- 「64シンボル制約」を自分の練習課題に：コードゴルフ、限られたRAMでのアルゴリズム設計練習になる。
- 学んだテクニックは小型マイコン（Arduino/RP2040など）や、ゲームジャム・プロトタイプのアイデア圧縮に応用可能。
