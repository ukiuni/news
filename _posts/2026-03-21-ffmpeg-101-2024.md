# FFmpeg 101 (2024) - FFmpeg入門 101
FFmpegの中身がイッキにわかる！初心者でも使える「動画デコードの流れ」と実践Tips

## 要約
FFmpegはコマンドラインツール群とライブラリ群から成るマルチメディア処理の基盤で、ファイル解析・デマックス（demux）→デコード→加工→再配信までをプログラムから扱える。

## この記事を読むべき理由
日本でも動画配信、リアルタイム処理、エンコード自動化の需要が高く、FFmpegは放送・VOD・Web配信・組み込み機器で事実上の標準。基本構造と最小限の実装手順を押さえれば、自分のサービスで即戦力になる。

## 詳細解説
- 構成要素  
  - ツール: ffmpeg（変換/トランスコード）、ffplay（簡易プレイヤ）、ffprobe（解析）  
  - ライブラリ: libavformat（入出力・mux/demux）、libavcodec（エンコーダ/デコーダ）、libavfilter（フィルタグラフ）、libswscale/libswresample（画面/音声変換）など。

- 基本的な処理フロー（アプリからの利用）  
  1. AVFormatContextでファイル/ストリームを開く（avformat_open_input）  
  2. ストリーム情報を取得（avformat_find_stream_info）→ AVStream配列を調査して映像/音声を特定  
  3. codec_idからavcodec_find_decoderでデコーダを選択し、AVCodecContextを初期化・開く（avcodec_alloc_context3／avcodec_open2）  
  4. av_read_frameでAVPacketを取り出し、対応するストリームだけをavcodec_send_packet→avcodec_receive_frameでデコードしてAVFrameを得る  
  5. 後処理（色変換、リサンプル、フィルタ適用、再エンコード／描画）を行う

- 重要なデータ構造（初心者向け）  
  - AVFormatContext: ファイル/コンテナ全体のハンドル  
  - AVStream: 個々のストリーム（video/audio）情報（time_base、codecparなど）  
  - AVPacket: エンコード済みパケット（demuxの単位）  
  - AVFrame: デコード後の生データ（生のフレーム／サンプル）

- 実装時の注意点  
  - デコーダはキュー式（send/receive）で非同期動作を考慮する。EAGAIN/EOFの扱いを正しく。  
  - コーデック・コンテナの四文字コード（FourCC）やtime_base/framerateを参考にタイミングを扱う。  
  - ビルドにはシステムのFFmpegとABIの整合が必要。ハードウェアアクセラレーション（VAAPI、NVENC、V4L2等）やライセンス（GPL/LGPL）も設計段階で確認。

- 簡単なコード断片（ファイルを開いてストリームを列挙）
```c
// c
AVFormatContext *fmt = avformat_alloc_context();
avformat_open_input(&fmt, filename, NULL, NULL);
avformat_find_stream_info(fmt, NULL);
for (unsigned i=0; i<fmt->nb_streams; ++i) {
  AVStream *s = fmt->streams[i];
  printf("Stream %u: type=%s\n", i, av_get_media_type_string(s->codecpar->codec_type));
}
avformat_close_input(&fmt);
```

- ビルド／実行（例）
```bash
# bash
pip3 install meson ninja
meson setup build .
ninja -C build
./build/ffmpeg-101 sample.mp4
```

## 実践ポイント
- まずはffprobeでファイル構成を確認：コーデック、解像度、time_baseを把握する。  
  - コマンド例: ffprobe -show_streams sample.mp4
- サーバ側での自動処理なら、デマックス→トランスコード→HLS/DASHパッケージ化の流れをスクリプト化する。FFmpeg CLIで済む処理はまずCLIで実験してからライブラリ化すると安全。  
- 日本向け注意点: 字幕（UTF-8/ASS/タイムコード）、放送用の色域/フレームレート（29.97/30/25）や商用配信のライセンス問題を設計時に確認する。  
- 性能最適化: 大量処理はハードウェアエンコーディング（NVENC/QuickSync/VAAPI）や複数プロセスでのワーク分割を検討。  
- 学習フロー: ffmpegコマンドでの実験 → ffprobeで解析 → 上記Cのフローをサンプルで実装して動作原理を体感。

以上を押さえれば、FFmpegを使った動画取り込み・解析・デコードの基礎が短時間で理解でき、次はフィルタ適用やリアルタイム配信の実装に進めます。
