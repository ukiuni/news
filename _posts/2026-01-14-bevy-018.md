# Bevy 0.18 - Bevy 0.18 リリース
Bevy 0.18でゲーム開発体験が一段と洗練：リアルな大気表現、PBR修正、UI/入力/ツールの使い勝手向上

## 要約
Bevy 0.18は大気散乱の柔軟化やPBR修正、レイトレ実験レンダラー「Solari」改善、UIナビゲーションや標準ウィジェット強化など、レンダリング品質と開発者体験（DX）を両方改善するリリースです。Cargoの機能集合やファーストパーティのカメラコントローラなど、実用的な導入ハードルも下がっています。

## この記事を読むべき理由
- 日本のインディーゲームやツール開発者にとって、リアルタイム品質向上（大気表現・PBR修正）と開発効率向上（UI自動ナビゲーション、カメラコントローラ、ビルドの最適化）は即効性のある恩恵だからです。
- ローカライズやツールUI整備が重要な国内市場でも、ゲームパッド対応やフォント機能強化は実務的に使えます。

## 詳細解説
- 大気とライティング（Atmosphere Occlusion / Generalized Atmospheric Scattering）
  - 太陽光が大気を通過する際の色変化（地平線近くで橙〜赤くなる）やボリューメトリックフォグとの連携が自動化され、シーン全体のライティングが一貫してリアルになります。
  - 新しい ScatteringMedium アセットにより、地球風だけでなく火星風、大規模霧、架空惑星など任意の散乱媒体を定義可能。レンダリング最適化も行われており、複雑な大気でも高速性を維持します。

  - 例（ScatteringMedium を追加する断片）:
  ```rust
  fn setup_camera(mut commands: Commands, mut media: ResMut<Assets<ScatteringMedium>>) {
      let medium = media.add(ScatteringMedium::new(
          256, 256,
          vec![ /* ScatteringTerm を複数定義 */ ],
      ));
      commands.spawn((Camera3dBundle::default(), Atmosphere::earthlike(medium)));
  }
  ```

- PBR修正とSolari（リアルタイムレイトレ）
  - 長年問題だった過度なスペキュラ（光沢）や環境マップのFresnel処理の不具合を修正し、物理的により自然な外観に。シーンの説得力が向上します。
  - Solari（実験的レイトレレンダラ）は反射・スペキュラ対応やソフトシャドウ改善、大規模シーンでのパフォーマンス改善などが入り、フォトリアル表現の試作がしやすくなりました。

- FullscreenMaterial（ポストプロセスの簡易化）
  - これまで低レベルでしか作れなかったフルスクリーンエフェクトが、FullscreenMaterialトレイトで簡単に導入可能に。トーンマッピング後やポスト処理のパス順も指定できます。エフェクトの試作→反映が速くなります。

- UI・ウィジェット周り
  - 自動方向ナビゲーション（AutoDirectionalNavigation）で、ゲームパッドや矢印キーによるフォーカス遷移が自動算出されます。動的レイアウトのメニュー設計が楽になります（家庭用ゲーム機向けの操作性改善に直結）。
  - Popover / MenuPopup / RadioButton改善など、標準ウィジェット群が拡充。Bevy Feathers の ColorPlane などツール向けウィジェットも追加。

- 開発者向けツールとDX
  - bevy_camera_controller（FreeCamera, PanCamera）が公式に提供され、シーンの探索やデバッグが容易に。エディタやツール作成時の初期コントローラとして使いやすい。
  - Cargo Feature Collections により 2D/3D/UI 等のシナリオ別機能をまとめて有効化・無効化でき、ビルド時間とバイナリサイズを節約できます。
  - フォント扱いの改善：可変フォントのウェイト、取り消し線・下線、OpenType機能に対応し、リッチなテキスト表現が可能になりました（ただし日本語向けの可変フォントは種類や対応範囲に注意が必要です）。
  - そのほか：スクリーンショット/動画記録の簡易化、glTF拡張対応、アセットのseekable読み取り、複数コンポーネントへの安全な可変アクセスなど、実用的な品質向上が多数。

## 実践ポイント
- まずは公式の Quick Start と 0.17→0.18 Migration Guide を読む。既存プロジェクトの更新は migration ガイドに従って段階的に。
- 開発フラグで bevy_camera_controller を入れて FreeCamera を使い、シーン検査を高速化する。デバッグやツール作りの初期コストが下がります。
- 大気表現を試すなら ScatteringMedium を使ったサンプルを動かし、シーンの時刻や惑星感を変更して比較する。ポストプロセスは FullscreenMaterial で素早く試作可能。
- UIは AutoDirectionalNavigation を有効にして、ゲームパッドや十字キーでの動作を早めに検証する（日本ではパッド操作が重要になる場面が多い）。
- 実運用では Cargo Feature Collections を活用して不要なサブシステムを除外し、コンパイル時間と最終成果物のサイズを抑える。
- フォント機能は、和文フォントでの可変軸サポート範囲を事前に確認して適用する。OpenType 機能を活用して表示品質を改善できる箇所を探す。
