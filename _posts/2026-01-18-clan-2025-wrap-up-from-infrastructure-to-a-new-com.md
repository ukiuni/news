# Clan 2025 Wrap-Up: From Infrastructure to a New Computing Paradigm - Clan 2025 総括：基盤から新たなコンピューティング・パラダイムへ
魅力的タイトル: 「自分のネットとアプリを取り戻す — Clanが描く“個人主権”コンピューティングの未来」

## 要約
Clanは「デジタル主権（digital sovereignty）」を誰でも手にできるようにするためのオープンソース基盤で、2025年に安定リリースを迎え、信頼性の高いネットワーキング抽象化、軽量なマイクロVM、使いやすいGUI、改良されたシークレット管理などで実用的な自己ホスティングを現実に近づけた。

## この記事を読むべき理由
中央集権的サービスや監視・抽出ビジネスモデルへの反動として、企業や個人が自前で安全にサービスを運用するニーズは日本でも高まっています。Clanはその選択肢を「実運用レベル」で提供しようとしており、自宅サーバーや中小企業のインフラ再設計、プライバシー重視アプリの導入を検討する技術者にとって即座に参考になる設計思想と実装の方向性を示しています。

## 詳細解説
- ミッションと設計哲学  
  Clanは単なる製品ではなく「組み合わせ可能な部品群（composable）、再現可能、自己主権」を重視するフレームワーク。ユーザーに暗黙の制約やブラックボックスを強いず、データやサービスの完全なコントロールを取り戻すことが目的です。

- ネットワーキングの抽象化と信頼性向上  
  Clanは特定のVPNやオーバーレイに依存せず、複数のネットワーク技術を共存させる抽象レイヤを導入。ネットワークを「サービス」として宣言的に有効化・優先付けでき、CLIは状況に応じ最適な接続手段を自動選択します。これによりTorやパブリックインターネット、オンデマンドのネットワークサービスを組み合わせ、個々のコンポーネント障害に耐える設計を実現しています。

- マイクロVM（micro VMs）によるアプリの主権化  
  アプリの隔離を強化するために、軽量で起動の速いマイクロVMを採用。ハードウェア仮想化で強い境界を作り、Nixの再現可能ビルドでほぼ瞬時に起動できる点を目指しています。GPUアクセラレーション（virtio-gpu + Wayland）やデスクトップポータル（ユーザー承認経由でファイルやカメラを共有）により、ローカルで安全かつ快適に動くアプリ環境を構築します。結果としてP2P環境でもウェブアプリ的な UX を実現可能にします。

- GUI と運用の見える化  
  NixOS と CLI の強さを活かしつつ、設定や秘密情報、サービスの関係性を視覚化する Clan GUI を開発。ブートストラップ、シークレットの自動生成・回転、マルチマシンのサービス適用など「運用でよくある面倒」を GUI で扱えるようにしています。これにより家族やチーム単位での参加ハードルが下がります。

- シークレット（vars）とインベントリ／サービス抽象  
  旧来の「facts」から宣言的な vars に移行し、シークレット生成・共有・ローテーションを改善。さらにインベントリ層で「マシン」単位から「フリート（群）」単位の構成管理へ移行し、サービスはエクスポート（出力値）を公開して他サービスと自動的に連携できるようになりました。これにより「部品化されたインフラ」が現実化します。

- macOS対応などの互換性拡張  
  （抜粋では詳細は限られるが）ClanはNixOS中心の流れを保ちつつ、将来的な汎用環境化や他OS対応（例：macOSをファーストクラス化）への道を模索しています。

## 実践ポイント
- 試してみる手順（初学者向け）  
  1. まずは公式の安定リリースを使って、ローカル1台＋ラップトップの簡単なClan環境を作る。  
  2. ネットワーク抽象化でTorやMesh VPNを組み合わせ、接続手段が切り替わる様子を体感する。  
  3. Nixの基本（flake含む）を学び、Clanのinventoryで「タグ付け→サービス適用」の流れを試す。  
  4. マイクロVMのサンプルを動かし、GUIでの操作性と分離の効果を確認する。

- 日本での活用シナリオ  
  - 在宅勤務や分散チーム向けのプライベートVPNと共有サービス基盤。  
  - 小規模事業者の自己ホスティング（メール、ファイル、社内ツール）を外部依存を減らして運用。  
  - プライバシー重視のパーソナルクラウド（家族の写真やデータを外部に預けない運用）。

- 注意点と学習コスト  
  Nix/NixOS の知識は依然必要。Clan GUIは補助になるが、設定思想（宣言的インフラ、vars、exports）を理解すると運用が楽になる。段階的に導入して運用手順を確立することを推奨。

- 参考コード例（inventoryのイメージ）
```nix
inventory.instances = {
  # One declaration enables VPN across all machines
  zerotier = {
    roles.controller.machines.server = { };
    roles.peer.tags = [ "all" ]; # All machines join the network
  };
};
```

まとめ：Clanは「自分で管理する」時代のインフラ設計を実用レベルに引き上げようとするプロジェクトで、日本の個人・中小事業者にとっても魅力的な選択肢になり得る。まずは小規模で試して、ネットワーク抽象化とマイクロVMの恩恵を体験してみることを勧める。
