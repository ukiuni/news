# Termux - Termux（Android向けターミナル）
ポケットのLinuxを手に入れる：TermuxでAndroidが最強の開発端末に

## 要約
TermuxはAndroid上で動くターミナルエミュレータ兼軽量なLinux環境で、プラグインやパッケージで機能拡張可能。アプリ本体とパッケージ群は別リポジトリで管理されています。

## この記事を読むべき理由
スマホだけでシェル、開発、サーバ操作、Automationを完結させたい日本のエンジニア／学生／現場担当者にとって、Termuxは移動中や検証環境として強力なツールになるため。

## 詳細解説
- 役割分担：termux-appリポジトリはUIと端末エミュレーションのみ。パッケージ群は termux/termux-packages に別管理。まずはアプリとパッケージの違いを理解すること。  
- 動作環境：公式にフルサポートはAndroid 7以上。Android 5/6は限定対応や過去のリリースに依存。  
- プラグイン群：Termux:API（端末のセンサーやカメラ等連携）、Termux:Boot、Termux:Tasker、Termux:Widget、Termux:Float、Termux:Styling などでモバイル固有の機能を補完。  
- 配布経路と注意点：
  - F‑Droid版（推奨）：安定版が提供され、署名はF‑Droid側で管理。Universal APK サイズは約180MB。
  - GitHub Releases／Actionsビルド：最新を試せるが、ビルドはテスト署名（注意）。アーキテクチャ別APKは約120MB。
  - Google Play（実験的）：ポリシー対応のため機能制限あり。バグや差分が存在するため注意。  
- 署名・sharedUserId制約：Termuxと各プラグインは同一 sharedUserId を使うため、異なる配布元（署名が違うAPK）を混在させるとインストール/更新エラーになる。乗り換え時は既存のTermux系APKを全てアンインストールする必要がある。  
- Android 12+ の問題点：OS側がプロセスを（「phantom」や高CPU使用で）強制終了する報告多数。設定や将来のAndroid 12L/13でのオプションにより回避可能になる見込みだが、アップグレードは自己責任で。不要な終了を避けるためバッテリー最適化の無効化を推奨（dontkillmyapp の指針参照）。  
- セキュリティ警告：GitHubのテスト署名鍵は公開されているため、サードパーティ配布のAPKは改竄リスクあり。信頼できる配布元（公式GitHub、F‑Droid）から入手すること。

## 実践ポイント
- まずはF‑Droid版を入手し、v0.118.0以上にアップデートする。  
- 初回はパッケージ情報を更新：
```bash
# bash
pkg update && pkg upgrade
```
- バッテリー最適化をオフにして「端末の勝手な停止」を防ぐ（設定／dontkillmyapp参照）。  
- プラグイン（Termux:API / Tasker / Widget）を使って端末のセンサー連携や自動化を始める。  
- 別配布元に切り替える前は必ずTermuxと全プラグインをアンインストール、バックアップを取る。  
- 不審なGitHubビルドやSNS配布のAPKは避ける。セキュリティ・署名情報を確認すること。

以上を押さえれば、スマホを“ポータブルなLinux開発機”に変える第一歩が踏み出せます。
