# Microsoft uses plagiarized AI slop flowchart to explain how Github works, removes it after original creator Vincent Driessen calls it out: "The AI rip-off was not just ugly. It was careless, blatantly amateuristic, and lacking any ambition, to put it gently. Microsoft unworthy." - Microsoftが盗用まがいのAI生成フローチャートを公式に掲載、作者指摘で削除：『雑で粗末』と非難

魅力的な見出し：Microsoftの公式ドキュメントが「AIで洗い流した」図を公開→誤訳と盗用疑惑で炎上、学べるガバナンスの教訓

## 要約
MicrosoftのLearnページに、Vincent Driessenの有名なGitブランチ図を明らかに元にしたAI生成画像が掲載され、文字や矢印が崩れた「continvuocly morged」などの誤表記が見つかり、作者の指摘で差し替えられました。問題は品質だけでなく、出典や検証プロセスの欠如です。

## この記事を読むべき理由
- Git/GitHubを使うエンジニアやドキュメント作成者にとって、誤った図は学習ミスを招くリスクがあります。  
- 日本の企業でもAI画像・ドキュメント生成が広がる中、著作権・品質管理・透明性の教訓になります。

## 詳細解説
- 背景：2010年に公開されたDriessenの「A successful Git branching model」は長年参照される定番図で、作者はソースを公開してきた。  
- 何が起きたか：Microsoftはその図をオリジナルのソースファイルで再利用せず、AI画像生成ツールで類似図を再作成してLearnに掲載。結果、矢印や色遣いが崩れ、軸ラベルが「Tim」になったり単語が「featue」「morged」のように壊れるなどの明白な劣化が発生。  
- 批判点：Driessenは「単に劣化させて公開しただけ」であり、出典明示や人による検証を欠いた点を強く非難。Microsoftは画像を差し替えたが、修正履歴や理由の明示はなし。  
- 技術的含意：AI画像生成は元素材の特徴を模倣しやすく、出典を消し去る「洗い替え」になり得る。これにより著作権侵害、誤情報拡散、信頼低下が起きる可能性がある。特に技術ドキュメントでは誤った図は誤解を生むため危険度が高い。

## 実践ポイント
- ドキュメント担当者へ：AI生成素材を使う場合は必ず原典確認・出典明示・人による校正を行う。図は元ファイル（SVG/Keynote等）を保存して差分管理する。  
- エンジニアへ：公式ドキュメントの図も鵜呑みにせず、原典や実装で検証する習慣をつける。  
- 法務/管理者へ：AI利用ポリシーを整備し、公開物の検証フロー（著作権チェック、レビュー、変更履歴の公開）を組み込む。  
- 発見時の対応：作者への連絡、証拠の保存（Wayback等でアーカイブ）、公開ページへの修正要求を行う。

短く言えば、AIは便利だが「自動生成＝信用」ではない。特に教育・技術資料では人の目と出典管理が必須です。
