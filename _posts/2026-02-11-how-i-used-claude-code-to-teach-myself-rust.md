# How I Used Claude Code to Teach Myself Rust - Claude Codeで自分でRustを学んだ方法
「AIに頼らず学ぶ」──Claudeを“先生化”して短期間でRustを習得した実践レポート

## 要約
著者はClaude（Claude Code）を「コードを書かせるツール」ではなく「学習を促すコーチ」に変えるプラグインを作り、実プロジェクトを通じて短期間でRustの基礎と実践力を身につけた。

## この記事を読むべき理由
AIが普及する今、単に自動生成に頼るのではなく「人間の学びを加速するAI活用法」は日本のエンジニア教育や企業研修にも直結する実践的手法だから。AIと共に学ぶ具体的ワークフローが参考になる。

## 詳細解説
- 目的とツール
  - 目的：AIに依存せず自分の思考を鍛えつつRustを学ぶ。
  - 使用ツール：Claude Code、Visual Studio Code、git。
- プラグイン設計（simian programmer plugin／spp）
  - Drive mode：Claudeに「質問のみ可、コードは書かない」制約を与え、学習の主体を人間にする。
  - Coaching：ファイルウォッチャーで自分の編集履歴と会話をログ化し、/spp:coachで過去履歴を読み取ってフィードバックを生成。
  - 週次コミット目標：人間コミット比率を設定（例：25％）し、割合が下回るとAIにコード生成を禁止。
  - Pause機能：学習中の一時停止（24時間）など運用上の配慮。
- 学習プロジェクト（ClawProxy）
  - 目的：OpenClawのような自律エージェントがAPIキーなど機密を読み取れないよう、OSレベルのサンドボックス＋プロキシでトークン注入する仕組みをRustで実装。
  - 設計→タスク分割：TDD的な設計書作成と、コミット単位のタスク分割（3/4をAI、1/4を人間に割当）で学習用タスクを明確化。
  - 実行：Claudeがボイラープレートを生成、難所や学習に有益な部分はDrive modeで自分が実装。実装後にClaudeへレビュー依頼、最後にCoach機能で振り返り。
- 学習効果
  - 約1週間・数時間の作業でRustの基本構文、メモリ管理の概念、エラー処理モデルの理解を獲得。プロジェクトは動作。ただし外部プロジェクト側の連携設定で未解決の課題あり。
- 発見と洞察
  - 手で書くことで「設計的視点」や「改善点の発見」が生まれる。AIは強力だが、人間の視点は異なる価値を提供する。
  - 学習にはモチベーション維持できる実プロジェクトが有効。

## 実践ポイント
- Drive modeを作る／使う：AIに「書かせる」のを制限して質問と解説に徹してもらう。
- タスクをコミット単位に分割：学習用タスクを明確にして進捗を可視化する（.mdで管理）。
- 人間コミット割合を設定：週次で自分の手を動かす強制力を作る（例：25％）。
- IDEのAI補完をオフにする：学習セッション中は自動補完を切ることで自己解決力を鍛える。
- 実用プロジェクトを選ぶ：興味が持てる小さな実装目標を設定すると継続しやすい。
- 質問を深掘りする：構文だけでなく設計やメモリ・エラーハンドリングの理屈をAIに問い、理解を定着させる。
- 定期的にレビューと振り返りを行う：編集ログ＋会話履歴を使ってフィードバックを得る。

日本の現場ではセキュリティ意識の高まりやRust採用の増加が見られるため、本手法は社内研修や個人のスキルアップにすぐ応用できる。
