# Plasma Effect - プラズマエフェクト
画面が「生きている」ように見える！クラシックなプラズマ効果を今すぐ試したくなる入門ガイド

## 要約
古典的なプラズマ効果は、複数の正弦・余弦波を重ねて干渉を作り、色勾配にマップするだけで有機的な動きを生むシンプルで強力な手法である。

## この記事を読むべき理由
ゲーム背景、ライブビジュアル、Webサイトのアニメーションなどに低コストで魅力的な動的表現を追加できる。日本の小〜中規模プロジェクトや教育用途にも最適で、GPUでリアルタイム実行可能なため実装の敷居が低い。

## 詳細解説
- 基本原理：画面座標を入力にして複数の波を作り、合成して色に変換する。代表的な式は  
  $value = \sin(x + t) + \cos(y + t) + \sin(r + t)$  
  （$r$ は画面座標からの距離、$t$ は時間）で、これが干渉パターンの元になる。
- 干渉（Wave Interference）：複数波が重なることで強め合う領域（構成的干渉）と打ち消し合う領域（破壊的干渉）が生じ、複雑で滑らかな模様ができる。
- 色付け（Color Gradients）：波の出力値を色空間にマップする際、コサインパレット法がよく使われる。例えば  
  $col = 0.5 + 0.5\cos(s + (0.2,0.5,0.9))$  
  のようにRGBチャンネルに位相差を与えると滑らかなグラデーションが得られる。
- スペキュラ（Specular）強調：色場の勾配を微分（GLSLの $dFdx, dFdy$）で求め、それを法線の近似としてライティング計算に使うと「光沢感」が出て深みが増す。
- 実装上の注意：古い機材ではルックアップテーブルで高速化していたが、現代のGPUならフラグメントシェーダで直接計算しても高速。座標正規化（アスペクト比補正）や時間スケーリングで表現が大きく変わる。

## 実践ポイント
- まずはShaderToyやWebGLのフラグメントシェーダで試すと理解が早い。短いGLSLスニペット例：
```glsl
// GLSL
void mainImage(out vec4 col, in vec2 uv){
  float t = iTime;
  vec2 p = (uv - 0.5) * vec2(iResolution.x/iResolution.y,1.0) * 4.0 + t*0.3;
  float d = length(p);
  float s = 7.0 * cos(d + sin(t)) * sin(0.1 + cos(t));
  col = vec4(0.5 + 0.5*cos(s + vec3(0.2,0.5,0.9)), 1.0);
}
```
- パラメータで遊ぶ：周波数・位相・時間スケール・スケール係数を少し変えるだけで表情が大きく変わる。
- カラーパレット：位相差を変えたコサインパレットやHSVマッピングで雰囲気を調整。
- 高品質化：$dFdx/dFdy$でスペキュラを追加、グラデーションをトーンマップしてコントラスト調整。
- 応用例：ゲームのメニュー背景、映像演出（VJ）、電子工作ディスプレイ、Shader教育用デモ。
