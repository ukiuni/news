# Memory Allocation Strategies - メモリ割り当て戦略
魅力をそそるタイトル: ゲームも組み込みも救う！「速く・ムダなく」メモリ割り当て設計入門

## 要約
メモリ割り当てには速度、断片化、オーバーヘッド、リアルタイム性といったトレードオフがあり、用途に応じたアロケータ（線形/アリーナ、スタック、プール、フリーリスト、バディなど）を使い分けるのが肝心、というシリーズの要点。

## この記事を読むべき理由
日本のゲーム開発、組み込み／IoT、スマホアプリ、クラウドサービスはいずれも「メモリ効率と速度」が直接UXやコストに影響します。正しい割り当て戦略を知ればパフォーマンス改善やバグ削減に直結します。

## 詳細解説
- 問題意識：一般的な malloc/free は汎用的だが、断片化や遅延が発生しやすく、ホットパスではボトルネックになり得る。用途に合わせた専用戦略が有効。
- 線形（Arena）アロケータ：大量の短命オブジェクトをまとめて割り当て、まとめて解放する。実装がシンプルで高速。ライフタイムがまとまっているケース（シーン単位のリソースなど）に最適。
- スタックアロケータ：LIFO のライフサイクルが保証されるときに超高速。関数スコープや一時領域用。
- プール（オブジェクトプール）：固定サイズオブジェクトを再利用する。小さな頻繁な割り当て／解放に強い（ゲームのエンティティ等）。
- フリーリスト：可変サイズに対応するが断片化とメタデータ管理が課題。高速化のために境界タグやサイズ分割を工夫する。
- バディアロケータ：2の累乗サイズで管理し、併合が容易。断片化制御と高速割り当てのバランスを取る場面で有用。
- 共通ポイント：アライメント、メタデータオーバーヘッド、スレッド競合（ロック）対策、キャッシュ効果を常に考慮する。必要ならスレッドローカルアロケータやロックフリー構造を用いる。
- 実装上のテクニカル：境界タグ、ビットマップ、フリーリストの分離（size-class）、ヒープ成長戦略、ページサイズとの関係などが性能に影響する。

## 実践ポイント
- ライフタイムで分ける：同じ寿命のオブジェクトはアリーナでまとめて管理する。  
- 小物はプール化：頻繁に作る小サイズはプールで再利用。new/mallocを避ける。  
- ホットパスを計測：プロファイラ（perf、Instruments、Android Systrace）で割り当てホットスポットを特定。  
- マルチスレッド対策：スレッドローカルアロケータや軽量ロックを導入。  
- ライブラリ活用：jemalloc/mimalloc/tcmalloc や C++ std::pmr、Rust のカスタムアロケータを検討。  
- テストとツール：ASan/Valgrind、ヒーププロファイラでリーク・断片化を検出。低メモリ環境での挙動も確認。  
- 日本向け優先ケース：モバイル（省電力＝少ないページフォルト）、車載/組み込み（決定論的な応答）、ゲーム（フレーム安定化）には特に重要。

短期的には「どのアロケータを使うか」を決めるより先に、プロファイル→ボトルネック特定→部分最適化の順で手を動かすと効果が出ます。
