# A simple web we own - 私たちの手に取り戻すシンプルなウェブ
個人とコミュニティで作る「シンプルなWeb」──Big Techに頼らない、書くことに特化したインターネット再設計案

## 要約
現在のWebは大企業と国家の支配下にあり、利用者は「居住者」であり「商品」になっている。Markdown＋簡素なツールチェーン＋個人所有のホストで、誰でも制御可能なシンプルなWebを取り戻せる、という主張。

## この記事を読むべき理由
日本でもプライバシーやプラットフォーム依存の問題は深刻です。個人やローカルな共同体でコンテンツ発信・運用できれば、監視経済やサービス依存から部分的に脱却できます。低価格ハードやMarkdownの普及で現実的な選択肢になっています。

## 詳細解説
- 問題点の整理：現状は中央集権的サービス（Big Tech）への収斂で、監視・収益化モデルが主流。複雑なCMSや大規模クラウドに頼る設計が常態化している。  
- シンプル路線の要素：  
  1. 表現はMarkdownのような軽量ハイパーテキスト（書くことに集中）  
  2. Markdown → HTML変換はPandocなど既存ツールで自動化  
  3. サイト構造管理（RSS、sitemap.xmlなど）の自動生成を含むツールチェーン  
  4. ホスティングは個人所有のデバイス（例：Raspberry Pi）や小規模協同組合で運用  
- なぜ静的サイトが有利か：生成が簡単・ホスティング安価・攻撃面が小さい。WordPress等のフル機能CMSは便利だが、運用・依存コストと複雑さが高い。  
- ネットワーク面：インターネットはプロトコルの集合体（IPv4/IPv6）。ローカルネットワーク（localhost／家庭内LAN）や個人が管理する小さなネットワークを起点に公開・連携できる。  
- 社会的インパクト：労働組合化が政治経済に影響を与えたように、個人・協同運営のインフラが一定割合を占めればWebの健全性が改善する可能性がある。

## 実践ポイント
- 今すぐ：書く素材はMarkdownで揃える（エディタは既存OSのテキストエディタでOK）。  
- ツール選び：PandocでMarkdown→HTMLを試す。RSSやsitemap生成機能を持つ軽量ジェネレータを検討（Jekyll/Hugoは機能豊富だが学習コストあり）。  
- ローカルホストで検証：Raspberry Piや家庭用NASでローカルサイトを立て、まずは公開せずに運用を学ぶ。  
- 公開方法：自宅ネットワーク＋プロバイダ、または低コストなドメインと静的ホスティング（CDN）を組み合わせる。動的公開が難しければ、Git-basedデプロイやトンネリングサービスで代替可。  
- 協同運用：個人運用が難しい場合は地域や職域の協同サーバを作り、運用知識とコストを共有する。

短くまとめると、必要なのは「個人が所有する小さなハード」「Markdownの単純さ」「生成と配信を自動化する軽量ソフト」。これで誰でも『自分たちのWeb』を現実的に持てます。
