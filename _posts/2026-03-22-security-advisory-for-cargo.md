# Security advisory for Cargo - Cargoに関するセキュリティ勧告
クリックせずにはいられないタイトル案: 「あなたのCargoが知らぬうちにフォルダ権限を書き換えられる可能性がありました — 今すぐ確認すべきこと」

## 要約
Cargoが依存して使う第三者クレート tar の脆弱性（CVE-2026-33056）により、悪意あるクレートがビルド時にファイル展開で任意のディレクトリ権限を変更できる可能性がありました。crates.io側では既に対策と一括監査が行われ、Rust 1.94.1で修正が配布されますが、代替レジストリを使う古いCargoは注意が必要です。

## この記事を読むべき理由
- 初心者でも使うCargo/クレート周りの安全性は開発者全員のリスク管理の基本です。  
- 日本の企業やOSSプロジェクトも、社内レジストリやプロキシを使っている場合は特に影響を受ける可能性があります。

## 詳細解説
- 問題点: tar クレートの処理により、展開時にファイル/ディレクトリのパーミッションを意図せず（あるいは意図的に）変えられる挙動が存在。悪意あるクレートを公開・配布すると、ビルド時に攻撃者が権限設定を変えることが可能になる。  
- 対応の流れ:
  - crates.io運営は3月13日に悪用防止のアップロード制限を導入し、過去に公開された全クレートを監査。公開レジストリ上のクレートに悪用は確認されていないと報告。  
  - RustチームはRust 1.94.1（3月26日予定）で tar の修正を含むアップデートを提供。Cargo自体もパッチ済み。  
  - ただし、代替レジストリ（社内レジストリやミラー）を使う環境では、レジストリ側の対応状況によっては依然リスクが残る。ベンダー確認が必要。  
- 報告と関係者: 脆弱性発見者と複数のメンテナが協力して対応を実施。

## 実践ポイント
- 今すぐやること:
  1. Rustツールチェーンを最新（少なくともRust 1.94.1以降）に更新する。  
  2. crates.io以外のレジストリを使っている場合は、レジストリ提供者に今回の脆弱性への対応状況を確認する。  
  3. CI/CDやビルド環境で使うCargoバージョンを一覧化し、古いバージョンが残っていないか確認する。  
  4. 社内レジストリがあるなら、公開済みクレートの監査（tarを含むパッケージ展開処理のチェック）を行う。  
- 予防策:
  - ビルド環境を最小権限で実行し、展開先ディレクトリの権限を厳格に管理する。  
  - 依存性のサプライチェーンリスクに備え、定期的な依存関係監査と自動アラートを導入する。
