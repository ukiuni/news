# Linus is vibe coding - ライナスは「バイブ」コーディング中
ライナス流「おもちゃのエフェクト」で学ぶ、超低レイテンシなデジタル音声処理入門

## 要約
Linus Torvalds が公開した「AudioNoise」は、IIR フィルタやディレイループなどを使った“遊び”のデジタル音声エフェクト集で、学習目的で書かれたシンプルな C と Python のサンプル群です。高度な FFT や機械学習は使わず、サンプルごとの処理で「遅延ゼロ」に近い挙動を目指しています。

## この記事を読むべき理由
日本でもギター／オーディオのDIYや低レイテンシ音処理への関心は高く、組込みやリアルタイム音声処理を学びたいエンジニアや趣味者にとって、実践的でコードが読みやすい入門教材になります。GPL-2.0 で公開されているため、学習・改造して試せる点も魅力です。

## 詳細解説
- リポジトリ構成  
  C ソースとヘッダ（biquad.h, flanger.h, phaser.h, echo.h など）と、ビルド用の Makefile、そして音声可視化用の Python スクリプト（visualize.py）が主要ファイルです。サンプル音源（MP3）も同梱。

- 目指す設計思想  
  Linus は「学び」が目的と明言しており、回路を忠実に模した高度なエミュレーションではなく、RC ネットワークの効果をデジタルのオールパスフィルタなど簡潔な手法で表現しています。ポイントは「single sample in, single sample out（1サンプル毎処理）」で、バッファリングや大きなバッチ処理を避け、サブミリ秒級の遅延を維持することです。

- 技術的な中身（初心者向け）  
  - IIR（無限インパルス応答）フィルタ：過去の出力を使って現在の出力を作るフィルタ。係数を変えるとローパス／ハイパス／オールパスの振る舞いを作れます。  
  - ディレイ（遅延）ループ：サンプルをリングバッファに入れて後で参照する。エコーやフランジャーの基礎。  
  - オールパスフィルタ：位相を変化させることでフェイザーなどの効果を得る。振幅特性をほぼ保ったまま位相だけ操作する。  
  - 低レイテンシ設計：ADC→処理→DAC の経路での遅延を最小化する実装思想。リアルタイム楽器・エフェクタに近い感覚です。  
  - 可視化は Python で「vibe-coding（ノリノリで書いた）」と本人が冗談めかしている通り、学習しながら作ったスクリプト。

- 何が「高度でない」のか  
  FFT ベースのボコーダーや機械学習で“キャビネットをエミュレートする”ような手法は使っていません。狙いは「まず仕組みを理解する」ことです。

## 実践ポイント
- まずやること（5分で開始）  
  1. リポジトリを git clone（https://github.com/torvalds/AudioNoise）。  
  2. README と Makefile を読む。C と Python の依存関係を確認。visualize.py を試して音を観察。  

- 触って学ぶ（30–60分）  
  - biquad.h や flanger.h の係数を変えて音の差を試す。サンプル毎処理の挙動をログ化して理解する。  
  - echo（遅延）ループの長さやフィードバック量を変えてエコーの挙動を体感する。  

- 発展（週単位のプロジェクト案）  
  - 低レイテンシの USB オーディオを用意して、実際のギター入力を通して試す（日本の宅録・ギター仲間にもウケます）。  
  - 組込み機器（小型マイコンやFPGA）での実装を目標に、サンプルレート・バッファ管理を最適化する練習をする。  
  - 可視化ツールを改良して学習ログを作る（Python の matplotlib や sounddevice の導入など）。

- 学習リソース（日本語）  
  - 「DSP 入門」系の書籍やオンライン講座で IIR/フィルタの基礎を押さえると理解が早い。  
  - 国内のギター・エフェクタ DIY コミュニティや電子工作フォーラムで実機試作のノウハウを得る。

最後に一言：このリポジトリは「上手に作るプロジェクト」ではなく「仕組みを遊びながら学ぶ教材」です。まずは動かして、改造して、自分の耳で確かめてください。
