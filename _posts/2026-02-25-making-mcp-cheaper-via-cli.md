# Making MCP cheaper via CLI - CLIでMCPを安くする
1コマンドでMCPコストが94%減る？CLI流「必要なときに読む」設計の衝撃

## 要約
MCPは全ツールのJSON Schemaを会話に投げるためトークン費用が高い。CLI的な「軽い目録＋必要時に詳細取得」へ切り替えるだけで、実測で約94%のトークン削減が可能だという報告。

## この記事を読むべき理由
日本でもLLMエージェントの運用コストは無視できない問題です。クラウド利用料やAPIトークン課金を抑える現実的な手法として、すぐ試せる設計指針を得られます。

## 詳細解説
- 問題点：MCPはセッション開始時に全ツール・全パラメータのJSON Schemaを会話に注入する（例：84ツールでセッション開始約15,540トークン）。これが「説明マニュアル税」となり、トークンコストを膨らませる。
- CLI方式の違い：最初はツール名と場所だけの軽い目録を読み込み、実行時に詳細（--help やコマンドヘルプ）を問い合わせる「遅延読み込み」を採用。セッション開始は数百トークン、実行時にのみ詳細を引くため合計トークンが大幅に減る。
- 実測例（典型構成：6サーバ・14ツール＝合計84ツール）
  - セッション開始：MCP ≈ 15,540トークン → CLI ≈ 300トークン（98%削減）
  - 1ツール呼び出し込み合計：MCP ≈ 15,570 → CLI ≈ 910（約94%削減）
  - 全体でCLIは約94%少ないトークン消費を示す。
- 他の手法との比較：AnthropicのTool Searchは「インデックス＋必要時フェッチ」で85%程度削減するが、フェッチ時にまだJSON Schemaを丸ごと取るためCLI方式より高コスト。加えてAnthropic限定の手法である点も留意。
- 実装支援：記事作者はMCPサーバからCLIを自動生成するツール群（CLIHubなど）を公開し、1コマンドで変換できるフローを示している。

## 実践ポイント
- まず既存エージェントのセッション開始とツール呼び出しでのトークン使用量を計測する。
- 「目録だけ先に渡す（軽量）」→「実行時に --help やAPIで詳細取得」のワークフローに変更してみる。
- MCPをそのまま使う場合は、不要なSchemaの注入を削減する設定を探す（遅延ロードやフィルタリング）。
- CLI生成ツール（CLIHubなど）やOpenclawのavailable_skills形式を参考に、自動変換を試すと導入コストが下がる。
- モデル・プロバイダ別の制約（例：Anthropic専用機能）を確認し、ベンダーロックインを避けつつコスト効果を評価する。

読んで試すだけで運用コストに直結する改善が見込めます。まずはトークン計測から。
