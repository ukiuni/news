# What Is My IP Address? Check IPv4/IPv6, Location & Proxy | My IP Details - 自分のIPアドレスは？ IPv4/IPv6・位置情報・プロキシを確認 | My IP Details
自分の「公開IP」が今どこを指しているか、簡単に確認してみませんか？ — 意外と知らない仕組みと、すぐ使えるチェック術を初心者向けに分かりやすく解説します。

## 要約
このツールは、あなたの「公開IPアドレス」（IPv4/IPv6）を表示し、推定される位置情報やプロキシ/VPNの有無などを確認できるオンライン診断サービスです。IPの取得方法や限界、セキュリティ上の注意点を理解すると実務や個人利用で役立ちます。

## この記事を読むべき理由
- ネットワークやクラウド運用をする人は、接続元のIPを把握することでアクセス制御やログ解析が正確になります。
- 日本国内のモバイルや家庭回線はCGNATやIPv6の浸透で見え方が変わるため、原因切り分けに必須の知識です。
- 個人ユーザーでも、VPNやプロキシが本当に効いているか、位置情報がどの程度正確かを簡単に確認できます。

## 詳細解説
1. 公開IP（グローバルIP）とプライベートIPの違い  
   - 自宅や会社のルータ内部で使われるプライベートIP（例: 192.168.x.x）は外部から見えません。インターネット上で「あなた」を識別するのが公開IPです。
   - 公開IPはISPがルータに割り当てるもので、ルータ、プロキシ、またはVPN経由で変わることがあります。

2. IPv4とIPv6の基本  
   - IPv4は32bit（例: 203.0.113.5）、アドレス枯渇のためIPv6（128bit、例: 2001:db8::1）が導入されています。日本の主要ISPや携帯事業者はIPv6を徐々に導入済みで、特に固定回線での普及が進んでいます。

3. ロケーション推定の仕組みと精度  
   - 多くのオンラインIP表示サービスは、IP→地理情報データベース（GeoIP: MaxMind等）やISPの登録情報から「国/市区町村レベル」を推定します。精度はまちまちで、携帯回線やプロキシ経由だと誤差が大きくなります。
   - 精密な位置情報が必要なら、ブラウザのジオロケーションAPI（ユーザー許可が必要）やアプリ側の位置情報取得が必須です。

4. プロキシ・VPN・CGNATの検出  
   - サービスは接続ヘッダ（X-Forwarded-For等）、既知のVPN/プロキシIPリスト、挙動（IPの逆引き）でプロキシの疑いを示します。完璧に検出できるわけではありません。
   - 日本のモバイル回線ではキャリアグレードNAT（CGNAT）を使っていることがあり、利用者ごとの公開IPが共有されているため「個人を特定しにくい」反面、外部からのポート公開などが制限されます。

5. プライバシーとセキュリティの注意点  
   - 公開IPはサービスへのアクセス制御やログで重要な手がかりになります。不要な公開IP漏洩（公開ポートやパブリックサービスの設定ミス）に注意してください。
   - VPNやトラフィック中継を使えばIPを隠せますが、逆にログや利用規約違反のリスクも生じます。

6. どの情報源でIPを判定しているか（技術的裏側）  
   - Webサービスは接続元のTCP/IPソケット情報、HTTPヘッダ、DNS逆引き、GeoIPデータベース、WHOIS情報などを組み合わせて結果を出します。仕組みを知ると結果の信頼度を評価できます。

## 実践ポイント
- すぐ自分の公開IPを確認する簡単コマンド（端末で実行）:
```bash
# 外部に見えているIPを取得（例）
curl ifconfig.me
curl -s https://ipinfo.io/ip
```
- ローカルIPを確認するコマンド:
```bash
# Windows
ipconfig

# macOS / Linux
ip addr show
```
- 接続元がプロキシやVPNか疑わしい場合は、複数のIPチェックサービスで確認し、逆引き（dig -x <IP>）やWHOISで登録情報を調べる。
- 日本での実務では、IPv6運用とCGNATの影響を考慮してアクセス制御（IPホワイトリスト）を設計する。必要ならISPに静的IPやグローバルIP割当の相談を。
- プライバシー重視なら、使用中のサービスがどの程度IPログを保存するかを確認し、VPN利用や不要な公開ポートの閉鎖を実施する。
