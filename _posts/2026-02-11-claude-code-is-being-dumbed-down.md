# Claude Code Is Being Dumbed Down - Claude Codeが“簡略化”されて見えなくなった
コードの内部を隠すアップデートで、$200/月の高額ツールが「何をやったか分からない」状態に――開発者が求めるのはトグル一つだけだった

## 要約
AnthropicのClaude Code v2.1.20で、ファイル読み取りや検索の詳細（ファイルパスや検索パターン）が「Read 3 files」「Searched for 1 pattern」のような一行サマリに置き換えられた。ユーザーは元に戻すか、最低でもオン／オフの切替を求めているが、運営は「verboseモードを使え」と回答して議論がこじれている。

## この記事を読むべき理由
- 日本の開発チームや企業でも有料AIコーディングツールの採用が増加中で、ツールがコードベースで何を行ったかを把握できないとトレーサビリティや品質保証に直結するため。  
- セキュリティや監査、チーム運用の観点で重要な議論。

## 詳細解説
- 変更点：v2.1.20でファイル読み取りや検索の「何を」したか（パス／パターン）を非表示にし、数だけ表示する簡潔化を導入。  
- ユーザーの要求：具体的なファイルパスや検索パターンをインラインで表示する、あるいは簡単な設定トグルで元に戻すこと。  
- Anthropicの応答：多数にとってはノイズ削減で好評、問題があるならverboseモードを使え、という立場。  
- 問題点：verboseモードは思考トレースやフック出力、サブエージェントの大量出力を全て吐き出し、欲しい「ファイル一覧だけ」を得るには過剰かつ扱いにくい。運営はverboseを段階的に“非verbose化”して対応しているが、結局複雑化して本質的解決になっていない。  
- 現状の実務影響：多くのユーザーが旧バージョン（例：2.1.19）に固定して運用、あるいは他ツール検討。企業利用では監査ログ・説明責任の観点で懸念が強い。

## 実践ポイント
- 当面の回避策：重要プロジェクトは安定版（変更前のバージョン）にピン留めする。  
- 交渉ポイント：組織で使う場合は「ファイルパス／検索パターンの表示トグル」を要求し、SLAや契約に反映させる。  
- ログ戦略：verboseをそのままターミナルで見るのは非推奨。出力をファイルに落としてフィルタリング（grep・jq等）で必要情報だけ抽出するワークフローを作る。  
- 代替検討：透明性が必須なら他のツールやオンプレミス型のAIアシスタントを評価する。  
- フィードバックの出し方：GitHub issueやサポートに「単純なbooleanトグルで復帰」を要望し、具体例（どの出力が必要か）を提示すると効果的。

短くまとめると、問題は「簡潔化＝透明性の損失」。高額な開発支援ツールを安全かつ説明可能に使うには、表示トグルやログの設計が不可欠です。
