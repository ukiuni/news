# FFmpeg 8.1 - FFmpeg 8.1「Hoare」
GPUで動画処理がもっと速く、柔軟になる――FFmpeg 8.1で開く映像ワークフローの次の一手

## 要約
FFmpeg 8.1「Hoare」はVulkanベースの計算（compute）コーデック強化、D3D12サポート拡充、xHE‑AACネイティブなど多数のデコーダ／エンコーダ追加と内部改善を含むマイナーアップデートです。起動速度改善や将来の大改修（swscale書き換え）の下地も整いました。

## この記事を読むべき理由
日本の配信・放送・映像制作現場やゲーム／組込み向けの映像処理はGPUやハードウェアアクセラレーション活用が鍵です。今回のリリースは「プラットフォーム横断でGPUを活かす」取り組みを一段進め、現場のワークフロー効率化や将来的なフォーマット対応（VVC/AV1等）に直結します。

## 詳細解説
- Vulkan computeベースのコーデック
  - FFv1やProRes系などをVulkanのコンピュートシェーダで実装。プラットフォーム依存の専用ハードではなく、Vulkan 1.3対応GPUで高速処理が可能。
  - GLSLの実行時コンパイル依存を減らし初期化を高速化。エンコーダは専用識別子（例: ffv1_vulkan）で利用。
- D3D12強化
  - D3D12上でのH.264/AV1エンコードやスケール・デインタレースなどのフィルタ追加（scale_d3d12等）。Windows環境でのGPUパイプライン活用が向上。
- 新規/改善デコーダ・フォーマット
  - xHE‑AAC（ネイティブ実装、現状一部機能は順次対応）、MPEG‑H via libmpeghdec、EXIFメタ解析などを追加。
  - LCEVCメタデータ対応、IAMF（Immersive Audio）でのAmbisonic要素サポート。
  - Rockchip向けH.264/HEVCハードエンコードサポート（組み込みボードでの実用性向上）。
- フィルタ・内部改善
  - drawvgやvpp_amfなど新フィルタ。swscale書き換え準備など大規模内部改良と多くのバグ修正。
- エコシステム面
  - Vulkan実装に関する技術記事がKhronosブログに掲載。ディストリビュータ／統合者へはアップグレード推奨（git master使用者を除く）。

## 実践ポイント
- バージョン確認と対応確認:
```bash
ffmpeg -version
ffmpeg -hwaccels
ffmpeg -encoders | grep vulkan
```
- 新しいVulkanエンコーダの試用例:
```bash
ffmpeg -i input.mp4 -c:v ffv1_vulkan output.mkv
```
- 導入方針
  - まずはテスト環境で既存ワークフローの互換性と性能を評価。特にドライバ／GPUごとに挙動が異なるためベンチを推奨。
  - ディストリビュータ（Linuxディストロ、OBS等）がバイナリを出すまで待つ選択肢も現実的。組み込み機（Rockchip等）はファーム連携で恩恵が大きい。
- 参考：リリースノートとKhronosの解説を確認して実装・最適化方針を決定すること。

以上。アップグレード前は必ず自環境での互換性と性能を検証してください。
