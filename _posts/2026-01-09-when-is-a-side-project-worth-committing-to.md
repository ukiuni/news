# When is a side project worth committing to? - サイドプロジェクトはいつ本気で取り組む価値があるのか?
クリックせずにはいられないタイトル: 「寝られる副業を作れ — ‘使えて・放置できる’ サイドプロジェクトの見極め方」

## 要約
サイドプロジェクトは「すぐ役立つ」「メンテが少ない」「運用コストが低い」ことが揃うとコミットする価値がある。著者はゴルフのランチモニター比較サイト（Choosy Golf）でこの基準を実践した。

## この記事を読むべき理由
時間と気力が限られる日本のエンジニアにとって、「作って終わり」にならないプロジェクト選びは重要。仕事や家庭と両立しつつ成果を出すための具体的なフィルターと実例が学べます。

## 詳細解説
- 問題意識
  - 多くの開発者は「技術的に面白い」けれど「運用が重い」プロジェクトに失敗しがち。疲弊して放置されるリポジトリが増える要因です。
- 著者の新基準（フィルター）
  1. すぐに有用であること（初期から誰かの問題を解決する）  
  2. ランニングコストが低いこと（静的ホスティングや無料プランで賄える）  
  3. 維持管理の負担が少ないこと（サーバ監視や定常的なアップデートをほぼ不要にする）  
- 実例：Choosy Golf
  - ゴルフのランチモニターは価格帯・方式（レーダー／カメラ）・サブスク有無とバラつきが大きく、比較が煩雑。
  - 著者は「対象を限定して徹底比較する」ことで即効性のある比較サイトを短時間で作成。
  - 技術面は React + Vite の静的サイト構成で、AIアシスト（プロンプトで初期スキャフォールド生成）＋人間の検証で情報の正確性を担保。
- AIの使い方と限界
  - AIは情報収集や雛形作成を高速化するが、誤情報（hallucination）を人間が検証する必要あり。
- メンテナンス対効用比
  - 「有用性／維持コスト」の比が高ければコミット価値あり。逆に維持が重いならミニマムで公開して終わる選択も合理的。

## 実践ポイント
- 着手前チェックリスト
  1. 問題の具体性：誰のどの困りごとを即解決するかを1行で書く。  
  2. スコープを有限に：対象（製品やデータ種類）は限定する。網羅はやめる。  
  3. ランニングコスト見積もり：月額0〜数百円で運用できるか確認（静的サイト推奨）。  
  4. 維持ルールを決める：年1回の更新でOK、バグ対応はしない等のポリシーを明示。  
  5. 決定ウィンドウを設定：2週間〜1ヶ月でプロト作る、3ヶ月で継続判断するなど期限を設ける。
- 技術スタックの提案（初心者向け）
  - プロト：Vite + React（またはNext.js）、GitHub Pages / Netlify / Vercel で無料ホスティング。  
  - ドメイン：安価なドメイン（お名前.com、ムームードメイン等）を1年購入すると信頼度UP。  
  - 情報収集：AIで下地を作り、人力でファクトチェック。日本語の情報源（国内販売ページ、楽天、価格.com）も参照。
- 日本市場へのヒント
  - ゴルフやシミュレーターは日本でも人気で、国内メーカーや販売ルート（ゴルフショップ、練習場）固有の事情がある。比較サイトを作るなら国内価格や流通情報を入れると差別化できる。  
  - 作ったらQiitaやZenn、Twitterで短く共有してフィードバックを集めると継続のモチベーションになる。

短い結論：楽しさだけでなく「寝てても壊れない」条件を満たすかを最初に確かめれば、サイドプロジェクトは負担ではなく資産になる。
