# Return of the Obra Dinn: spherical mapped dithering for a 1bpp 1st-person game (2017) - Obra Dinnの帰還：1bpp一人称ゲームのための球面マップディザリング（2017）
Obra Dinn風モノクロ表現を球面マッピングとディザで再現する――制約が生む表現力の秘密

## 要約
1ビット（白黒）表現の一人称ゲームで、球面マッピングを使ったディザリングによって滑らかな陰影や奥行きを演出する手法を紹介する記事の話題を、初心者向けに分かりやすく解説します。

## この記事を読むべき理由
モノクロ／1bpp表現はビジュアル面で強い個性を出せる一方、陰影表現が難しいです。低リソースやレトロ風ビジュアルを狙う日本のインディー開発者やアート志向のエンジニアにとって、実装アイデアと実践ポイントが即役立ちます。

## 詳細解説
- 1bppとは：各ピクセルが「0か1」しか取れない（白か黒か）表示方式。色数が極端に少ないため、ディザリングで中間階調を擬似的に表現する。
- ディザリングの種類：
  - 順序付きディザ（Bayerなど）：固定マスクで高速・安定。ただしパターンが目立つ。
  - 誤差拡散（Floyd–Steinberg）：階調感が良いが実装と計算コストが増える。
  - ブルーノイズ：時間的なちらつきを抑えやすく見た目が自然。
- 球面マップ（spherical mapping）とは：法線ベクトルや視線方向から球座標（緯度・経度）に変換し、そこを基準にディザマスクやテクスチャをサンプリングする手法。これにより、照明に応じた“まとめて回転する”ような階調パターンが得られる。
- 1人称視点での応用：ワールド内の各ピクセルに対し、法線やライト強度から求めた輝度を用い、球面座標ベースのディザマップで閾値比較して白黒化する。視点移動やカメラ回転でもパターンが破綻しにくい。
- 実装上の注意：
  - ディザマップの回転／スクロールでアーティファクトを隠す。
  - 時間的安定性：フレーム間でディザの位相を少しずつずらすとちらつきが減るが、意図的に残すことでフィルム感を出せる。
  - パフォーマンス：1bppバッファに直接書き出すビット演算や、GPUシェーダでの閾値比較が有効。

## 実践ポイント
- まずは順序付きディザ（小さめのBayerマトリクス）を輝度に適用して挙動を見る。
- 法線→球面座標 (u = 0.5+atan2(z,x)/2π, v = 0.5+asin(y)/π) を使ってディザテクスチャをサンプリングする。
- しきい値比較で1/0を決定：sample(dither_uv) < luminance ? 白 : 黒。
- 動きでジャギーが気になる場合は、ブルーノイズを導入して時間的に拡散させる。
- モバイルや低スペック向けは、ディザマップを小さくしタイル状に使うことでメモリと帯域を節約。

短めのGLSL風スニペット（概念例）：
```glsl
// glsl
float lum = compute_luminance(color);
vec2 sphUV = vec2(0.5 + atan(normal.z, normal.x)/(2.0*PI), 0.5 + asin(normal.y)/PI);
float d = texture(ditherTex, fract(sphUV * scale)).r;
float outBW = (lum > d) ? 1.0 : 0.0;
```

日本のインディーシーンでは、制約を活かした表現が好まれるため、この手法はビジュアル差別化や低リソースでの高い雰囲気作りにすぐ役立ちます。試作は小さなシーンで始め、ディザ種類やスケールを調整して好みの見た目を見つけてください。
