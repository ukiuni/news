# Show HN: Subth.ink – write something and see how many others wrote the same - Subth.ink：思ったことを書いて、同じことを考えた人が何人いるかを見るサービス
思わず「それ私も！」と言いたくなる匿名マイクロ投稿サービス — 投稿は保存しない代わりにハッシュで集計する、というプライバシーと可視化のトレードオフを巧みに設計したサービス

## 要約
Subth.ink はユーザーの文章をそのまま保存せず、ハッシュ（salted SHA256 と将来的に公開され得る unsalted MD5）を使って「同じ考えを書いた人数」をカウントする匿名投稿サービスです。API と CLI も用意されており、気軽に使えますが短いテキストの再現（逆算）リスクに注意が必要です。

## この記事を読むべき理由
日本でも匿名で感情やアイデアを共有する場は根強く需要があります。Subth.ink の仕組みは「個々の投稿を直接見せないで、どれだけ多くの人が同じことを考えたか」を可視化する新しいアプローチで、プライバシー設計の良い議論材料になります。開発者やプロダクト担当は設計上のメリットとリスクを理解しておくと役立ちます。

## 詳細解説
- 仕組み（要点）
  - 投稿本文そのものは保存しない（原文のままのDB保存はしないと明言）。
  - 投稿はまず salt を用いて SHA256 ハッシュ化され、そのハッシュで集計される。salt はハッシュの衝突やレインボーテーブル対策になります。
  - さらに unsalted MD5 も保存しているが、通常は表示されない。人気のある（カウントが閾値を超えた）思考については、将来的に MD5 を公開する可能性があるとしています。MD5 は短いテキストに対して総当たり（ブルートフォース）で復元されやすい点が問題です。
- API/CLI
  - POST /api/thoughts に {"contents":"..."} を送るとレスポンスで count（同じハッシュの投稿数）や hashed（ハッシュ値）、作成・更新時刻を返します。
  - top を得る GET /api/thoughts/top で上位ハッシュとカウントが取得できます。
  - CLI からは標準入力経由の curl で投稿可能：下例参照。
- プライバシー上のトレードオフ
  - Salted SHA256 は強力だが、もし後に unsalted MD5 が公開されると短いフレーズは辞書攻撃や総当たりで復元可能になります。
  - 「同じ文を複数人が投稿した」という事実自体がメタデータとして残るため、希少な内容や個人特定情報は危険です。
- 技術的に気をつけるポイント
  - 短く簡単な文（例：「好き」「疲れた」）は MD5 の辞書に含まれやすく、復元される可能性が高い。
  - salt の管理方法（公開されるか否か、ローテーション）次第で安全性は大きく変わります。公開設計が明記されていない点は要注意。

## 実践ポイント
- 投稿する時の注意
  - 個人情報や識別可能な固有名詞、場所、連絡先は書かない。
  - 想いを匿名で残したいなら、長めかつ具体的すぎない表現にする（長い文章は総当たりが難しい）。
- 開発者向けチェックリスト
  - サービスを導入・参考にする場合、salt の管理ポリシーと MD5 の公開基準を明確にすること。
  - カウント表示だけでプライバシーが担保されているわけではない旨を利用規約や UI で明示する。
- 手元でハッシュを試す（CLI例）
  - 送信前に自分で md5 / sha256 を確認する：
```bash
# bash
echo -n "hello world" | md5sum
echo -n "hello world" | sha256sum
```
  - Subth.ink の CLI 例：
```bash
# bash
echo "hello world" | curl -d @- https://subth.ink
```

まとめ：Subth.ink は「匿名で同意の可視化」をシンプルに実現していて面白い実験です。ただし、短文の復元リスクや salt 運用に依存するため、使う側・作る側ともにプライバシーの限界を理解しておく必要があります。
