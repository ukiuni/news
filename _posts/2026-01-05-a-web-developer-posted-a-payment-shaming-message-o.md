# A Web Developer Posted a Payment Shaming Message on Their Client's Site - ウェブ開発者がクライアントのサイトに支払い督促メッセージを掲載
未払いトラブルがウェブ上で公開化──「支払え、さもなくばアクセスできない」とサイトに直書きされたときに起こること

## 要約
ある開発者が、納品済みのクライアントサイトに「サービスは提供済み。支払いがない。アクセスが必要なら支払え」といった督促メッセージを直接掲載して公開した。技術的には簡単でも、法的・倫理的・業務上のリスクが大きい行為だ。

## この記事を読むべき理由
日本では対立を公にしない文化が強く、企業の評判は事業継続に直結する。未払い対応を誤ると、開発者・クライアント双方にとって法的問題・信用喪失・ビジネス損失を招くため、技術者としての「安全で実務的な対応方法」を知っておく必要がある。

## 詳細解説
何が起きたか
- 開発者はクライアントの公開サイトを書き換え、ページ上に未払いを主張する文言を表示した（抜粋: "Services were delivered. Payment from Joseph Smith Furniture remains outstanding. If you need access, pay me."）。
- 手段としては、CMS管理画面、FTP/SFTP、ホスティングのファイルマネージャ、もしくはサーバ上のindex.html/phpを書き換えるなど、持っているアクセス権で簡単に実行可能。

技術面のポイント
- サイト上の公開メッセージは誰でも目にするため、ブランド被害が即時に発生する。
- サーバ側での一時的な表示（メンテナンスページ）ならHTTPステータス503を返すべきで、意図せずSEOやキャッシュに悪影響を与えない配慮が必要。
- 管理者権限を乱暴に奪うと、最終的にサイトの復旧やデータ保全に支障を来す可能性がある（バックアップの消失、設定変更、メール機能停止など）。
- 法的観点では、公開の内容が名誉毀損や営業妨害に当たる恐れ、契約条項によっては損害賠償問題に発展する可能性がある。

日本市場との関連
- 日本では「内容証明郵便」や支払督促制度、少額訴訟が一般的な解決手段。先に公開恫喝を行うと、事後に当事者が不利になるケースが多い。
- 企業間取引では取引継続・信用が重要視されるため、公開の感情的対応は将来的な受注機会を自ら失うリスクが高い。

倫理・実務の観点
- 開発者としてのプロフェッショナリズムは守るべき。支払い遅延への対応は契約書・請求書・催告（書面）→法的手続きの順が安全。
- クライアント対応に備えた明確な契約（納品後の権利移転条件、保守の停止条項、遅延利息、アクセス制御ルール）を事前に用意することが重要。

## 実践ポイント
- 契約を強化する
  - 納品・検収・所有権移転のタイミング、支払遅延時の措置（遅延利息、サービス停止の条件）を明文化する。
- 支払い保全の仕組み
  - エスクロー、リテイナー（前金）、段階リリースを導入する。
- 技術的に安全な一時停止方法
  - 公共の「恫喝メッセージ」ではなく、403/503返却によるアクセス制限や非公開の管理画面ロックを利用する。例：シンプルな503メンテナンスページ
```html
<!-- html -->
<!doctype html>
<html lang="ja">
<head>
  <meta charset="utf-8">
  <title>メンテナンス中</title>
  <meta name="robots" content="noindex">
</head>
<body>
  <h1>サイトは現在メンテナンス中です</h1>
  <p>サービス提供者にお問い合わせください。</p>
</body>
</html>
```
  - サーバからは必ずHTTP/1.1 503 Service UnavailableとRetry-Afterヘッダを返す（SEO/キャッシュ対策）。
- 未払い発生時の実務フロー（日本向け）
  1. まずは請求書再送とメール・電話での督促。
  2. 重要書類は内容証明郵便で送付。
  3. それでも解決しない場合は支払督促や少額訴訟を検討。
  4. 最終手段として公開する場合は法的助言を得て、名誉毀損や信用毀損にならない表現に限定する。
- もしクライアント側で公開されたら
  - ホスティング会社やドメインレジストラに連絡し、（不正なアクセスであれば）アカウントの一時停止やログ確認を依頼。
  - バックアップからの復旧、パスワード変更、二段階認証の導入を迅速に実施。
  - 記録（変更ログ、アクセスログ、やり取りの保存）を確実に残す。

短く言うと：技術的に可能だからといって、公開での「支払い恫喝」は最善策ではない。契約と手続き、そしてプロとしての対応を整えておくことが、最もコストが低く安全な解決につながる。
