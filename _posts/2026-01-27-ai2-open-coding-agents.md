# AI2: Open Coding Agents - オープンなコーディングエージェント
驚くほど安く・簡単に自分のリポジトリに最適化できる「SERA」が来た：小〜中チームでも本番級のコーディングエージェントを作れる理由

## 要約
AI2が公開したSERA（Soft-verified Efficient Repository Agents）は、合成データ生成と低コストな微調整手法で、誰でも自分のリポジトリ向けに高性能なコーディングエージェントを作れるようにしたオープンな仕組みです。

## この記事を読むべき理由
企業や個人の内部コードは公開モデルに知られていないため、その差を埋める「自社化」手法が重要です。SERAは高性能を比較的少ない計算資源・費用で実現し、日本のスタートアップや組織でも実践可能な現実解を提示します。

## 詳細解説
- 何が新しいか：SERAは「すべてオープン（モデル、データ、レシピ、Claude Code連携）」で、標準的な教師あり微調整（SFT）のみでエージェント的振る舞いを学ばせます。大規模なRLインフラや高額な検証パイプラインは不要です。  
- コア技術
  - Soft-verified generation (SVG)：完全に正しいパッチでなくても学習に役立つ「部分的に正しい」合成パッチを大量に作ることで、厳密なテスト不要に。これによりデータ生成コストと運用複雑性を大幅に削減します。
  - バグ種類メニュー：51種類の一般的バグパターンを用い、リポジトリ中の関数ごとに多様なバグ風プロンプトを生成。数千関数から数万の軌跡を低コストで作れます。
  - リポジトリ訓練：開発者のワークフローに即した疑似軌跡（修正・レビューの流れ）を作ることで、単なる「正解コード」よりも実務的な能力を伸ばします。
- 性能とコスト指標
  - モデル群：Qwen3系をベースに8B～32Bパラメータ。SERA-32BはSWE-Benchで64Kコンテキスト時に約54.2%を達成。32Kでも強豪と互角。  
  - 再現コスト：既存の最良オープン結果を再現するのに約$400、業界トップ級に迫る性能再現は最大約$12,000。SERAは従来手法より数十倍安価です。  
  - プライベート適応：例としてSERA-32Bは教師モデル（110B）を用いたケースで、DjangoやSymPyのような実リポジトリに対し8,000サンプル（約$1,300）で教師を上回ることが確認されています。小さいモデルでも特化すれば大きい汎用モデルを超えうる点が肝。
- 推論最適化：NVIDIAと協業しH100/B200等で高スループット（BF16で約1,950 tok/s、FP8で約3,700 tok/s、次世代でさらに向上）を実現。Claude Code互換や軽量デプロイ用CLIも提供。

## 実践ポイント
- 小〜中チームはまずSERAの公開レシピで「ターゲットリポジトリの8k合成軌跡」を作ってSFTを試すと効果検証が早い。コストは数千ドル〜に収まる可能性が高い。  
- 内部APIやコーディング規約が重要なら、リポジトリ訓練（repository training）でその部分を重点的に含める。  
- 効率重視なら完全検証を目指さずSVGを活用：テストインフラを作り込む前に、部分的に正しい合成データで挙動を評価してみる。  
- 本番導入時は推論最適化（FP8/NVFP4など）を検討してコストとレイテンシを抑える。  
- まずは公開モデル＋SERAレシピでローカルまたはクラウド少数GPUで再現し、成果が出れば運用化を検討する。

SERAは「オープンで安価に自分ごとのエージェントを作れる」ことを実証したリリースです。日本の開発現場でも、内部コードに強いエージェントを小規模コストで導入する現実的な道筋になります。
