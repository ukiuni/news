# 80% of Rye in 20% of the Time [1/3] - 20%の時間でRyeの80%を学ぶ（1/3）
たった数分で触れるRye入門：Nullなし・式で書く新世代スクリプト言語の最重要ポイント

## 要約
RyeはRebolやShellに影響を受けた動的でホモアイコニック（コードとデータが同形）の言語。Nullを排し、既定で定数、すべてが式、キーワードを持たない設計で、評価は「ワードの型」によって制御される点が特徴。

## この記事を読むべき理由
- Null参照や副作用に悩まされがちな日本の開発現場で、より安全で明快なスクリプト／ビルド用言語の選択肢になる可能性があるため。
- 小さなランタイムで実用的なツールを作れる設計思想は、社内ツールやCIスクリプトの刷新に向く。

## 詳細解説
- 基本性質
  - 動的でホモアイコニック：コードはブロック（配列的な構造）で表現され、データと同じ構造で扱える。
  - Nullがない：不在の値がない設計は多くのバグを未然に防ぐ。
  - 定数がデフォルト：明示的に変数を作る操作が必要。状態変更を明示することで副作用を抑える。
  - すべてが式：値や関数呼び出しは常に式として評価されるため、柔軟な合成が可能。
  - キーワード無し：制御構造も関数として実装され、言語コアが非常に小さい。

- ワード（識別子）の型による評価制御
  - set-word, mod-word, get-word, lit-word など、ワードには「どう評価されるか」を決める種類がある。  
  - 例えば get-word は関数を呼び出さずに値を取得し、set/modify 系は結びつきを作る・更新する。
  - op-word（演算子風のワード）や pipe-word（パイプ）といった特殊な振る舞いもある。

- ブロックはデータにもコードにもなる
  - ブロックを渡して遅延評価したり、データとして処理したりできる。これがホモアイコニック設計の強み。

- 評価モデル（簡易）
  - リテラルは自分自身に評価され、ワードはその型に従って振る舞う。関数値なら呼び出しが行われる。
  - 言語コアは非常に小さく、機能は主に組み込み関数（built-ins）で提供される。

- 最小ランタイムの例（記事の示唆）
  - まずは4つの組み込みで動かせる：true（真値）、print（出力）、if（条件実行）、loop（繰り返し）。
  - そこから prns, range, for, either, is-multiple-of などを追加すると簡単なループや条件分岐が実用的に書ける。

- 実用的ビルドブロック（よく使う組み込み）
  - 算術・比較（+、< など）、any/all（論理集約）、map/reduce、join/head/max/length?、transpose、parse-json、match など豊富。
  - op-words を使えば演算子風の短い表現で処理をチェインできる。

コード例（最小の挨拶とループ）
```rye
if true { print "Hello Rye" }
loop 3 { print "Wah" }
```

## 日本市場との関連性
- CI/CDや社内ツールの小型化：GoやShellに近い思想を取り入れつつ、より安全で表現力の高いスクリプト言語は、日本の企業が運用自動化を進める際の選択肢になりうる。
- 学習コストの低さ：式中心で副作用が明示的なので、教育用や新人のオンボーディングにも向く。
- Nullレス設計は金融・組み込みなど堅牢性が求められる分野で魅力的。

## 実践ポイント
- まずはドキュメントとREPLで手を動かす：公式のFunction referenceを参照し、基本のprint/if/loopとrange/mapを試す。
- 小さなShellスクリプトをRyeに置き換えてみる：ログ整形やJSONパースなど、ツールの試作に最適。
- 定数デフォルトと明示的変数（var）の違いを意識し、状態変更は最小限にする設計に慣れる。
- VS CodeではRyeファイルを開いて実験し、CIジョブの一部を簡単なRyeスクリプトにして動かしてみる（まずはローカルで動作確認）。
