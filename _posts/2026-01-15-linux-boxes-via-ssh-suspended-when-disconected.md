# Linux boxes via SSH: suspended when disconected - SSHで使えるLinuxボックス：切断で一時停止
ブラウザ不要・即起動の「SSHだけ」で使える軽量Linuxボックス——使い方も料金もシンプル

## 要約
shellboxは「sshだけ」で瞬時に立ち上がるLinuxインスタンスを提供するサービス。切断するとボックスが一時停止し、再接続でそのまま復帰。課金は利用時間に応じた従量課金で、公開HTTPSエンドポイントとフルSSH機能を備える。

## この記事を読むべき理由
日本の開発者や教育現場、スタートアップにとって、環境構築の手間削減・コラボの簡素化・コスト最適化に直結するサービスだから。ローカル環境を汚さずにデモや検証を素早く行える点は特に魅力的だ。

## 詳細解説
- アクセス方法：専用クライアント不要で標準のSSHだけで操作。セットアップは極めて簡単。
  - 例：ボックス作成、接続、一覧表示はCLIで完結。
```bash
# ボックス作成
ssh shellbox.dev create dev1

# ボックスに接続（ttyを使う）
ssh -t shellbox.dev connect dev1

# ボックス一覧
ssh shellbox.dev list
```
- インスタンス仕様：軽量プランの例で2 vCPU、4GB RAM、50GB SSDなど。即座に使えるスペックが用意されている。
- 永続性と一時停止：SSHで切断するとボックスは「停止（paused）」状態になり、次回接続で前の状態から再開される。これにより、稼働時間＝課金対象の時間のみを最小化できる。
- 課金モデル：実行中は $0.05/時、停止中は $0.005/時（excerptに基づくサンプル）。前払いバランスをチャージして使い、残高が$5未満だと自動停止、$0で削除（サービスのルールに従う）。
- ネットワークと公開URL：各ボックスには自動でHTTPS付きの公開URLが割り当てられ、TLSが自動管理されるため、簡単にウェブ公開できる。加えて通常のSSHポートフォワーディングやscpもサポート。
- ファイル転送注意点：OpenSSH 9.0以降のscpでは互換オプションが必要な場合がある（-Oフラグ）。
```bash
# ローカルからボックスへ
scp -O file.txt shellbox.dev:dev1:/root/

# ボックスからローカルへ
scp -O shellbox.dev:dev1:/root/file.txt ./
```
- 管理と安全性：支払い・残高確認、返金対応などのCLIコマンドが用意されている。自動停止ルールはコスト管理に便利だが、機密データの保存・バックアップポリシーは自分で設計する必要がある。

## 実践ポイント
- すぐ試す：短時間の検証やペアプログラミング、ハンズオン教材に最適。まずは1つ作って接続・公開URLで動作確認してみる。
- コスト制御：使わないときは必ず切断（自動的に一時停止）。残高は最低ラインを超えるよう運用ルールを決める（例：残高$10を下回ったら通知）。
- デモ用公開：HTTPS URLで簡易デモを共有できるため、外部レビューやクライアント向けプロトタイプに便利。ただし公開URLには認証やCORS等の保護を付与すること。
- ファイル転送：scpで転送する際はOpenSSHのバージョンに注意し、必要なら -O を付ける。
- セキュリティとバックアップ：一時停止は“稼働停止”であって削除ではないが、残高0で削除されるリスクがあるため重要データは外部バックアップやgitに置く。

短時間で環境を立ち上げて手放せる「使い捨て＋永続状態」のハイブリッドは、日本の開発現場や教育での実務効率化に役立つ。まずは無料トライアルや少額チャージで挙動を確認してみると良い。
