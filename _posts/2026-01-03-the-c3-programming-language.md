# The C3 Programming Language - C3 プログラミング言語
ワークホースなCを「進化」させた新世代言語 — 親しみやすく、低レイヤーで使えるCの延長線上にある選択肢

## 要約
C3はCの文法感覚を残しつつ、安全性・生産性・拡張性を強化した言語で、CとのABI互換を維持したまま段階的に既存プロジェクトへ導入できる。

## この記事を読むべき理由
既存のC/C++資産が豊富な日本の開発現場（組み込み、ゲーム、システムソフトウェア）にとって、互換性を保ちながら安全性と表現力を上げられる選択肢は実際的で価値が高い。レガシーコードを捨てずに改善したい技術者は特に注目すべき。

## 詳細解説
C3は「革命」ではなく「進化」を標榜する言語で、Cプログラマに馴染みやすい記法を保ちながら次のような機能を追加している。

- Cとの完全なABI互換  
  Cで書かれたライブラリやバイナリとそのまま連携でき、特別なラッパー型や制約なしに混在ビルドが可能。既存コードベースへの部分導入が容易。

- モジュールシステム  
  シンプルで邪魔しないモジュール機構を備え、名前空間や依存管理を過度に複雑化しない設計。

- 演算子オーバーロード  
  C++の複雑さを持ち込まず、ベクトルや行列、固定小数点などの用途で読みやすい演算子定義が可能。

- コンパイル時／セマンティックマクロ  
  Cのプリプロセッサよりずっと強力で分かりやすいマクロを提供。関数のように記述でき、コンパイル時処理を安全に表現できる。

- グラデュアルな契約（Gradual Contracts）  
  実行時とコンパイル時の制約を自然に書ける仕組み。段階的に導入して安全性を高められる。

- ゼロオーバーヘッドなエラー処理  
  Result型的な明確さと例外の使いやすさを組み合わせ、ランタイムコストを抑えつつ扱いやすいエラー設計。

- ジェネリックモジュールと型の簡潔さ  
  汎用型を分かりやすく表現でき、テンプレート地獄を避ける設計。

- コンパイル時・ランタイムの型反射  
  型情報をコンパイル時と実行時の両方で参照でき、柔軟なメタプログラミングが可能。

- インラインアセンブリとデバッグ支援  
  生のアセンブリを文字列化せず普通のコードとして埋め込め、デバッグ時は境界チェックや詳細なスタックトレースを標準で得られる。

実例として、vkQuakeの一部をC3に移してビルドしたケースがあり、実用性が示されている（現在のリリース例: 0.7.8）。公式コンパイラは c3c で、Windows／macOS／Linux向けに提供されている。

簡潔なサンプル（モジュールとエントリ）:
```c3
module hello_world ;
import std :: io ;

fn void main () {
  io :: printn ( "Hello, world!" );
}
```

## 実践ポイント
- まずは公式コンパイラ c3c をダウンロードして、既存小モジュール1つだけC->C3に置き換えてビルドする。ABI互換が確認できるはず。  
- デバッグビルドで境界チェックやスタックトレースを有効にして、既存のバグ検出に活用する。  
- パフォーマンスが重要な箇所はインラインアセンブリで最適化しつつ、上位ロジックはC3の安全構造で書くハイブリッド設計を検討する。  
- マクロと契約を少しずつ導入し、テストで副作用を確認しながらコードベースの堅牢性を高める。  
- 日本の組み込み系やゲーム会社では既存Cライブラリが資産になるため、全置換ではなく段階的導入を提案しやすい。
