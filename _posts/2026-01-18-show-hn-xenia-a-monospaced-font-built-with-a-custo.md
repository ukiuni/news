# Show HN: Xenia – A monospaced font built with a custom Python engine - Xenia — カスタムPythonエンジンで作られた等幅フォント
「見た目が良くて読みやすいモノスペース。Xeniaでコード表示を一新しよう」

## 要約
XeniaはPythonで自動生成された等幅（モノスペース）フォントで、700以上のグリフを持ち、数字や記号の曖昧さを排した設計が特徴。無料・MITライセンスで配布されており、エディタで簡単に試せます。

## この記事を読むべき理由
読みやすい等幅フォントはコーディング効率と可視性に直結します。日本のエンジニア環境でも、記号や数学記号が多いドキュメントやコードを扱う場面は多く、視認性の高いフォント選びは小さな生産性向上につながります。Xeniaは「見た目の良さ」と「記号対応」を両立している点で試す価値があります。

## 詳細解説
- 何が特徴か  
  - 700+ グリフ：シンボルや数学記号の対応が厚めで、数式や特殊文字を含むコードやドキュメントでの崩れが少ない。  
  - non_ambiguous設計：1（数字の1）、l（小文字のエル）、I（大文字アイ）、0（ゼロ）、O（オー）などの区別が明確で、誤読を減らす意匠。  
  - クリーンなジオメトリ：不必要に凝った小文字（作者の表現では「yucko」なaなど）を避け、読みやすさを優先。  
  - Pythonで生成：グリフを手作業ではなくカスタムの手続き型エンジンで生成しているため、将来的にウェイト追加や派生フォントを自動生成しやすい設計になっています。  
- 形式・配布  
  - TTF（xenia_regular.ttf）がリポジトリにあり、MITライセンスで利用可能。スターが付いている小さな個人プロジェクトですが、商用利用も問題ありません。  
- 欠点・注意点  
  - リポジトリの説明からは日本語（CJK）グリフの対応は明記されていないため、エディタ上で日本語表示が必要な場合は別途フォールバック設定が必要です。  
  - 現時点でウェイトは1つのみ。太さや斜体が必要なら追加開発を待つか、自前で組み合わせる必要があります。

## 実践ポイント
- インストール（簡単）  
  1. リポジトリから xenia_regular.ttf をダウンロード  
  2. OSでフォントをインストール（ダブルクリック→インストール／Linuxは~/.fontsへ配置）  
  3. エディタでフォントを指定して再起動

- VS Codeでの設定例
```json
{
  "editor.fontFamily": "Xenia, 'Noto Sans JP', Consolas",
  "editor.fontSize": 14,
  "editor.fontLigatures": false,
  "editor.lineHeight": 20
}
```

- 日本語環境での実用Tips  
  - 日本語（CJK）文字が主に必要なら、フォールバックにNoto Sans JPやメイリオを指定して混植する（上の例参照）。  
  - 記号や数学記号の表示崩れをチェックする：ドキュメント内の特殊記号やMarkdownプレビューで確認。  
  - ペアプログラミングやスクリーンショットで見栄えを試す：視認性の差はチームでのレビュー効率に影響します。

- コミュニティ貢献・評価  
  - MITライセンスで商用利用OK。気に入ったらリポジトリにスター、Issueで要望（日本語グリフ対応など）を出すと反応が返るかもしれません。作者はウェイト追加を検討しているようなのでフィードバックが有効です。

興味があれば、まずはローカルに入れて数時間コードを書いてみて、数字・記号・括弧類の見え方を評価してみてください。
