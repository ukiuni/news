# Anthropic: Developing a Claude Code competitor using Claude Code is banned - Anthropic：Claude Codeを使ってClaude Codeの競合を開発することは禁止されている
Anthropicが示した「モデルでモデルを作るな」というライン — あなたの開発は知らずにルール違反になっていませんか？

## 要約
海外のツイートでは、Anthropicが「Claude Codeを使ってClaude Codeの競合製品を開発することを禁止している」と指摘されています。もし実際にそうなら、モデル出力を使った再学習や競合製品開発に法的・契約的な制約がある可能性があります。

## この記事を読むべき理由
日本のスタートアップや開発者も、外部APIや生成AIを活用してプロダクト開発をする機会が増えています。APIの利用規約やモデルの出力をどう使えるかを理解しておかないと、知らぬ間に契約違反や知財問題に巻き込まれるリスクがあります。

## 詳細解説
- 元ツイートの主張
  - ツイートは簡潔に「AnthropicがClaude Codeを用いてClaude Codeの競合を作ることを禁止している」と述べています。X（旧Twitter）の表示制限やスクリーンショットの問題で断片的な情報しか得られないことが多い点に注意。

- 想定される禁止の対象
  - 「モデル盗用（model extraction）」：大量のクエリで出力を集め、別モデルを再構築・再学習する行為。
  - 「出力を学習データにすること」：Claude Codeの生成物をそのまま新モデルのトレーニングデータに使うこと。
  - 「競合プロダクトの直接的開発」：出力を基に同等機能の商用サービスを作る行為。

- 技術的背景（初心者向け）
  - 現代の大規模言語モデルは「訓練データ」と「学習手法」双方が価値ある資産です。外部APIから得た出力を大量に集めて別モデルを作ると、元モデル提供者の技術的優位性やデータ価値が侵害される恐れがあります（これが「禁止」される理由）。
  - 防止手段としては利用規約での明確化、出力のウォーターマーク、API利用ログの監査などがあります。

- 契約的・法務的な側面
  - APIやクラウドサービスには利用規約（Terms of Service）や追加の企業向け契約があり、「派生モデルの作成禁止」「商用利用の制限」などの条項が含まれることがあるため、契約違反リスクを常に検討する必要があります。

## 実践ポイント
- 利用規約をまず読む：APIを使う前に「禁止事項」「再配布」「派生利用」の条項を確認する。
- 目的を明確に：研究用途・評価用途・商用用途で許可範囲が変わることが多い。用途に合った契約を取る。
- ログとデータ管理を徹底：どの入力と出力を保存し、どのように学習データに使うかを記録しておく。
- 代替案を用意：完全自社開発が困難なら、オープンソースのモデルやライセンスの緩い提供者を検討する（国内のコミュニティ製品も含めて比較）。
- 法務に相談：不明点や大規模な商用展開をする場合は弁護士や契約担当に相談する。

（注）この記事は元ツイートの指摘を基に解説したもので、Anthropic社の正式なポリシー全文を示すものではありません。実際の契約条項は公式ドキュメントで必ず確認してください。
