# Vibe Coding Is Killing Open Source Software, Researchers Argue - 「バイブコーディングがオープンソースを殺している」と研究者が警告
バイブコーディングの“便利さ”が、気づかないうちにOSSの基盤を蝕んでいる理由

## 要約
欧州の研究チームは、LLMを使った「vibe coding」（勢いで素早くコードを書く行為）がオープンソースソフトウェア（OSS）エコシステムに深刻な負荷を与えていると指摘しています。短期的には生産性向上に見えても、長期的には保守者疲弊や依存性崩壊を招く恐れがあります。

## この記事を読むべき理由
日本のスタートアップや企業開発でも、OSSと生成系AIの組み合わせは既に日常です。今後の開発品質・セキュリティ、そしてOSSコミュニティの持続可能性に関わる問題なので、エンジニア／マネージャーともに知っておくべき話です。

## 詳細解説
- vibe codingとは：LLM（大規模言語モデル）を利用して短時間でコードを生成し、十分なレビューや理解なしにそのまま流用・デプロイする開発スタイルの俗称。開発効率は上がるが「ブラックボックス化」が進む。  
- 依存関係の問題：LLMは大量のOSSコードを学習源にしているため、生成物は既存ライブラリやパターンに強く依存する。結果、さらに多くの開発者が同じ小さなプロジェクト群に依存し、保守者（maintainer）への負荷が集中する。  
- 保守者疲弊と供給側リスク：研究は、小規模プロジェクトの維持者が疲弊して離脱すると、ライブラリの更新や脆弱性対応が止まり、サプライチェーン全体が脆弱になる点を警告している。  
- セキュリティと品質の低下：レビュー不足の生成コードはバグやセキュリティ穴を含みやすい。さらに、ライセンス違反や帰属の問題も顕在化する可能性がある。  
- エコシステム的視点：OSSは「共有の資産（commons）」であり、消費だけでなく継続的な貢献や資金供給がなければ持続できないという古典的な問題に、LLMが新たな圧力をかけている。

## 実践ポイント
- LLM出力は「下書き」と割り切り、必ずコードレビューとテストを課す。  
- 依存ライブラリの棚卸し・SBOM作成と定期的な脆弱性スキャンを実施する。  
- 社内OSSポリシーを整備し、外部プロジェクトへのコントリビューションやスポンサーを検討する。  
- 自動生成コードの出所（モデル／プロンプト）とライセンス要件を記録する。  
- 小さなOSSプロジェクトの保守者支援（寄付、スポンサー、ドキュメント整備）を組織的に行う。

（出典：欧州の研究チームによる調査を踏まえた報告記事の要約と考察）
