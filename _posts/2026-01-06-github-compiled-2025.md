# GitHub Compiled 2025 - GitHub Compiled 2025（2025年のコードまとめ）
自分だけの“コード年鑑”をブラウザで作って動画でシェアする新定番 — GitHub Compiled 2025で作る“見せる履歴書”

## 要約
GitHub Compiled 2025は、公開APIとブラウザの最新メディアAPI（Web Codecsなど）を使い、ドラッグ＆ドロップでコンポーネントを配置して自分の2025年のGitHub活動をカスタム動画として書き出せるクライアントサイドのツールです。Remotion系の技術を活用した、ソーシャル向けに最適化された“Wrapped”キャンペーンの挑戦作です。

## この記事を読むべき理由
日本のエンジニアやコミュニティ運営者にとって、自分の活動を短い動画で効果的に伝える手法はポートフォリオ、採用発信、OSS貢献の可視化で即戦力になります。しかもこれをブラウザ上で完結させるアプローチは、社内のキャンペーンやエンジニアブランディングへの応用余地が大きいからです。

## 詳細解説
- 仕組みの要点  
  - 公開GitHub APIからデータを取得し、ユーザーが選べる「コンポーネント」をブレッドボード上で配置してレイアウトを作成する。  
  - 動画生成はサーバーではなくクライアント側で行う設計。Web Codecs API（Chromium系ブラウザが必要）やブラウザのメディアレンダリング能力を活用してフレームを生成・エンコードするため、API呼び出しだけで完結し低遅延かつコストを抑えた実装が可能。  
  - Remotionの2023キャンペーンに影響を受けており、Reactベースのコンポーネント駆動レンダリングを踏襲している可能性が高い（UIでのドラッグ＆ドロップ、テキスト入力、カメラエフェクトを用意）。  
- デザイン思想（Wrappedキャンペーン論）  
  - ただ数字を並べるだけでなく、ユーザー自身が見せたい統計やコメント、目標を入れられることで「誇れる」「話題になる」コンテンツを目指す。  
  - 成功の3要件として「共有されるか」「第三者が関心を持つか」「アルゴリズム（主に動画優先）に乗るか」を挙げ、動画化＋パーソナライズでこれをクリアしようとしている。  
- 制約と注意点  
  - 動作はChromium系（Chrome / Edge 等）推奨。Web Codecs等の未対応ブラウザやモバイルは非対応・最適化不足。  
  - 公開APIのみを使用するためプライベート情報は反映されない。公開リポジトリ中心の可視化である点を理解すること。  
  - クライアントでエンコードするためPCスペックやブラウザの実装差で処理時間やFPSに差が出る（UIにフレーム進行表示などがある）。

## 実践ポイント
- まずはChromium系ブラウザのデスクトップで開く（モバイルは非対応）。  
- GitHubのユーザー名と公開リポジトリを整理しておく：表示したいデータが公開であることを確認。  
- コンポーネントを選んで自分の“ストーリー”を作る：数字だけでなく短いコメント・目標を入れると拡散力が上がる。  
- 出力された動画はSNS向けに短め（15–60秒）にまとめ、字幕やサムネイルを付けると再生率が改善する。  
- 自社やコミュニティで類似キャンペーンを企画する場合は、Remotion/Web Codecsを調査してクライアント側レンダリングのコスト・UX設計（モバイル対応の落とし所）を早めに検討する。  
- セキュリティ/プライバシー：公開データのみ扱う点は強みだが、個人情報や企業内秘情報を含めないよう注意する。

興味があれば、社内ブログや採用ページでの「年次まとめ動画」導入案を短く作って提示してみると反応が取りやすいでしょう。
