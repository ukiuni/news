# Show HN: I built an Open Source screen timer for the m5stickc (Arduino) - m5stickc（Arduino）向けオープンソース・スクリーンタイマーを作った

子どもも納得する「物理タイマー」で家族のスクリーン時間をスマートに管理する方法

## 要約
イギリスの開発者が、子どもの「正確なスクリーン時間計測」と家族内の信頼を両立させるために、m5stickc（Arduino/ESP32ベース）の小型デバイスと親側のウェブアプリを組み合わせたオープンソースのスクリーンタイマー「Screenie」を作成したプロジェクトです。

## この記事を読むべき理由
日本でも複数デバイスにまたがるスクリーン時間管理は家庭の悩みになりがちです。プラットフォームごとの制御がバラバラで子どもの自己申告に頼るしかない状況を、手頃なハード＋ソフトで改善するアイデアは、家庭向けプロダクトや教育系サービスのヒントになります。

## 詳細解説
- 問題設定：iOS/Android/ゲーム機など複数のプラットフォームで親の制限を横断的に同期するのは事実上難しく、子どもの「感覚的な時間把握」が大きくずれる。  
- デザイン哲学：強制的にデバイスをロックする仕組みを増やすのではなく、「計測を正確にし、子どもに信頼を置く」アプローチ。ゴールは取り締まりではなく自己管理の促進。  
- ハードウェア：m5stickc（ESP32系の小型デバイス）＋Arduino開発環境を想定。小さな画面とボタンを使い、ワンタッチで開始/停止できるシンプルなUI。物理的に目に付く専用機器にすることで利用を促す効果をねらう。  
- ソフトウェア：デバイス側はタイムスタンプでセッションを記録し、Wi‑Fi経由で親のウェブアプリと同期。ウェブアプリは上限設定・ベッドタイム設定・現在のセッション表示・追加時間リクエスト受信などを担う。サーバー側はREST APIやシンプルなDBで実装可能（リアルタイム性はWebSocketや短周期ポーリングで補う）。  
- 運用ルール：唯一の掟は「必ずタイマーを使うこと」。監視するなら視覚的なチェック（デバイスが手元にあるか、稼働中か）やウェブからのセッション確認で済ます。

## 実践ポイント
- まず試すなら：M5StickCを1台、Arduino IDEを用意して公式/コミュニティのサンプルをビルド。ボタンでの開始停止とローカルのセッションログを書き出すところから始める。  
- 同期方法の選択：家庭内Wi‑Fi経由で小さなREST APIサーバ（ラズパイやクラウド）にPOSTして同期するのが手軽。プライバシー重視ならローカル限定で設計する。  
- 家族の合意形成：技術よりもルール作りが肝心。「タイマーを持つこと」を習慣化し、スポットチェックと褒める仕組みを用意する。  
- 拡張案：スマホ/スマートウォッチ版への移植、学校や習い事向けの一括管理、オープンソースを活かしたコミュニティ改良。

原案はオープンソースで公開されています。日本の家庭事情にもマッチする単純明快なアイデアなので、プロトタイプ作りや製品化のヒントになるはずです。
