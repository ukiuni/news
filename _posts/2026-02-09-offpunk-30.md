# Offpunk 3.0 - Offpunk 3.0「コミュニティの誕生」
ターミナルでネットを「保存して読む」──Offpunk 3.0が切り開くオフライン×協同の新常識

## 要約
コマンドライン向けのオフラインファーストブラウザ「Offpunk」が3.0に到達。外部貢献で機能拡張が進み、記事抽出やログイン対応、Gemini画像表示など実用性が大きく向上した。

## この記事を読むべき理由
ターミナル好き、オフラインで情報収集したい人、プライバシー重視の日本の技術者やライターにとって、ブラウザの新しい選択肢とコミュニティ主導の開発モデルはすぐ役立つ知見です。

## 詳細解説
- コミュニティ化：3.0は多くの貢献を取り込んだ最初の大きな節目。翻訳・パッケージング・機能提供が外部から活発になり、協調開発が成立した点が重要。
- コンテンツ抽出（unmerdify）：FiveFiltersのftr-site-configを使うunmerdifyが統合され、記事本文のノイズ除去が改善。ルールがない場合は従来のreadabilityにフォールバックする。
- ローカル／ログイン対応：cookie-txt形式のクッキーをインポートして、自分のブラウザでログインしたセッションをOffpunkから利用可能（例：購読制サイトの閲覧）。
- 新ツール群：opnkは openk に改名。xkcdpunkでターミナル上にXKCDを表示できる独立ツールが追加。
- Gemini/画像とRSS：Geminiでの画像表示がデフォルトで大型化。HTMLページの隠れたRSS/Atomフィードを検出して一覧化する機能も追加。
- UI・挙動改善：blockedリンクの可視化、テーマプリセット、リダイレクトのネットキャッシュレベルでの処理（ブロック回避を防止）など多数の改良。コマンド名変更（ls→links）や新しいwebsearchなどの操作性向上も含む。

## 実践ポイント
- ftr-site-configを使って記事抽出を最大化するにはリポジトリをクローンしてoffpunkrcで参照を設定する：
```bash
# bash
git clone https://github.com/fivefilters/ftr-site-config.git /path/to/ftr-site-config
```
offpunkrcに追記：
```bash
# bash
set ftr_site_config /path/to/ftr-site-config
```
- ブラウザでログインした状態を持ち込みたいとき：
```bash
# bash
# ブラウザで cookie-txt をエクスポートした後
cookies import mycookie.txt https://domain-of-the-cookie.net/
```
- 既存ユーザーはopnk→openkへの置換を忘れずに。表示やテーマはoffpunkrcで細かく調整可能。

Offpunk 3.0は「端末で読み、保存し、共有する」ワークフローを堅牢にする一歩。ターミナル中心の情報収集を試したい人は試してみる価値あり。
