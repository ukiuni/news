# CSS Clicker - CSS クリックゲーム
驚愕：JavaScriptゼロで動く“サイト育成”クリックゲーム — CSSだけでここまでできるのか？

## 要約
CSSだけで動くアイドル系クリックゲーム。サイトを育てる要素（バッジ、カスタムカーソル、デザインシステム等）を購入してビュー数を稼ぎ、最終的にGeocities風の“エンドゲーム”をアンロックする作品。

## この記事を読むべき理由
「HTML/CSSはデザインだけ」という常識を覆す実例で、フロントエンド入門者がCSSの表現力とトリックを学べる教材的プロジェクトだから。

## 詳細解説
- 実装の肝：JavaScriptを一切使わず、HTMLの<input>（checkbox/radio）と<label>、CSSの隣接/一般兄弟セレクタ、擬似要素、アニメーション、カスタムカーソル、そして（採用されていれば）メディアクエリ等を組み合わせてUIの状態管理と表示切替を実現している。  
- 状態とUIの連動：チェックボックスのchecked状態をCSSで拾って表示・非表示やアニメーションを切り替える「:checkedハック」が中心。ボタンやアップグレードのオンオフ、タブ切替などはほぼすべてこの仕組みで動く。  
- JS検出の工夫：サイトはブラウザのスクリプト有無をCSS側で検出する（例：scriptingメディアクエリ等）し、JS無効時でも正常にプレイできるよう配慮している。  
- 開発・配布：コードは手書きのプレーンHTML/CSSで、ソースはブラウザの「表示ソース」で見られ、Ctrl+Sでオフライン保存できる。開発ブラウザの互換性注記（Chrome/Firefoxの特定バージョン）もあり、他ブラウザでは挙動が異なる可能性がある点に注意。

## 実践ポイント
- まずはページを開いて「表示ソース」を確認し、:checked + label や擬似要素の使い方を追ってみる。  
- ブラウザのJavaScriptをオフにして動作確認し、CSSだけでUI状態を管理する感覚を掴む。  
- ミニ演習：チェックボックス＋labelでトグルを作り、CSSアニメーションでフェードやスケールを付けてみる（実装の第一歩として有効）。  
- 学びを活かす場面：オフライン・制約付き環境のUIプロトタイプ、CSSコンテスト／チャレンジ、教材やポートフォリオの遊び心あるデモに最適。
