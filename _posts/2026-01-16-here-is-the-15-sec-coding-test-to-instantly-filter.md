# Here is the 15 sec coding test to instantly filter out 50% of unqualified applicants by JOSE ZARAZUA - 元CTOが使った「15秒コーディングテスト」で不適格者を瞬時に50%ふるい落とす方法

魅力的なタイトル: 15秒で合否が分かる？元CTOが使った“コピー＆ペースト”を見抜く超簡単テスト

## 要約
元CTOが応募フォームに超簡単なコード問題を置くだけで、応募者の半分を瞬時にふるい落とせたという手法の紹介。肝は「人間が脳内で解ける／そのままコピーして実行する」の差を利用すること。

## この記事を読むべき理由
リモート採用や大量応募が発生する採用活動で、短時間で不適切な応募を減らす実戦的テクニックを知れる。日本でもスタートアップやプロダクト開発チームの採用効率を上げる実務的アイデアになる。

## 詳細解説
元記事の肝は「とても単純なコードを出して、頭で即答できる人とコピー＆実行して答える人を分ける」こと。出題例（Python風）：

```python
result = 0
for x in [3, 3, 5]:
    if x >= 3:
        result = result - x
    else:
        result = result + x
# 答えは -11
```

見た目では作者がHTMLの表示トリックで条件を書き換え（例：画面上は `if x > 3` に見えるが、実際のコードには `>=` が隠れている）しておき、読む人が頭で実行すると「$1$」になる一方、コピーしてインタプリタやAI（ChatGPT等）で実行すると実際のコードの挙動に従い「$-11$」が返る――というわけです。投稿者の実運用では、約50%がコピー＆実行の答え（AI/インタプリタ由来）、約47%が頭で正しく解いた回答、残りは誤答だったと報告されています。

注意点・実務的な問題点:
- HTMLやRSS、ブラウザ差で「隠し文字」があらわになったり、スクリーンリーダーに影響する可能性がある。
- 手法はあくまで迅速な一次フィルタであり、誤判定（有能な人を落とす）も起き得る。
- 倫理・公平性や障がい者対応の観点で配慮が必要（アクセシビリティ確保を怠らない）。

## 実践ポイント
- 即実行できる簡単な「ノックアウト」問題を採用フローの最初に置く（30秒以内で解ける）。
- 問題は言語非依存か、応募ポジションに合わせて選ぶ（アルゴリズムより読み取り力を問うものが有効）。
- 隠しトリックを使うならアクセシビリティと公開経路（RSS等）での見え方を必ず確認する。スクリーンリーダーに配慮するか、別途アクセシブルな選択肢を用意する。
- 自動化：応募フォームで正解チェックを自動化して早期に合否振り分けする。ただし最終判断は面接で行う。
- モニタリング：フィルタの通過率・誤判定率を定期的に分析し、偏りや差別的影響がないかチェックする。
- 候補者の再申請やフォローアップも検討する（間違いから学んで良いハイヤーにつながることもある）。

短く言えば、この手法は「小さな工夫で採用効率を大幅に上げる」実用的なトリックだが、万能ではない。日本の採用文化や法令、アクセシビリティ配慮を守りつつ、一次スクリーニングの一要素として取り入れるのが現実的です。
