# Dynamic Programming - 動的計画法
めざせ最短解！初めてでもわかる「動的計画法」でアルゴリズム力アップ

## 要約
動的計画法（Dynamic Programming, DP）は、重複する部分問題と最適部分構造を利用して問題を高速化する手法。再帰＋メモ化かボトムアップで、指数時間を多項式時間に変えることが多い。

## この記事を読むべき理由
競技プログラミング、就職面接、業務システムの最適化など、日本のエンジニア領域で頻出のアルゴリズム手法です。理解すると実務でのルート最適化、在庫配分、コスト削減などに直接役立ちます。

## 詳細解説
- 基本アイデア  
  DP の核は「同じ計算を繰り返さない」こと。問題を小さな状態（state）に分解し、各状態の最適解を保存して再利用します。必要な性質は主に二つ：重複する部分問題（overlapping subproblems）と最適部分構造（optimal substructure）。

- 状態と遷移  
  問題を解くには「状態の定義」と「遷移（再帰式）」を決めることが第一歩。たとえば Fibonacci は
  $$
  F(n)=F(n-1)+F(n-2)
  $$
  という遷移と初期条件 $F(0),F(1)$ を定義すれば良い。

- メモ化（トップダウン）とタビュレーション（ボトムアップ）  
  - メモ化：再帰で解きながら結果を辞書に保存。実装が直感的でデバッグしやすい。  
  - ボトムアップ：小さい状態から順に配列を埋める。再帰オーバーヘッドがなく性能が安定する。  
  選択は問題サイズやスタック制約、実行速度の要件で決める。

- 計算量改善のイメージ  
  単純な再帰が指数時間になるケースを、状態数 $S$ と遷移辺数 $T$ によって $O(S\times T)$ 程度に落とせることが多い。メモリは状態数に比例するためメモリ設計が重要。

- 代表的な応用例（簡単に）  
  - 0/1 ナップサック：$$dp[i][w]=\max(dp[i-1][w],\ dp[i-1][w-weight_i]+value_i)$$  
  - 最長共通部分列(LCS)  
  - 区間DP（文字列・配列の最適化）  
  - グラフ上の最短経路（ただしベルマンフォードやダイクストラと用途が異なる）

## 日本市場との関連性
- 物流・配送最適化や製造ラインのスケジューリングは日本企業が重視する課題で、DPはコスト最小化や制約付き最適化で強力な武器になります。  
- AtCoder や企業コーディング面接（外資系・日系問わず）では基本問題として頻繁に出題されるため、転職や採用にも直結します。

## 実践ポイント
- 1) 状態を明確に定義する：何をキーに保存するか（例：インデックス、残り容量、フラグ）を決める。  
- 2) 再帰式を書いてみる：紙に遷移を書き、境界条件を明示する。  
- 3) 小さな例で手で計算してテーブルを埋める：バグ発見が早くなる。  
- 4) メモ化→ボトムアップへ順に実装して比較：速度・メモリで利点が変わる。  
- 5) 実務では状態削減（圧縮）を検討：1次元配列にすることでメモリを削減できることが多い。

簡単なメモ化の例（Python）：
```python
python
from functools import lru_cache

@lru_cache(None)
def fib(n):
    if n < 2:
        return n
    return fib(n-1) + fib(n-2)

print(fib(30))  # 高速に計算される
```

始めは Fibonacci やナップサックのような基本問題で手を動かし、次に LCS や区間DP、状態空間が大きい問題で状態設計の工夫を学んでください。実務では「正しくて速い」実装が価値を生みます。
