# We recreated Steve Jobs's 1975 Atari horoscope program - スティーブ・ジョブズの1975年アタリ用ホロスコーププログラムを再現
ジョブズの“はじめての占い”を手元で動かす — 1975年アタリ再現プロジェクトの舞台裏

## 要約
Adafruitが、スティーブ・ジョブズが関わったとされる1975年のアタリ向けホロスコープ・プログラムを再現し、現代のエミュレータやブラウザ上で動かせる形にしたという報告です。コンピュータ史の「コード考古学」としての価値と、当時のハード制約から学べる点が魅力です。

## この記事を読むべき理由
この種の再現は単なるノスタルジーではなく、限られたリソースで動くプログラム設計や低レイヤーの実装技法を学べる教材になります。日本のレトロコンピューティング／ものづくりコミュニティや組み込み開発者にとって、古いハード制約下での最適化や可読性の低いコードの復元は実務的な示唆を与えます。

## 詳細解説
元記事（Adafruit）は、1970年代のアタリ環境向けに作られたとされるホロスコープ・プログラムを収集・解析し、動作する形に復元したと伝えています。こうしたプロジェクトで典型的に行われる工程は次の通りです。

- ソース／スクリーンショットの収集：雑誌掲載や写真、個人コレクションなどからプログラム断片を集める。ネット上の断片情報や当時の記録が断片化している点が難所です。
- 文字起こしと整形：写真や印刷物からOCRや手入力で行を起こし、欠落や判読不能な箇所を推測して補完します。
- 言語・プラットフォームの推定：1970年代のアタリ周辺ではBASICやアセンブリ、専用のファームが用いられていました。元コードがどの言語かを見極め、現代のアセンブラやインタプリタに変換します。
- 動作検証とエミュレーション：当時のROMサイズやメモリ制約（数キロバイト単位）を考慮しつつ、StellaやAltirra、あるいはブラウザ上のWASMベースエミュレータで動作を再現します。入出力は粗いテキストや簡易的なグラフィック、ジョイスティック入力などに制約されます。
- ドキュメント化と公開：復元したソースや実行方法をGitHubやブログで共有し、再現性と保存を確保します。

技術的ポイントとして注目すべきは「極小メモリでの文字列処理」「疑似乱数や簡易アルゴリズムによる占いロジック」「ユーザー入力の扱い（生年月日→星座変換など）」です。これらは現代の言語で再実装しても、設計上の考え方やトレードオフの理解に役立ちます。

（注）元記事のページではアクセス保護（CAPTCHAなど）が発生している旨の警告が出ているため、現地の表示や添付リポジトリを確認する際はその点に注意してください。

## 実践ポイント
すぐに試せる行動リスト：

- Adafruitの記事と付随するGitHubリンクを確認し、復元ソースやROMが公開されているか探す（公開されていればクローンして動かせます）。
- エミュレータを用意する（例：Stella、Altirra、あるいはブラウザベースのWASMエミュレータ）。公開されているバイナリをロードして実行してみる。
- 復元プロセスを学びたいなら、断片的なプログラムをOCR→手修正→エミュレータで動作確認、という流れを自分で試すと良い練習になります。
- この題材を教育に使う：組み込みや低メモリ環境の最適化演習、コード考古学のワークショップに最適です。
- ひとつの練習例として、簡単なホロスコープ選択プログラムをBASICで書いてみる（以下はその骨子）。

```basic
10 INPUT "生まれた月(1-12):",M
20 IF M=1 THEN Z="山羊座"
30 IF M=2 THEN Z="水瓶座"
40 GOSUB 100
50 PRINT "あなたの星座は";Z
60 PRINT "今日の運勢: ";RND(1)
70 END
100 REM 星座処理（簡易）
110 RETURN
```

こうしたプロジェクトは単に「歴史を追う」だけでなく、低レベル実装や設計の良い教材になります。興味があれば元記事を起点にソースの有無を確認し、自分の手で再現／移植してみてください。
