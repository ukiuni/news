# Iran Protest Map - イラン抗議デモ地図
イラン抗議デモを可視化する「リアルタイム地図」で、現地の動きを技術視点から追う

## 要約
GitHub Pagesで公開されている「Iran Protest Map」は、抗議活動の発生地点や時系列を地図上で示す可視化プロジェクトです。地理情報の収集・表現・検証という観点で学びが多い事例です。

## この記事を読むべき理由
地図での可視化は、散発する情報を体系化して理解を助けます。日本のエンジニアやデータ可視化に興味がある人にとって、データパイプライン、マッピング技術、検証・倫理の実践が学べる良い教材になります。URLが GitHub Pages（pouyaii.github.io）で公開されている点も、プロジェクトの再現や技術学習に好都合です。

## 詳細解説
- 公開形式と配布: 提示された URL から、このプロジェクトは GitHub Pages 上の静的サイトとしてホストされていると推測できます。静的サイトはデプロイが容易で、地図ライブラリと組み合わせて高速に配信できます。
- 典型的な技術要素:
  - データ形式: GeoJSON や CSV を使い、位置情報（緯度経度）、タイムスタンプ、ソースメタデータを持たせるのが一般的です。
  - 地図表示: Leaflet や Mapbox GL JS がよく使われます。大量のポイントはクラスタリングや WebGL レンダリングで対処します。
  - 時系列表示: タイムスライダーやアニメーションで発生順を追えるようにします。D3 や内蔵の時間制御コンポーネントが使われます。
  - データ取得: SNS、ニュース、ユーザー投稿など複数ソースを組み合わせるケースが多く、API 経由の収集や手動キュレーションが混在します。
- 検証と信頼性: 地理的・時間的に誤った報告が混入しやすいため、クロスチェック（複数ソースの一致）、メディアの裏取り、メタデータ記録が重要です。
- セキュリティと倫理: 個人を特定できる情報や位置情報の生データを公開すると危険を招くため、匿名化・集約（ポイントをグリッドに落とすなど）や公開ポリシーの整備が不可欠です。

## 実践ポイント
- サイト確認: pouyaii.github.io ドメインは GitHub Pages。ソースが公開されていればリポジトリをクローンして構成を読むだけで学べます。
- ミニマムで試す: Leaflet + GeoJSON でまずはサンプル点を地図に表示し、タイムスタンプで表示を切り替える機能を増やすと学習効率が高いです。
- データ倫理: 公開前に個人情報を除去し、誤報の可能性を明記する運用ルールを作ること。報道や市民活動で使う場合は法務面の確認も行ってください。
- 日本での応用: 災害時の被害分布マップや地域イベントの可視化など、同じ技術を安全・合法に応用できます。自治体やNPO向けプロトタイプ作成に役立ちます。
