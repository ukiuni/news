# Release age v1.3.0: post-quantum (and more) - age v1.3.0リリース：耐量子計算機暗号など

## 要約
age v1.3.0はネイティブなポスト量子（post-quantum）受信者を導入し、I/O APIの改善やファイルメタ情報表示ツールなど実用的な強化を含むマイナーだけど重要なアップデートです。

## この記事を読むべき理由
量子耐性暗号は長期秘匿性が重要なデータ保存や政府・金融分野での運用に直結します。日本のエンジニアやセキュリティ担当者は、移行計画や互換性検証を早期に始める必要があります。

## 詳細解説
- ポスト量子受信者  
  ageはHPKEベースのハイブリッド受信者を追加。新しい受信者は古典的曲線とポスト量子KEMを組み合わせたハイブリッド設計（例: ML-KEM-768 KEM を組み合わせたもの）で、受信者IDは age1pq1...、秘密鍵識別子は AGE-SECRET-KEY-PQ-1... のような形式になります。これにより将来の量子攻撃に対する耐性を高めつつ、既存エコシステムとの相互運用性も狙っています。
  - 鍵生成は簡単に:
```bash
# bash
age-keygen -pq
```
  - 実装/spec: C2SP による仕様と、CCTV のテストベクタが公開されており、独自実装の相互検証が可能です。
  - 古いクライアント向けにプラグイン提供があり、既存ワークフローを壊さず段階的導入ができます。

- I/O APIの改善  
  - DecryptReaderAt: シーク可能なストリームの復号を可能に（zip.NewReader 等と組み合わせて大きなアーカイブに直接対応）。
  - EncryptReader: io.Readerをラップする「プル」型の暗号化API。既存のio.Writerラップ型と使い分け可能で、ストリーム処理やパイプ処理が便利になります。

- age-inspectツール  
  ファイルを復号せずにメタ情報（暗号方式、サイズ見積もり、ポスト量子使用の有無など）を表示。機械可読のJSON出力オプションもあり、CIや監査ツールと連携しやすいです。

- ハードウェアプラグイン互換の受信者  
  age1tag1... と age1tagpq1... といった受信者をネイティブサポートし、ハードウェア鍵管理プラグイン（HSMやスマートカード等）との連携が容易になりました。

## 実践ポイント
- まずは検証環境で v1.3.0 を試す（鍵生成・暗号化・復号の互換性を確認）。プラグインを使えば既存クライアントとも段階的に運用可能。
- 長期保存データ（研究データ、法務書類、重要なログ等）はハイブリッド暗号への移行を検討。古典＋PQのハイブリッドは当面の現実的選択。
- 大きなアーカイブ処理やストリーム処理を行う場合は DecryptReaderAt / EncryptReader を使い、I/Oパフォーマンスを再評価する。
- age-inspect をCIに組み込み、暗号化ファイルのメタ監査や自動分類に使う（--json 出力を利用）。
- 実装検証には公開された仕様とテストベクタを活用し、互換性テストを自動化すること。
