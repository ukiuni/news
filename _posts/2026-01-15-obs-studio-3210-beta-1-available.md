# OBS Studio 32.1.0 Beta 1 available - OBS Studio 32.1.0 ベータ1公開
魅力的なライブ配信・録画がもっと簡単に。新しいオーディオミキサーや WebRTC Simulcast 対応など、配信ワークフローを一段と洗練するアップデート

## 要約
OBS Studio 32.1.0 Beta 1 が公開され、UI改善（新しい Add Source ダイアログや編集トランスフォーム改良）、新オーディオミキサー、WebRTC の Simulcast サポート、複数のバグ修正やパフォーマンス改善が含まれます。ベータ版のため本番運用前にテスト推奨です。

## この記事を読むべき理由
OBS は日本でもゲーム配信、配信イベント、オンライン授業、VTuber 活動など幅広く使われています。本アップデートは配信品質や操作性に直結する改良を複数含むため、配信者・配信エンジニアは早めに機能を把握してワークフローへ取り入れる価値があります。

## 詳細解説
- 新しいオーディオミキサー
  - ミキサーの UI と機能が強化され、Light テーマでもパレットが有効になりました。音声調整が視覚的に把握しやすくなり、音量管理やトラック分離がしやすくなります。
- Add Source ダイアログの刷新
  - ソース追加の操作フローが改善され、初心者でも迷いにくい設計に。既存ソース名から "Source" が削除されるなどラベルの整理も行われています。
- WebRTC Simulcast サポート
  - ブラウザ向けの低遅延配信や視聴者ごとに異なる画質を自動配信する Simulcast に対応。低帯域の視聴者へ低解像度を同時配信することで視聴体験を向上できます。
- シーンアイテムの undo/redo とコピー動作改善
  - シーン内のアイテム編集の取り消し・やり直しが改善。シーンアイテムをコピーするとすべてのプロパティが複製されるようになり、レイアウト作業が効率化します。
- グラフィック/映像処理の改善
  - スケールフィルタ、ブレンドモード、デインタレース設定などが追加／改善され、特にマルチエンコーダ構成時のスケーリング精度や NVIDIA のブラー処理の帯域落ち問題など複数のバグが修正されています。
- UI と挙動の変更
  - ドックアニメーションの無効化、デフォルトドック配置の再編、トランジションプレビューのボタン移動などで操作感の安定化を図っています。
- デフォルトビットレートの変更
  - デフォルトのビットレートが 6000 kbps に変更。配信プラットフォームによって上限があるため、配信先の推奨値は要確認です。
- 不具合修正
  - プロジェクター解像度の小数点表示やプレビュー/プログラムの同期、HEVC の SEI、ファイル分割時のチャプタタイミングなど、多数の実用的なバグが修正されています。

配布は各プラットフォーム（Windows x64/arm64、macOS Intel/Apple Silicon、Linux ディストリ）向けに用意され、チェックサムも公開されています。ベータリリースのため、安定版と併用するか、事前にシーンコレクション／プロファイルのバックアップを推奨します。

## 実践ポイント
- ベータ導入前に必ずプロファイルとシーンコレクションをエクスポートしてバックアップする。
- WebRTC Simulcast を試してみる：ブラウザ視聴を想定した低帯域対応を検証し、視聴者ごとの画質配分を確認する。
- デフォルトビットレート（6000 kbps）はそのまま使わず、Twitch/YouTube/配信プラットフォームの推奨値に合わせて設定する。
- 新しいオーディオミキサーを触ってみる：トラック分割や視覚的なメーターでミックスの安定化を図る。
- macOS や Linux（PipeWire）環境でのキャプチャ挙動を確認：修正項目がプラットフォーム固有の問題を解決しているかテストする。
- ダウンロード後はチェックサムでファイル整合性を確認する。
- 問題が見つかったら GitHub の issue に報告するか、コミュニティで情報を共有する（ベータ検証が改善を早めます）。

短期間で使い勝手と品質面の改善が多数入っているので、配信ワークフローを改善したい方はテスト導入を強くおすすめします。
