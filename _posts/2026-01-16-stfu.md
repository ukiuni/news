# STFU - 黙ってくれ
旅先の「うるさい奴」を黙らせる？AIが一発で生成した“2秒遅延リピート”Webアプリの話

## 要約
空港で隣の人が大声でリールを見ている――そんな状況を解消するために作られた、マイク入力を受けて約2秒遅らせてその音声をスピーカーに返すシンプルなWebアプリ。作者はAI（Claude）にワンプロンプトで動く実装を出してもらい、Web Audio APIで動くプロトタイプを公開している（MITライセンス）。

## この記事を読むべき理由
- 公共のマナー問題が日常的に話題になる日本で、テクノロジーで“場の騒音”にどう対処できるかは関心が高い。  
- Webブラウザだけでマイク入力を処理して即座に音を返す仕組みは、音響プログラミング入門として学びやすい実例になる。  
- AIが実装支援に使われた点も、実践的なツール作りの新潮流を示している。

## 詳細解説
このプロジェクトの核は「短い遅延を付けて音声をそのまま返す」ことによる心理効果。人は自身の声や笑い声がわずかに遅れて返ってくると違和感を覚え、声を抑える傾向がある（いわゆる聴覚フィードバックの干渉）。作者はこれを利用して、周囲の大音量を抑えさせることに成功したと報告しています。

技術的にはブラウザの標準APIのみで実現されています。主に使われる要素は次の通りです。
- navigator.mediaDevices.getUserMedia: マイク入力を取得する（ユーザー許可が必要）。
- AudioContext / MediaStreamSource: マイク音声をWeb Audioのノードに流し込む。
- DelayNode: 指定した時間（ここでは約2秒）だけ音声を遅延させる。
- GainNode: ボリューム調整やフィードバック制御に利用。

簡潔な実装例（概念）：
```javascript
const ac = new AudioContext();
const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
const src = ac.createMediaStreamSource(stream);
const delay = ac.createDelay();
delay.delayTime.value = 2.0; // 約2秒
const gain = ac.createGain();
gain.gain.value = 1.0;
src.connect(delay);
delay.connect(gain);
gain.connect(ac.destination);
```

公開リポジトリはGitHubにあり、READMEには「Claudeにワンプロンプトで動くコードを書いてもらった」との一文があり、AIによるコーディング補助の実例としても興味深いです。ライセンスはMITなので、自由にフォークして改変可能です。

注意点（倫理・法的側面）
- 人の行動を意図的に変えようとするツールは、ハラスメントや迷惑行為に繋がる恐れがある。公共の場での使用は慎重に。  
- マイク・スピーカーを通すため、プライバシーや周囲への影響を常に考慮する必要がある。  
- 音を返すことで予期せぬエコーやハウリング（音のループ）が起きる場合があるため、ボリューム管理やフィルタが必要。

## 実践ポイント
- 試す場所は自宅や個室で。公共の場での無断使用は控える。  
- 実装を学びたい場合は、まずローカルで上のサンプルコードを動かしてみる。AudioContextの開始はユーザー操作（クリックなど）で行うと許可フローがスムーズ。  
- 遅延時間は0.5〜3秒あたりで挙動が変わるので、自分で試して最適値を探す。  
- ボリューム制御や簡単なハイパス/ローパスフィルタを入れると、ハウリング防止と周囲への干渉軽減に効果的。  
- リポジトリ（https://github.com/Pankajtanwarbanna/stfu）を参照すれば、実装とライセンスを確認できる。商用利用や改変はMITライセンスの範囲内で可能。

短く言えば、STFUは「シンプルで学びやすい音響トリック」として面白く、Web Audio APIやAI支援コーディングに触れる良い入り口になる。ただし公共での利用は倫理的・法的配慮を忘れずに。
