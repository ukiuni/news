# enclose.horse - 馬を囲め（仮）
魅力的なワンカラム小品から学ぶ「ミニマルUI × ゲーム設計」の教訓

## 要約
ほんの数文字とスコア表示だけで成り立つミニマルなウェブ体験は、短時間の没入と拡散を生む。表示されている「Walls: 0 / 10 Score: 0 Best: 0」だけでも、設計上の重要な意図が読み取れる。

## この記事を読むべき理由
短時間でユーザーを惹きつけ、スマホで楽しめる小さな体験は日本のプロダクト開発でも有効。ミニマルなゲーム／インタラクションを作ってバイラル性やUX改善に活かしたいエンジニア／プロダクト担当は、本質的な設計指針を得られる。

## 詳細解説
- 表示要素から読み取れる設計（想定）
  - Walls: 0 / 10 — 制約（壁は最大10本）。制約の提示は学習コストを下げ、戦略性を生む。
  - Score / Best — セッションスコアとローカルのベストを比較する簡潔なフィードバックループ。結果はlocalStorageで管理するだけで充分なことが多い。
- インタラクション設計
  - ワンクリック／ドラッグで壁を置くような直感的操作が想定される。入力はタッチ優先にし、クリックも対応する形がベスト。
  - フィードバックは即時に（スコア更新、アニメーション、音）返すことで短時間のリテンションを高める。
- 実装候補と技術的注意点
  - 描画: HTML5 Canvas または SVG。シンプルな衝突判定やライン配置ならSVGで十分で保守性が高い。
  - ゲームループ: requestAnimationFrame を使った軽量ループ。タイミングはブラウザ主導で管理。
  - 状態管理: 小規模なら単一のJSオブジェクト＋localStorageで完結。複雑化するなら小さなステートマシンを導入。
  - 衝突判定/物理: 単純な距離計算や境界判定でOK。重い物理エンジンは不要。
  - 配布: 静的ホスティング（Netlify, Vercel, GitHub Pages）で十分。ドメインの語呂（enclose.horse のような語呂・ドメインハック）は拡散に有利。
- UX / 成長のポイント
  - セッションを短く設計（30秒〜2分）してリプレイを促す。
  - ベストスコアの共有ボタン・スクリーンショット誘導で拡散を狙う。
  - アクセシビリティ: キーボード操作や色覚配慮を最低限実装。

## 実践ポイント
- まずは「最小実装」を24時間で作る：SVG＋Vanilla JS、localStorageでスコア管理、レスポンシブ対応。
- 壁本数やスコア計算はパラメータ化してA/Bテストする（ゲーム性の最適化）。
- 軽量な分析（イベントのみ送る）を入れて離脱ポイントを観測する。
- 日本の流入経路を意識：Twitter・LINEでの共有UX、短尺動画（リプレイGIF）対応を優先。
- ドメインやUIの“覚えやすさ”を大事にする。短くユニークなURLは拡散効果が高い。

短く洗練されたインタラクションは、小さな工数で大きな波及を生む。enclose.horseに見える最小限のUIから、実践的なミニゲーム設計のヒントを取り入れてみてください。
