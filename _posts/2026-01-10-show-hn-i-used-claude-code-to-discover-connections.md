# Show HN: I used Claude Code to discover connections between 100 books - Claude Codeで100冊の本のつながりを発見した話
100冊がつながる「知の地図」──AIが暴く意外なテーマと発想の交差点

## 要約
Claude Code（AI）を使って100冊の本の間にあるテーマ的なつながりを自動発見し、短いラベルとキーワードで「道筋（Trails）」として可視化したプロジェクトです。表層を越えて、模倣・直感・制度・技術史などの共通モチーフが浮かび上がります。

## この記事を読むべき理由
- 大量の読書メモや文献が「断片」になっている人にとって、知識の再発見とアイデア発想に直結する手法だからです。  
- 日本のものづくりや組織論（トヨタ式や職人文化、技術移転）の文脈とも強く結びつき、製品開発や研究のヒントになります。

## 詳細解説
Trailsは、各書籍間の「テーマ的リンク」を自動で見つけ出し、短い説明と関連キーワードで接続を示します。抜粋の例を見ると、「Self-deception（自己欺瞞）」「Tacit knowledge（暗黙知）」「Containerization（コンテナ化）」「Innovation spillover（イノベーションの外部効果）」など、幅広い概念が織り交ざっており、単なるジャンル分類より深い「思考の接点」を提示しています。

仕組み（公開されている情報から読み取れる一般的な流れ）は次の通りです（実装細部はプロジェクトにより異なります）。
- 文献（要約・抜粋・目次など）をテキスト化する。  
- 各テキストの意味的な表現を得る（LLMのプロンプトで関係性を抽出する、あるいは埋め込みベクトルで類似度を計算する等）。  
- ペアごとに関連度を算出し、高いものをグラフとしてつなぐ。  
- LLMで各エッジに短いラベルやキーワードを生成し、人が読める形に整える。  
- 最後に可視化して「道筋」を辿れるようにする。

抜粋に見えるテーマ群からは、以下のような洞察が得られます。  
- コピーと模倣はイノベーションの一側面（Innovation spillover / Copy Machine）。  
- 暗黙知や職人性がスケール化と衝突する（Illegibility Premium / Tacit knowledge）。  
- システム設計では「最弱点」が全体を規定する（Pacemaker Principle / Weakest link）。  
- 指標最適化は逆効果になり得る（Proxy Trap / Perverse instantiation）。  

こうした「跨領域の共通テーマ」は、人が別々に読んだときには見落としがちな発想の接点を作り、アイデア創発や戦略立案に資します。

## 実践ポイント
- まず試す：Trailsのような可視化を自分の読書リスト（技術書・論文・ビジネス本）で試してみると、新しい組み合わせアイデアが見つかります。  
- 自作パイプライン（シンプル版）：書評や要約を集め、Sentence-BERTなどの文ベクトルで類似度を計算、閾値でエッジを作り、Gephiやd3で可視化。エッジラベルはLLMで要約する。  
- チーム活用：プロダクト要件や過去の設計文書を同様にマッピングすると、ナレッジの断片化を防ぎ、オンボーディング資料にも使えます。  
- 注意点：AIが示す関連は「示唆」であり因果ではありません。誤った類推やノイズ（見かけ上の類似）を検証する工程を必ず入れてください。  
- 日本の現場での応用例：技術移転の履歴、職人の暗黙知の可視化、サプライチェーンの脆弱性パターン抽出など、実務課題に直結するテーマで試すと効果が出やすいです。

Trailsは単なるギミックではなく、読書や調査を「ネットワーク化」して思考の接点を可視化する強力なアイデアツールです。あなたの本棚や社内ナレッジをつなげることで、意外な発明のヒントが出てくるかもしれません。
