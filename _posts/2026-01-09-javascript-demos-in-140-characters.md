# JavaScript Demos in 140 Characters - 140文字で描くJavaScriptデモ
140文字の魔法：ブラウザだけで作る一行アートとアニメーション

## 要約
ブラウザ上のキャンバスに、たった140文字で描かれた驚きのビジュアル群――Dwitterは短く尖ったJavaScript表現の宝庫で、トリックと最適化の勉強にもってこいです。

## この記事を読むべき理由
短い制約はアイデアの本質を鍛えます。日本のハッカソン、クリエイティブコーディング、ポートフォリオ制作に直結するテクニックが詰まっており、初心者がJavaScriptの挙動（描画ループ、三角関数、ビット演算など）を実践的に学べます。

## 詳細解説
- プラットフォームの前提  
  Dwitterでは「u(t)」という関数が1秒あたり60回呼ばれ、tは経過秒数です。グローバルに用意された変数（c: 1920×1080のcanvas、x: 2Dコンテキスト）と短縮名（S= Math.sin、C= Math.cos、T= Math.tan、R= rgba生成）が使えます。これらが制約下での省略表記と表現力の源です。

- よく使われるテクニック
  - キャンバスのクリアに c.width = 1920 や c.width |= 0 を使う（幅を再設定すると自動的にクリアされるブラウザ挙動を利用）。  
  - ループとインライン代入を多用し、変数を再利用して文字数を節約。  
  - 三角関数（sin/cos）で周期的な動きを作る。$S(t)$ や $C(i)$ を使った同心円や波の表現が鉄板。  
  - ビット演算（|0, ^ など）で整数化や不思議なノイズを生成。  
  - x.fillRect をドット代わりに大量描画してピクセルアートを作る、または x.fillText でASCIIアートを描く。  
  - eval/unescape/escape の組み合わせで圧縮・難読化したペイロードを展開する裏ワザ（ただしセキュリティリスクが高いため公開サービス以外では避けるべき）。

- 表現と制約の相互作用  
  140文字という短さは「何を残し、何を捨てるか」を鋭く問います。結果として数学的パターン、ビジュアル最適化、ブラウザAPIの“クセ”を学べる場になります。コードが短いほどアイデア自体の魅力が浮き彫りになるため、プレゼンやポートフォリオでも映えます。

- セキュリティと互換性注意  
  evalやエンコード系のトリックは便利ですが、外部評価やXSSに弱く、モバイルや低スペック環境では描画負荷が問題になる場合があります。実運用や共有前にはリファクタリングと安全性チェックを。

## 実践ポイント
- 小さく始める：まずはキャンバスのクリアと1つの fillRect を動かすところから。下例は140文字以内を意識した単純な波の一例です。

```javascript
function u(t){for(c.width=1920,i=100;i--;)x.fillRect(i*12+S(i+t)*20,540+i,4,4)}
```

- 書き方のコツ
  - Math.sin → S と短縮（dwitterでは事前定義されることが多い）  
  - ループでインクリメント/デクリメントを巧妙に使う（for(i=100;i--;) など）  
  - c.width を利用して高速クリア、fillRect を使ってまとめて描く

- ツールとワークフロー
  - まずブラウザのDevToolsで試す（Consoleに貼って動作確認）  
  - 文字数は常に確認。短縮や変数再利用で節約する。  
  - 公開前にevalや外部文字列展開を消すか説明を付ける（セキュリティ配慮）

- 日本での活用例
  - ハッカソンの即席デモ、ポートフォリオの「ショートコード」コーナー、JS勉強会での演習素材として最適。JSConf JPや国内コミュニティで話題にしやすいテーマです。
