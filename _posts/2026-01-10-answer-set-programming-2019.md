# Answer Set Programming (2019) - アンサーセットプログラミング（解説・入門）
驚くほど直感的に「答え」を書ける論理プログラミング ― ASPで複雑な制約や推論をシンプルに表現する方法

## 要約
Answer Set Programming (ASP) は、知識表現と論理的推論のための宣言型プログラミング手法で、制約付き探索や計画、構成問題に対して高い表現力と競争力のある性能を発揮します。

## この記事を読むべき理由
日本の製造・物流・組込システムや研究開発の現場では、「制約を満たす組合せ探索」や「複雑なルールに基づく意思決定」が頻繁に発生します。ASPはこうした課題をモデリングし、短期間で実用的なソリューションを試作できるため、エンジニアや研究者が知っておく価値があります。

## 詳細解説
- 基本アイデア  
  ASPは「どう動かすか」ではなく「何が正しい答えか」を記述する宣言型手法です。プログラムは事実とルールで構成され、実行系（ソルバ）が可能な解集合（answer sets、解）を列挙します。解集合は「安定モデル意味論（stable model semantics）」に基づき定義され、直感的には与えたルールを矛盾なく満たす解の集合です。簡潔に言えば、ある集合 $S$ がプログラム $P$ の解集合であるとは、$S$ がプログラムの帰結を安定的に満たす最小モデルである、という条件に対応します（形式的には $S$ が $P^S$ の最小モデルである）。

- 言語と実装  
  代表的な実装に Potassco の clingo（グラインゴ）や DLV などがあります。clingo はグラウンダー（gringo）とソルバ（clasp）を統合しており、コマンド一発で *.lp ファイルから解を得られます。ASPは SATソルバやILPとも関連しますが、規則・選択ルール・制約を直接表現できる点が特徴です。

- 典型的な構文要素（概念）  
  - 事実（fact）: a.  
  - ルール（rule）: head :- body.  
  - 選択ルール（choice rule）: {color(X,C)} :- node(X), color(C).  
  - 制約（constraint）: :- color(X,C1), color(X,C2), C1 != C2.  
  これらにより、組合せの生成とそれに対する制約の適用を分かりやすく記述できます。

- 適用例  
  - スケジューリング（製造ライン、テストスケジュール）  
  - 構成問題（機器/車両の構成最適化）  
  - 計画（ロボット行動計画、タスク配分）  
  - 知識ベース推論・診断（故障診断やポリシールールの検証）  
  産業分野での強みは「ルール変更に強い」「複雑な制約を素直に書ける」点です。

## 実践ポイント
- まずは手を動かす（短い例）  
  次はグラフ彩色問題の簡単な例（clingo 互換の記法）です。ファイル名 example.lp として保存し、ターミナルで `clingo example.lp` を実行してください。

  ```asp
  % ノードと色の定義
  node(a). node(b). node(c).
  color(red; green; blue).

  % 各ノードはちょうど1色を持つ
  { assign(N,C) : color(C) } = 1 :- node(N).

  % 隣接ノードは同じ色にできない（辺の例）
  edge(a,b). edge(b,c). edge(a,c).
  :- assign(X,C), assign(Y,C), edge(X,Y), X != Y.
  ```

- 実行環境（VS Codeでの試行）  
  1. clingo をインストール（公式 Potassco ページかパッケージマネージャで入手）。  
  2. VS Code のターミナルで `clingo example.lp` を実行。出力に解集合が出ます。  
  3. 大きなモデルや複雑な制約はオプション（最適化、複数解列挙）で制御可能（例: `--opt-mode` や `-n 0` で全解列挙）。

- モデリングのコツ  
  - 問題を「生成（generate）」＋「検査（test）」で分けると設計しやすい。  
  - 過度に否定や相互依存を使うと意味が捉えにくくなるため、局所的に検証を繰り返す。  
  - 最適化問題は #minimize や #maximize を使う（clingo の最適化機能を利用）。

- 実務への組み込み（日本市場での期待値）  
  - 中小製造業のライン最適化や工程スケジューラのプロトタイプ化に適する。  
  - ルールベースのポリシー管理（社内規程や法令チェック）で、変更に強い実装を低コストで作れる。  
  - Python から clingo API を呼べるため、既存システム（ERPやIoTプラットフォーム）との連携が容易。

- 学習・参考リソース  
  - Potassco/clingo リポジトリとドキュメント（チュートリアルが豊富）  
  - ASP 入門記事・講義ノート、Handbook of Answer Set Programming（論文集）  

まとめると、ASPは「複雑なルールと制約を直感的に表現して高速に解を得たい」場面で強力な選択肢です。まずは小さな問題をモデル化してclingoで試し、現場の制約に合わせて拡張していくと実用化までの道筋が見えてきます。
