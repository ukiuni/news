# x86CSS - An x86 CPU emulator written in CSS - CSSだけで作られた x86 エミュレータ

魅惑のタイトル: CSSで「本物の」x86が動く──ブラウザのスタイルだけで動かす8086エミュレータ

## 要約
Lyra Rebaneのx86CSSは、HTML/JSを使わず「純粋にCSSだけ」で8086系のx86命令を実行するデモ。Cで書いたプログラムを8086マシンコードにして、CSS表現に変換して動かしています。

## この記事を読むべき理由
- CSSの限界と表現力を体感でき、フロントエンドの“言語観”が変わる。  
- 教育・デモ・ポートフォリオ用途で「意外性のある技術」を示せる。  
- 日本でもブラウザベースの軽い教育ツールやアート作品に応用可能。

## 詳細解説
- 何が動いているか：作者はCで書いたプログラムをgcc（8086ターゲット）でコンパイルし、生成された8086マシンコードをCSSルール群に変換。ブラウザのスタイル評価（アニメーション／コンテナクエリ／擬似要素など）を利用して命令実行と状態遷移を表現しています。  
- 時計（クロック）の仕組み：CSSアニメーション＋style container queriesを使う実装と、ホバーで駆動する実装（高速・安定だが操作が必要）の2種類があり、ページはJS不要で動作するよう工夫されています。サイト上には補助的なスクリプトがあるが必須ではありません。  
- 入出力とメモリ：デフォルトで0x600バイト（1.5KB）程度のメモリ、プログラムは0x100にロード。I/Oは特定アドレス（例: 0x2000〜）にマッピングされ、画面出力や入力取得はその領域経由で行います。  
- 実装の範囲と制約：x86命令セットの大半を実装していますが、全命令やフラグの挙動すべてを完全再現しているわけではありません（CF/OF未設定などの差分あり）。パフォーマンスは実用的ではなく「技術的デモ／アート」が主目的です。ブラウザの対応は現状Chromium系が中心。  
- ビルドフロー：自分のプログラムを動かすにはリポジトリをクローンし、8086バイナリ（program.bin）と開始アドレス(program.start)を用意して、付属のbuild_css.pyでCSS出力（x86css.html）を生成します。C言語からビルドする場合はgcc-ia16（または対応環境）を使い、build_c.pyが補助します。Linux/WSLでの動作が確認されています。

## 実践ポイント
- まずChromium系ブラウザでデモページを試してみる（動作環境の確認）。  
- リポジトリをクローンして、付属のbuildスクリプト（build_css.py / build_c.py）を触ると仕組みが理解しやすい。  
- C→8086にコンパイルするには gcc-ia16 が必要（PPAやソースビルドを利用）。  
- メモリサイズやI/Oはbuild_css.pyで調整可能。簡単な表示／入力プログラムを作って動かすと学びが大きい。  
- 応用例：フロントエンド勉強会の教材、ブラウザでの低レイヤーデモ、技術ポートフォリオやハッカソンの展示ネタに最適。
