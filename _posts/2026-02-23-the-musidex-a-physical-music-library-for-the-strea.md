# The Musidex: A physical music library for the streaming era - Musidex：ストリーミング時代のための物理的音楽ライブラリ
いまの音楽体験に「触れる楽しさ」を取り戻す──Rolodex＋QR/NFCで作る自分だけのアルバム索引

## 要約
ストリーミングで埋もれがちな愛聴アルバムを、Rolodexカードにアートとメタデータ、QRコード／NFCを付けて物理化したプロジェクト。カードを触ることでアルバム再生や発見ができる“アナログ＋デジタル”のコレクションだ。

## この記事を読むべき理由
日本でもストリーミング利用は拡大する一方で、昔の名盤や個人的な“好き”が見えにくくなる問題は共通。Musidexはエンジニアリングと手作りを組み合わせ、手元で再発見を促す実践例として参考になる。

## 詳細解説
- コンセプト：アルバム単位でカードを作り、QR（ストリーミングURL）とNFCで再生やキャストが即できる物理ライブラリ。装飾・ランダム配列・ジャンル分けなどのキュレーション性も重視。
- ハード面：既製のRolodex本体、カード用の厚紙、Rolodex形状用のパンチ（穴形状）、印刷（アルバムアート）とステッカー（タイトル・アーティスト・年・QR）を用意。NFCタグを側面に埋め、スマホでタップしてリビングスピーカーへ接続する運用。
- ソフト／データ面：
  - 入力元は古いiTunesエクスポートXML＋ストリーミングサービスのプレイリスト。
  - スクリプトでトラック→アルバムを参照し、アルバムメタとストリーミングのURLを表に出力。
  - 重複除去や表記統一はほぼ手作業で調整（1アーティスト1枚ルールなど）。
  - ストリーミング上に無いタイトルは割愛か別管理（ローカル配信を構築しない限り制約あり）。
  - アルバムのストリーミングURL変換にはSoundiiz等のサービスを試用（完全自動化は難しい）。
- 実例仕様：おおむね300枚（表裏で600アルバム）、一部シャイニー紙でプレイリストページ、2台作成経験あり（自分用と父親用）。2作目ではアート＋メタを白ステッカー一体化して組み立て効率化。
- 課題：iTunes XMLのバージョン差やメタデータずれ、ストリーミング非対応楽曲の存在、キュレーションの労力。将来的には形式やファクターを変えた派生（映画版、書籍版、鳥の図鑑etc.）も提案。

## 実践ポイント
- はじめの一歩
  1. iTunes／Apple Musicのライブラリをエクスポート（XML/CSV）またはストリーミングのプレイリストを用意。
  2. スプレッドシートに「アルバム名／アーティスト／年／ストリーミングURL／アートファイルパス」を作る。Soundiiz等でURL変換を試すが、手直し前提で。
  3. QRコードはqrencodeやオンラインジェネレータで一括作成。NFCはNTAGシリーズの汎用タグを購入してURLを書き込む（スマホで書込アプリ使用）。
  4. Rolodex本体を入手、カード厚紙をカット、必要ならRolodex用のパンチを用意。
  5. アートを印刷／ステッカーに出力してカードに貼る。メタは印刷済ステッカー一体化が作業効率良。
  6. カードごとにQR/NFCの動作確認（スマホ→ストリーミング再生／キャスト）。
- 運用のコツ
  - まずは「お気に入り10〜30枚」で小さく試す。完成までの学びが多い。
  - Bandcamp等、アーティスト寄りのリンクも併用すると発見価値が上がる。
  - プレゼントに向く：親や友人のライブラリを基に作ると喜ばれるが、XMLバージョン違い等の手作業が発生する点に注意。
- ツール候補
  - データ処理：Python（pandas, ElementTree）、CSVエディタ
  - URL変換：Soundiiz（試用）、手動チェック
  - QR生成：qrencode、Google Charts API
  - NFC書込：NFC Tools（Android/iOSアプリ）
  - 印刷：地元のプリントショップ＋ミニステッカー・サーマルプリンタ

短時間で作れるミニ版から、しっかりキュレーションした300枚版まで応用が効きます。手触りのあるインターフェースで「思い出のアルバム」をもう一度日常に取り戻してみてください。
