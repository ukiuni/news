# Traditional user-interface graphics: icons, cursors, buttons, borders, and drawing style - 伝統的なユーザーインターフェースのグラフィックス：アイコン、カーソル、ボタン、境界線、描画スタイル
懐かしくて実務に活きる「レトロUI流儀」──Windows95世代のボタン・アイコン設計を今に活かす

## 要約
1990〜2003年頃までに確立した「伝統的UI」の描き方（解像度とピクセル密度、ボタン／境界線の色と立体表現、アイコン／カーソルのサイズとカラーパレット、アニメーションや障害時の表現など）を整理し、現代のデザインや組み込み・低色数環境で使える実践ノウハウを解説する。

## この記事を読むべき理由
レトロUIのルールは「限られた色・ピクセルで見やすく伝える」ための秀逸な設計思想。IoT機器や低解像度ディスプレイ、ピクセルアート系UI、あるいはモダンUIのアクセントとして、実務で今すぐ活かせます。

## 詳細解説
- 表示モードとピクセル密度  
  - 解像度（例: 640×480）とピクセル密度（ppi）がUIの見え方を決める。従来は96ppiを基準にし、他は比率でスケール（例: EGA の縦縮小は 72/96 = 3/4）。高DPIは2×、3×スケールで扱う。  
- ボタン・境界線の色と立体表現  
  - 伝統的なシステム色は「button face」「window frame」「highlight」「shadow」「light highlight」「dark shadow」など6色で構成（Windows95の既定値が代表例）。  
  - 立体感は上左からの光源を仮定し、上／左を明るく、下／右を暗く描く。押下時はラベルを(1,1)ピクセル右下にずらすのが慣例。  
  - 無効状態は半透明50%かチェッカーボード（1ピクセルおき）で表現。混在状態（indeterminate）は背景をチェックや混色で示す。  
- アイコンとカーソルの作法  
  - 推奨サイズ：16×16, 24×24, 32×32, 48×48, 64×64。伝統的に32×32（VGAパレット）が最も一般的。  
  - カラーデプス：2色（白黒）→16色（VGAパレット）→256色、8bpcは2000年代以降の主流。透明ピクセル可、半透明は8bpcでのみ。  
  - 黒白カーソルは見やすさのため輪郭を白／黒で1ピクセル縁取る。カーソルも通常32×32。  
- 描画スタイルの制約と工夫  
  - 曲線・線はアンチエイリアス無しで1ピクセル幅、斜めは整数傾き／1/nの傾きで描く。  
  - 16色環境ではチェッカーパターンで色を疑似的に作る（ディザリング）。グラデは控えめか背景限定。  
  - アイコンは可能ならベクターで設計し、ラスタに書き出してサイズ別に微調整するのが効率的。  
- アニメーションと資産管理  
  - 伝統的手法はフレーム画像を個別ファイルやスプライトシートで扱う。AVIやビデオに頼らない古典的な実装が一般的。

## 実践ポイント
- アイコン制作フロー：まずベクターで作成 → 16/256/8bpc用にパレットを最適化 → 16×16〜64×64を個別に手直しして出力。  
- ボタン実装のチェックリスト：6つのシステム色を定義 → 光源は上左に固定 → 押下で(1,1)ピクセルずらす → 無効状態は50%不透明かチェッカーで表現。  
- 低色数環境対応：チェッカーボードやディザリングで疑似色を作り、重要なシルエットは黒／白輪郭で潰れを防ぐ。  
- DPI対応：基準を96ppiと見なし比率でスケール、アイコンは整数倍（1×、2×、3×）で用意する。  
- 実装チャレンジ：6色のみ＋透明で境界線／各種ボタン（通常・押下・混在・無効）とチェックボックス／ラジオ／ツールバーのホバーを描くコードを作ると理解が深まる（ディザ／マスク処理の練習に最適）。

この設計ルールは「制約内での可読性と一貫性」を重視する実務的な知見です。レトロ風UIの再現や組み込みディスプレイ対応、新しいデザインへの応用にぜひ活用してください。
