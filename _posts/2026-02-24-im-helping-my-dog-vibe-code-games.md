# I'm helping my dog vibe code games - 愛犬に「バイブ」でゲームを作らせてみた
犬がキーボードで“タイプ”してAIがゲームを自動生成する──可愛い実験から実用的な自動化パイプラインまで丸ごと公開

## 要約
犬が踏んだ/打ったキー入力をラズパイ経由で受け取り、Claude Code（LLM）に解釈させてGodotでゲームを生成・検証し、成功したら自動でおやつを出す仕組みを作った話。

## この記事を読むべき理由
AIと自動化で「アイデア→実装→検証→報酬」を閉ループ化した実例は、プロトタイプ高速化やQA自動化のヒントに直結。日本のインディー開発者やプロダクトチームにも応用できる実践的ノウハウが詰まっています。

## 詳細解説
- 全体構成：Bluetoothキーボード（Logitech Pebble Keys 2）→Raspberry Pi 5がUDPで各キーを放送→DogKeyboard（Rust製）が特殊キーをフィルタしてClaude Codeへ転送。Claudeが生成・修正したGodot 4.6のシーン（.tscn）とC#ロジックがビルドされる。生成が一定条件を満たすとスマート給餌器（Aqara C1 via Zigbee）を制御して報酬を出す。
- プロンプト設計：単なるランダム文字列を意味あるゲーム指示に変換するため、LLMに「風変わりな天才ゲームデザイナーが暗号めいた入力で指示する」という人格を与え、さらに最低要件チェック（音声、操作（WASD/矢印）、プレイヤー表示、敵/障害など）で品質を担保。
- 自動検証ツール：スクリーンショットを撮るスクリプト、入力シーケンスを再生するテスター、シーン/シェーダのリンター、入力アクションマッパー。LLMが自分でビルドを起動→操作→スクショ→不具合発見→修正→再実行、というループを回すことで「ほぼ自動で改善」できた。
- ハードウェア的配慮：犬が押す特殊キー（Esc等）でプロセスが中断しないよう先にフィルタ。給餌器は最小給餌サイズが大きい問題を回避するため個別トレイに小分けし、ZigbeeアダプタはPi側へ移設して安定稼働。
- なぜGodotか：.tscn のテキストベースでLLMが直接編集しやすく、Unity/Bevyで遭遇したエディタブリッジや座標系の問題が回避できたため。
- トレーニングと運用：犬の学習は約2週間。最終的にDogKeyboardはClaudeのアイドル時に入力が16文字以上あれば自動送信、作業中は余計な入力を消す等のガードレールを入れて安定化した。

## 実践ポイント
- テキストベースのアセット（.tscnなど）を使うとLLMが直接編集しやすい。  
- 自動QAは「スクリーンショット + 入力再生」の組合せが効果的。  
- LLMプロンプトには必須要件チェックリストを入れてから作業させる（音、操作、可視化など）。  
- 外部入力は必ずフィルタ／サンドボックス化（特殊キー、OSコマンド対策）。  
- 小さなハードウェアプロキシ（Raspberry Pi）はキーボード仲介・Zigbee管理・監視に便利。  
- 開発プロジェクトの“外部メモリ”（例：LLMの記憶ファイル）はスタイル偏りの原因になるので用途に応じてリセットを検討する。

短く言えば、可愛い実験に見えても得られるのは「LLMとツールを組み合わせた堅牢な自動生成パイプライン」の設計図です。日本の小規模開発やプロトタイピング、自動QA投入の出発点として活用できます。
