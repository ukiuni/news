# Sashiko: An agentic Linux kernel code review system - 刺し子：エージェント型Linuxカーネルコードレビューシステム
AIがカーネルの「レビューチーム」になる――自動化でバグを見つけ、レビュー負荷を下げる刺し子の実力

## 要約
SashikoはLKML（Linux Kernel Mailing List）を監視して提案パッチを多段階で自動レビューするオープンソースのシステム。サブシステム別のプロンプトとLLMを使い、Gemini 3.1 Proでの評価では過去コミットの約53.6%のバグを検出したと報告されている。

## この記事を読むべき理由
日本の組込み・家電・自動車分野の製品でもLinuxカーネルは基盤。コントリビュータや保守担当者が多忙な中、パッチ品質を保ちつつレビュー効率を上げる技術は即戦力になるため注目に値する。

## 詳細解説
- 動作概要：SashikoはLKMLを監視し、投稿されたパッチに対して複数ステージの自動レビューを実行。レビューは高レベル設計チェック、セキュリティ監査、リソース管理や並行性の分析など領域別に分かれる。  
- プロンプト設計：サブシステムごとの専用プロンプトと汎用プロンプト群（Chris Mason氏らによる初期セット）を用い、LLMに対して専門家チームのような振る舞いを指示する。  
- モデルと精度：公開インスタンスはGoogle提供の計算資源とLLMトークンで稼働。評価ではGemini 3.1 Proを用い、過去の「Fixed:」付きコミット1000件から53.6%のバグを特定。ただし出力は確率的で、ランで結果が変わる可能性があり、人間のレビュアーを置換する設計ではなく補助が目的。  
- ライセンスと公開：プロジェクトはLinux Foundation所属、Apache License 2.0で公開。現在のインスタンスはLKMLの投稿を自動でレビューしている。

## 実践ポイント
- まずリポジトリを確認して使い方・プロンプト設計を把握する（GitHub参照）。  
- 保守中のサブシステムに対してトリアージ目的で導入し、誤検知を見越して人手レビューと併用する。  
- CIに組み込み、Sashikoの指摘を自動でチケット化・テスト追加に結びつけるワークフローを作る。  
- 機密パッチは公開インスタンスに流さない運用ポリシーを整備する。  
- 日本語での貢献（プロンプト改善やサブシステム知見の共有）でコミュニティに参加すると効果的。
