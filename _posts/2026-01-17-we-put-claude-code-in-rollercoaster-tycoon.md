# We Put Claude Code in Rollercoaster Tycoon - Claude CodeをRollerCoaster Tycoonに組み込んでみた
AIが遊園地をプロデュースする──古典経営シムを「思考するコード」に任せたら何が起きるか

## 要約
海外の実験記事は、Anthropicの「Claude Code」を使って古典的な経営シミュレーションゲーム RollerCoaster Tycoon（RCT）を自動プレイさせる試みを紹介しています。画面を読み取り、目標を立て、操作コマンドを生成してループさせることで、AIが遊園地運営を学び実行します。

## この記事を読むべき理由
日本はテーマパーク産業、シミュレーションゲーム、ゲーム開発人材に強みがあります。AIがインタラクティブなUIを理解して自律的に操作できることは、ゲームAI、テスト自動化、プロシージャル生成、UX研究など幅広い応用につながるため、日本のエンジニアやゲーム開発者にとって実務的な示唆が得られます。

## 詳細解説
元記事の核は「大規模言語モデル（LLM）を単なるテキストではなく、実際のアプリケーション操作へつなげる」点にあります。一般的にこうした実験で採られる技術要素は次の通りです。

- 観測（Perception）
  - ゲーム画面のスクリーンショットを取得し、UI要素や数値（資金、評価、来客数など）をOCRや画像解析で抽出する。これにより「世界の状態」を生成する。
- 計画（Planning）と命令生成
  - Claude Codeのようなモデルに現在の状態と目標（例：来客数を増やす、収益を上げる）を与え、次に取るべき高レベル方針や具体的操作（建設、価格変更、従業員配置）を生成させる。出力は人が読める説明や、実行可能な構造化データ（JSONやコマンド列）にする。
- 実行（Actuation）
  - 生成されたコマンドをマウス/キーボード操作やゲーム内スクリプトに変換して実行する。自動化ツール（例：pyautogui等）でUIを操作するのが典型的。
- 評価とループ
  - コマンド実行後に再度観測して効果を評価し、報酬（評価点や収益）に基づいて方針を調整する。長期的な目標達成のために再計画を繰り返す。

技術的に面白い点は、LLMが「抽象的なゲーム設計判断」をコードや手順として出力できることです。従来のRL（強化学習）やルールベースAIと比べ、言語モデルは少量の指示や例から柔軟な戦略を生成できるため、探索空間が大きいゲームで迅速に有用な振る舞いを示します。一方で課題も明白です：リアルタイム性、部分観測、UI変化への脆弱性、長期的帰結の扱いなどです。

## 実践ポイント
- 小さく試す：まずは「画面→テキスト化→単一命令の自動実行→評価」の単純ループを作る。Parkの資金を読み取り、「価格を上げる」など単純操作から始めると成功しやすい。
- ツールセット案：画面キャプチャ（OpenCV）、OCR（Tesseract）、自動操作（pyautogui / OSのUI自動化）、LLM呼び出し（AnthropicのAPIや同等のモデル）。出力は構造化（JSON）にして実行系で検証する。
- プロンプト設計：状態の要約（数値＋重要なUIイベント）と「望むゴール」を明確に与え、出力形式を厳格に指定する（例：{action: "build", x: 12, y: 5, type: "coaster"}）。
- 評価指標を決める：短期（来客数、満足度）と長期（収益、リプレイ性）を分けて計測し、モデルの振る舞いを定量化する。
- 法的／倫理的注意：商用ゲームの自動操作や配信、改変はEULAに抵触する可能性があるため注意。学術・実験目的でも著作権と運用規約を確認すること。

日本向けの応用例としては、ゲームQAの自動化、遊園地運営シミュのプロトタイプ生成、教育用途（プログラミング学習のデモ）などが考えられます。古い名作ゲームを使った実験は、低コストでUI理解や長期計画の研究に適しています。

## 実践ポイント（即できること）
- 小さなPOCを作る手順：スクリーンショット→OCRで数値抽出→簡単なプロンプトでLLMに次アクションを出力させ→pyautoguiで実行→結果をログして改善。
- 日本の開発現場で役立つ用途：自動テスト、チュートリアル自動化、AIアシストでのデザイン案生成。
- 注意点：EULA遵守、個人情報や外部サービスの使用量に伴うコスト管理。

この記事を読んで、「手元のゲームや社内ツールで同じ仕組みを試してみよう」と思ったら、まずは画面情報の取得と構造化出力（JSON）を作るところから始めると再現性が高いです。
