# Interactive eBPF playground - 対話式 eBPF プレイグラウンド
魅力的なタイトル: ブラウザだけで学べる！実践で覚える eBPF ハンズオンプレイグラウンド

## 要約
ebpf.party はブラウザ上で eBPF プログラムを書き、コンパイルして動かしながら学べる対話型チュートリアルです。基礎から状態管理やカーネルプローブまで段階的に練習できます。

## この記事を読むべき理由
eBPF は観測性（トレーシング）、ネットワーク、セキュリティ分野で急速に普及しており、日本のクラウド／コンテナ運用、ネットワーク開発者にとって必須のスキルになりつつあります。ブラウザで手を動かしながら学べる環境は、初学者の壁を大きく下げます。

## 詳細解説
- eBPF の位置づけ：Linux カーネル内で安全にカスタムコードを実行できる仕組みです。これにより、カーネルレベルのイベントを低レイテンシで観測・操作できます。代表的な用途はシステムコールやネットワークパケットのトレース、プロセス・接続の追跡、セキュリティポリシーの適用などです。
- ebpf.party の構成（学習ロードマップ）：
  - Chapter 0（導入）：eBPF の基本概念とプラットフォームの使い方説明。
  - Chapter 1（概念の把握）：プロセスコンテキスト、イベントデータの読み取り、システムコールのトレースなど、まずは観測の基本を学びます。
  - Chapter 2（ステートフル eBPF）：Map を使った状態保持、複数プログラム間の連携、syscall 間の状態追跡、ネットワーク接続の追跡など応用的なパターン。
  - Chapter 3（カーネルプローブ）：kprobe/uprobes 経由でカーネル関数や TCP パケットを読み取る練習。
- インタラクティブの利点：サンドボックスされた環境で即座に試せるので、コンパイルエラーや verifier（検証器）での失敗を体験しながら理解できます。サイトは「issue を立てる」しくみもあり、演習追加やバグ報告でコミュニティに参加できます。
- 実装の仕組み（簡潔に）：サイトはブラウザ上でコード編集 → バックエンドで eBPF ツールチェインを使ってコンパイル／ロードするフローを提供していることが多く、ソースで「どう作っているか」の説明も用意されています。

## 実践ポイント
- 学習順序の推奨：Chapter 0 → 1（特に syscall トレース）→ 2（Maps と状態管理）→ 3（kprobe） の順で進めると理解が定着します。
- 小さく試す：まずは単純なトレース（例：あるシステムコールの発生をログする）から始め、徐々に Map を導入して状態を持たせる。失敗は学習の一部です。
- ローカルで追試する準備：ebpf.party で概念を掴んだら、実機や開発環境で bpftrace / bcc / libbpf などを使って再現してみましょう。簡単な bpftrace の例：
```bash
# bash
sudo bpftrace -e 'tracepoint:syscalls:sys_enter_execve { printf("%s %d\n", comm, pid); }'
```
- 日本市場での応用例：クラウド運用（コンテナ内のトラブルシューティング）、ネットワーク機器開発、インシデント対応やランタイムセキュリティの強化などに直結します。社内の observability チームや SRE に価値が高いスキルです。
- 安全面の注意：カーネル空間で動くため、本番環境で試す際は十分に注意し、権限や影響範囲を限定してください。
