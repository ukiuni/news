# The Four-Color Theorem 1852–1976 - 四色問題 1852–1976
たった4色で世界は足りるのか？──四色定理の150年にわたる発見とドラマ

## 要約
1852年に提起された「地図の領域を境界を共有する隣接領域が異なる色になるように4色で塗れるか」という問題は、1870年代の重要な帰着（立点を3つに揃える還元）、ケンペの鎖法、そして1976年のAppel–Hakenによるコンピュータ補助証明へと至る長い歴史を持つ。要点は「平面地図は4色で塗れる」という命題と、その証明過程で生まれた手法群である。

## この記事を読むべき理由
グラフ理論やアルゴリズム、可視化、周波数割当やレジスタ割当などの実用問題に直結する直感的かつ奥深い定理。歴史的経緯は数学・計算機科学がどう交差して現在の手法（特にコンピュータ支援）に至ったかを示す好例で、日本のエンジニアや教育者にも学びが多い。

## 詳細解説
- 発端と人物: 1852年にFrancis Guthrieが問いを投げ、Augustus De Morganらが議論を広めた。その後Arthur Cayleyが「複数の領域が交わる点にパッチをはめて常に3つ重なる（立点次数3の）地図に帰着できる」ことを示し、議論は「立点次数3（cubic maps）に限れば良い」と集約された。
- オイラーの式と小多角形の存在: 平面地図の頂点数$N$, 辺数$E$, 領域数$R$について $N - E + R = 2$ が成り立つ。各辺・面の次数を数えることで、少なくとも辺数が5以下の面（多角形）が存在することが示される。たとえば面の個数$c_k$を用いると次の関係が導かれる：  
  $$4c_2 + 3c_3 + 2c_4 + c_5 - c_7 - 2c_8 - \dots = 12.$$
  これにより「小さい次数の面が必ずある」ことが得られ、帰納法の起点になる。
- Kempe のアプローチと「Kempe鎖」: Alfred Kempeは1890年代に一度証明したと主張した。彼のアイデアは、局所的に色を交換する「Kempe鎖」によって四色に収めるというもの。鎖法は今も局所的な色修正手法として有用だが、Kempeの元の議論には後に反例が見つかり（Heawoodらによる修正と指摘）、完全な証明には至らなかった。
- コンピュータ補助証明（Appel–Haken, 1976）: 最終的にKenneth AppelとWolfgang Hakenは、多数の還元可能な局所配置（約千を超えるケース）をコンピュータで照合することで四色定理を証明した。これは大規模なケース分岐を機械的に検証する「コンピュータ支援証明」の先駆で、数学的証明のあり方に議論を呼んだ。鎖法や還元の考え方はその後のアルゴリズム設計にも影響を与えた。

## 実践ポイント
- 小さく試す: Python + networkxで平面グラフを作り、貪欲法や局所的な色交換（Kempe鎖の実装）を試すと理解が深まる。
- 可視化: GraphVizやmatplotlibで領域を色付けして挙動を観察する（教育・プレゼンに有効）。
- 応用分野: 周波数割当、スケジューリング、グラフ可視化、コンパイラのレジスタ割当などでヒントになる。
- 参考コード（試験的）:

```python
python
import networkx as nx
G = nx.grid_2d_graph(6,6)                 # 平面な格子グラフで試す
G = nx.convert_node_labels_to_integers(G)
colors = nx.coloring.greedy_color(G, strategy='largest_first')
print("使用色数:", len(set(colors.values())))
```

短時間で理解したい場合は、Kempe鎖のアイデア（隣接色の連結成分を丸ごと反転して空きを作る）を紙で追うことと、ネットでAppel–Haken論文の解説記事や再検証（後続の検証や簡素化）の歴史を参照することをおすすめします。
