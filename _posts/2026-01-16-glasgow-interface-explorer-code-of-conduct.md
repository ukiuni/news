# Glasgow Interface Explorer Code of Conduct - Glasgow Interface Explorer 行動規範
「技術も人も守る」— 多様性を前提に作られたオープンハードウェアの行動規範

## 要約
Glasgow Interface Explorerの行動規範は、長期的な共同開発を支えるために「安全で敬意ある場」を明確に定め、差別やハラスメントをほとんど容認しない立場を取っている。報告・解決の手順や運営者の責任範囲も具体的に示されている。

## この記事を読むべき理由
オープンソースのハード／組み込み分野は日本でも活発化しており、プロジェクト運営やコミュニティ形成の課題が直面する。Glasgowの規範は、多様性を前提にした実践的なモデルで、コミュニティ運営、トラブル防止、法的リスク回避の観点で参考になる。

## 詳細解説
- 背景と目的：Glasgowは長期的な公共資産として機器とコミュニティを維持することを目的とし、技術面だけでなく社会的側面（誰がどう参加できるか）を明文化している。  
- 包摂の立場：プロジェクトは多くをクィア、トランス、複数アイデンティティを持つ人々が築いており、その安全を最優先する姿勢を打ち出している。だれでも参加できるが、当事者性と敬意が要請される。  
- 禁止事項：性差別、性暴力的言動、人種差別、外国人排斥、トランス・クィア差別、ストーキング、ドキシング、同意を無視する行為など、ほぼ例外なく許容しないことを明確にしている。未成年やサバイバーを配慮して性的表現は避けるよう求める。  
- 法的・実務的姿勢：プロジェクトは最も寛容なライセンスで公開しているが、法的問題を招く投稿は避けるよう警告し、必要ならば貢献を拒否する場合があると明言している（外部機関への通報に依存しない方針も示唆）。  
- 解決プロセス：まずはプライベートな対話での解決を推奨し、失敗した場合はプロジェクトリーダーへ報告する。現行のリーダーは Catherine (whitequark) と Piotr (esden) で、報告時の秘密保持・迅速な調査が義務付けられている。対応は公平かつ比例的に行うとされる。  
- 文脈メモ：原文には感情的・修辞的な表現も含まれるが、実務面では「被害の防止」「参加者の安全確保」「運営の透明性」が中心である。

## 実践ポイント
- 自プロジェクトの行動規範を作るときは、禁止行為を具体列挙し、対応フローと責任者を明示する。  
- 規範は英語版だけでなく日本語訳を用意し、誰でも理解できる表現にする。  
- 報告チャネル（メール／フォーム／DM）とプライバシー保護の手順を整備する。  
- モデレーターやリーダー向けの対応ガイドライン（テンプレート応答、エスカレーション基準）を準備する。  
- 公開リポジトリやフォーラムで「法的に問題がある投稿を避ける」旨を明示し、必要に応じて外部法律相談の窓口を確保する。  
- 日本の文化や法制度に合わせた調整（個人情報保護、未成年者対応、刑事事件の扱い）を行う。

短くまとめると、技術プロジェクトは「コードだけでなく場の設計」も不可欠で、Glasgowの行動規範はその明快な実例と言える。日本の現場でもすぐ役立つ具体的な手順が多く含まれているため、コミュニティ運営のテンプレートとして参照をおすすめする。
